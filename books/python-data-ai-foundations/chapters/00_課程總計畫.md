# 2026 Python 數據分析與 AI 工程基礎：從資料思維到系統底盤

## 24 小時線上課程完整計畫

---

## 壹、課程核心主張

本課程以 Python 為核心主線，從資料分析出發，逐步建立物件導向、進階 Python、作業系統與計算機組織的底層直覺，幫助初學者理解軟體系統如何支撐 AI 應用。學生不只會寫分析程式，更會知道程式如何被組織、資料如何在系統中流動、模型如何依賴硬體與作業系統執行，最終銜接機器學習、深度學習與大數據應用。

**一句話定位：**
2026 年學 Python 做數據分析，不只是學一門語言，而是在替未來的機器學習、深度學習、統計建模、資料工程與 AI 應用，先搭一條共通底盤。

## 肆、為什麼 2026 還要學 Python？— 四大理由

### 1. 它仍是資料與 AI 的共通語言
Python 在 JetBrains/PSF 2024 調查中持續維持高能見度，被明確描述為 AI、資料科學與後端開發的重要語言。這不是單一圈子的偏好，而是跨研究、教學、產品原型、資料工程、AI 應用的共同選擇。

### 2. 生態不是單點強，而是整條鏈都能接
從 Jupyter Notebook 的互動分析 -> NumPy 數值計算 -> pandas 表格處理 -> scikit-learn 傳統 ML -> PyTorch 深度學習 -> PySpark 分散式處理，這種「從學習到產業」的連續性，仍然很難被單一工具取代。

### 3. 近五年它不是原地踏步，而是在升級
- pandas 2.0：Copy-on-Write + Arrow backend
- NumPy 2.0：自 2006 年後第一個 major release
- PyTorch 2.0：編譯式優化正式進入主流
- Python 3.13 + PEP 703：free-threaded 路線啟動

### 4. 資料分析已經不是報表工作，而是 AI 前處理工程
Anaconda 2024 報告顯示，AI adoption 持續升高，實務上很多工作不是先訓練模型，而是先處理資料、清理欄位、建立特徵、驗證品質。很多人不是輸在模型，是還沒把資料處理到能餵模型。

## 伍、課程雙主線架構

本課程採「雙主線交織」設計，而非先教完資料再補系統。

```
主線 A：資料與 AI 能力線          主線 B：軟體與系統底層線
─────────────────────          ─────────────────────
Python 基礎                    Computational thinking
NumPy / pandas                 OOP / module / package
EDA / 視覺化                    venv / dependency
統計思維                        exception / logging / I/O
ML 前導                        process / thread / memory
DL 前導                        CPU / GPU / execution model
Big Data 前導                   Linux / OS 基礎直覺
```

**交會點設計：**
- pandas 大表格 -> 連到 memory management
- PyTorch tensor -> 連到 GPU / autograd
- DataLoader -> 連到 process / I/O
- PySpark -> 連到 distributed execution
- 專案實作 -> 連到 OOP / package structure

## 捌、教學敘事節奏 — 六幕結構

整門課的敘事像一部電影，不是一本手冊。

### 第一幕：看見全貌 (M0)
你不是來學語法，你是來學電腦怎麼幫你理解世界、處理資料、驅動 AI。

### 第二幕：先學會用 Python 表達問題 (M1)
從變數、容器、流程控制到函式，先建立表達能力。

### 第三幕：讓程式開始有結構 (M2)
用 OOP、module、package，讓程式不是一團線，而是有骨架的系統。

### 第四幕：用資料分析看見模式 (M3 + M4)
NumPy、pandas、EDA、統計，讓資料說話。

### 第五幕：看見電腦底層怎麼支撐一切 (M5 + M6)
CPU、記憶體、process、thread、file system、I/O，把 AI 應用背後的機械室打開。

### 第六幕：走向 AI 應用 (M7)
ML、DL、PyTorch、PySpark，讓學生知道未來怎麼接。

## 玖、每個章節的投影片風格原則

### 原則 1：每頁只做一件事
不要一頁塞三個概念。一頁只回答一個問題。

### 原則 2：每個章節要有「高層視角頁」
像顧問簡報那種安靜但有力的金句頁：
- "Data is the raw material; Python is the operating language."
- "Analysis turns records into decisions."
- "Statistics turns observations into confidence."
- "Machine learning turns patterns into prediction."
- "Systems turn local scripts into scalable applications."

### 原則 3：節奏要有鬆緊（導演式節奏）
1. 先拉高視角：讓學生知道自己為什麼在這裡
2. 再落到工具：讓學生摸到實際操作
3. 中段給成就感：做出一個能看的分析結果
4. 後段再升維：告訴他這東西怎麼接到 ML / DL / Big Data
5. 結尾給道路圖：不讓學生下課後空掉

### 原則 4：底層概念用 AI 場景帶入
- 記憶體 -> 為什麼 DataFrame 爆掉
- process -> 為什麼 DataLoader 可以加速
- GPU -> 為什麼深度學習訓練快
- OOP -> 為什麼聊天機器人要有狀態管理

## 拾貳、小白教學四原則

### 原則 1：底層不等於先教最難
不要一上來就講 CPU pipeline、virtual memory page table、GIL 細節。底層課要教的是「能建立直覺的最小模型」。

### 原則 2：每個底層概念都要有 AI 場景
沒有場景的底層知識，對小白來說就是考古學。

### 原則 3：大量圖解
程式 -> CPU -> RAM -> Disk 流程圖、class/object 關係圖、process/thread 圖、notebook -> script -> package -> service 演進圖。

### 原則 4：目標是建立方向感，不是變專家
學生要知道系統在做什麼，未來遇到問題時知道該往哪裡查。先不要貪。
