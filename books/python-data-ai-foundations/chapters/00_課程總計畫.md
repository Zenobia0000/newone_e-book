## 壹、課程核心主張

本課程以 Python 為核心工具，從資料思維出發，帶領初學者建立完整的數據分析能力鏈：從理解資料結構、清洗轉換、視覺化探索、統計判斷，一路走到機器學習前導。學完之後，你不只會寫程式，還會知道資料該怎麼被理解、被整理、被驗證，最後被拿來做決策。

**一句話定位：**
2026 年學 Python 做數據分析，不只是學一門語言，而是在替未來的機器學習、深度學習、統計建模、資料工程與 AI 應用，先搭一條共通底盤。

## 肆、為什麼 2026 還要學 Python？— 四大理由

### 1. 它仍是資料與 AI 的共通語言
Python 在 JetBrains/PSF 2024 調查中持續維持高能見度，跨研究、教學、產品原型、資料工程、AI 應用的共同選擇。

### 2. 生態不是單點強，而是整條鏈都能接
Jupyter Notebook -> NumPy -> pandas -> scikit-learn -> PyTorch -> PySpark，這種「從學習到產業」的連續性，仍然很難被單一工具取代。

### 3. 近五年它不是原地踏步，而是在升級
pandas 2.0（Arrow backend）、NumPy 2.0（首個 major release）、PyTorch 2.0（編譯優化）、Python 3.13（free-threading 路線）。

### 4. 資料分析已經不是報表工作，而是 AI 前處理工程
Anaconda 2024 報告：AI adoption 持續升高，但真正拉開差距的是資料處理能力。

## 柒、教學敘事節奏 — 六幕結構

### 第一幕：看見全貌（M0）
你不是來學語法，你是來學用 Python 理解世界、處理資料、準備進入 AI 的入口。

### 第二幕：建立資料思維（M1）
資料有結構、有品質問題、有分析生命週期。先看懂資料，才能開始分析。

### 第三幕：掌握表達工具（M2）
用 Python 表達資料問題。變數、容器、迴圈、函式——只教分析真的會用到的。

### 第四幕：進入數值與表格世界（M3 + M4）
NumPy 給你陣列思維，pandas 給你表格操作力。這是資料分析的雙引擎。

### 第五幕：從表格到洞察（M5 + M6）
視覺化讓你看見模式，統計讓你判斷模式是否可信。從描述升級到判斷。

### 第六幕：走向 AI（M7）
機器學習不是魔法，是函數逼近。用 scikit-learn 體驗建模流程，然後看清未來的五條路。

## 拾、課程結束後的五條學習路線

### 路線 A：統計分析深化
機率分布 -> 抽樣理論 -> 假設檢定 -> 回歸分析 -> A/B Test

### 路線 B：機器學習
scikit-learn -> 特徵工程 -> 模型評估 -> 交叉驗證 -> 集成方法

### 路線 C：深度學習
線性代數 -> 張量觀念 -> PyTorch -> MLP / CNN / RNN / Transformer

### 路線 D：大數據 / 資料工程
SQL -> PySpark -> ETL -> Arrow -> DuckDB / Polars -> workflow orchestration

### 路線 E：AI 應用 / LLM 工程
向量資料處理 -> embeddings -> prompt workflow -> RAG -> agent pipeline
