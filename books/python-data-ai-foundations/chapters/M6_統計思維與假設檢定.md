### Slide 1：為什麼分析需要統計？

**核心訊息**：沒有統計，你只是在描述現象，不是在判斷現象。

**講師講解要點**：

- **情境題**：你做了一個新的 App 引導流程，A 版轉換率 12%，B 版轉換率 14%。B 版比較好嗎？
- 純描述的回答：「B 版高了 2 個百分點。」——這是觀察，不是判斷
- 統計的回答：「在樣本數 n=500 的情況下，這 2% 的差異在 95% 信賴水準下無法排除是隨機誤差造成的。」——這是判斷
- 統計的本質：**在資料永遠不完整的情況下，用嚴謹的方式量化我們對判斷的信心**
- 不用統計的代價：做了很多「看起來有效」但其實只是噪聲的決策，浪費資源，甚至反效果
- **統計不是數學課**：不是為了算出答案，而是為了知道答案有多可靠

**直覺範例**：

```
情境 A：兩組樣本各 50 人，A 組均值 100，B 組均值 105
→ 「5 的差異」可靠嗎？每組只有 50 人，波動很大，可能只是抽樣運氣

情境 B：兩組樣本各 50000 人，A 組均值 100，B 組均值 100.1
→ 「0.1 的差異」統計上顯著！但這個差異對業務有意義嗎？

統計回答的問題：「差異是真實的嗎？」
業務需要額外問的問題：「差異重要嗎？」
```

**視覺建議**：
左右對比：左側「純描述的世界」——一張柱狀圖，A=12%, B=14%，下方標注「看起來 B 贏了」。右側「有統計的世界」——同一張圖加上誤差棒（error bar），誤差棒重疊，下方標注「差異在信心範圍內無法確認」。兩個問題框對比：「哪個高？」vs「差異是真實的嗎？」

### Slide 2：母體、樣本與描述 vs 推論統計

**核心訊息**：統計不是計算器，是不確定性的語言。每個數字背後都有一個關於「我們能知道多少」的假設。

**講師講解要點**：

- **母體（Population）**：你真正想了解的全集——所有用戶、所有訂單、所有未來可能的實驗結果。我們幾乎永遠無法取得完整母體
- **樣本（Sample）**：我們實際拿到的那一部分。分析的對象是樣本，但判斷的目標是母體
- **母體參數 vs 樣本統計量**：
  - 母體均值 μ（mu）：未知，是我們想估計的目標
  - 樣本均值 x̄（x-bar）：已知，是我們手上的估計值
  - 標準誤（Standard Error）= σ / √n：衡量 x̄ 有多可能偏離 μ
- **描述統計（Descriptive Statistics）**：總結手上的樣本資料——均值、中位數、標準差、分佈形狀。目的：「描述我有什麼」
- **推論統計（Inferential Statistics）**：從樣本推斷母體——假設檢定、信賴區間、回歸分析。目的：「推論我不知道的」
- 關鍵直覺：樣本越大，樣本統計量越接近母體參數，不確定性越小

**概念對照表**：

| 概念 | 符號 | 性質 | 可得性 |
|------|------|------|--------|
| 母體均值 | μ（mu）| 固定但未知 | 幾乎不可得 |
| 樣本均值 | x̄ | 已知，隨樣本不同而變化 | 從資料計算 |
| 母體標準差 | σ（sigma）| 固定但未知 | 幾乎不可得 |
| 樣本標準差 | s | 已知，隨樣本不同而變化 | 從資料計算 |
| 標準誤 | SE = s/√n | 衡量樣本均值的不確定性 | 從資料計算 |

**程式碼範例**：

```python
import numpy as np
from scipy import stats

np.random.seed(42)
# 模擬「真實母體」（在現實中我們不知道這個）
population = np.random.normal(loc=500, scale=80, size=100_000)
mu_true = population.mean()
print(f"母體真實均值 μ = {mu_true:.2f}")

# 從母體中抽取不同大小的樣本
for n in [10, 50, 200, 1000]:
    sample = np.random.choice(population, size=n, replace=False)
    x_bar = sample.mean()
    se = sample.std() / np.sqrt(n)  # 標準誤
    print(f"  n={n:5d}: x̄={x_bar:.2f}, SE={se:.2f}, 偏差={abs(x_bar - mu_true):.2f}")

# 重複抽樣：展示樣本均值的分佈
sample_means = [np.random.choice(population, size=50).mean() for _ in range(1000)]
print(f"\n1000次重複抽樣（n=50）的樣本均值：")
print(f"  均值的均值 = {np.mean(sample_means):.2f}（接近母體 μ = {mu_true:.2f}）")
print(f"  均值的標準差 = {np.std(sample_means):.2f}（這就是標準誤的直覺）")
```

**視覺建議**：
四個方塊概念圖：母體（大圓虛線）、樣本（小圓實線）、描述統計（總結樣本）、推論統計（從樣本推估母體）。加入「n 越大，SE 越小」的視覺示意：多個小樣本均值形成的分佈，隨 n 增大而收窄。

### Slide 3：分佈直覺：三種你必須認識的分佈

**核心訊息**：不同的隨機現象有不同的自然形狀。選對分佈，你的模型才接地氣。

**講師講解要點**：

- **為什麼要認識分佈？**
  - 理解資料的「自然形狀」，才能做出正確的統計判斷
  - 假設資料符合常態分佈但實際上是偏態 → 統計結論全錯

- **常態分佈（Normal Distribution）**
  - 參數：μ（中心）、σ（寬度）
  - 為什麼到處都是？**中央極限定理**：大量獨立隨機因素疊加的結果，趨向常態
  - 現實例子：測量誤差、身高、考試成績
  - 68-95-99.7 法則：μ±1σ 包含 68%，μ±2σ 包含 95%，μ±3σ 包含 99.7%

- **二項分佈（Binomial Distribution）**
  - 參數：n（試驗次數）、p（成功機率）
  - 適用場景：「n 次獨立試驗中，成功 k 次的機率」
  - 現實例子：n 個用戶中有幾個轉換、n 個郵件中有幾個被點開
  - A/B Test 的轉換率比較就是二項分佈的應用

- **泊松分佈（Poisson Distribution）**
  - 參數：λ（單位時間/空間內的平均事件數）
  - 適用場景：「固定時間/空間內，罕見事件發生次數」
  - 現實例子：每小時客服來電數、每天系統錯誤次數、每週大單數量

**程式碼範例**：

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats
import seaborn as sns

sns.set_theme(style='whitegrid')
fig, axes = plt.subplots(1, 3, figsize=(15, 5))

# ---- 1. 常態分佈 ----
x = np.linspace(-4, 4, 300)
for mu, sigma, label, color in [(0, 1, 'μ=0, σ=1（標準）', 'steelblue'),
                                  (0, 2, 'μ=0, σ=2（較寬）', 'coral'),
                                  (1, 1, 'μ=1, σ=1（右移）', 'mediumseagreen')]:
    axes[0].plot(x, stats.norm.pdf(x, mu, sigma), label=label, linewidth=2, color=color)
axes[0].axvspan(-1, 1, alpha=0.1, color='steelblue')
axes[0].set_title('常態分佈\n（中央極限定理的自然結果）')
axes[0].set_xlabel('值'); axes[0].set_ylabel('機率密度')
axes[0].legend(fontsize=9)
axes[0].text(0, 0.15, '68%', ha='center', fontsize=10, color='steelblue')

# ---- 2. 二項分佈 ----
k = np.arange(0, 31)
for n, p, color, label in [(30, 0.3, 'steelblue', 'n=30, p=0.3（低轉換）'),
                             (30, 0.5, 'coral',     'n=30, p=0.5（均等）'),
                             (30, 0.7, 'mediumseagreen', 'n=30, p=0.7（高轉換）')]:
    pmf = stats.binom.pmf(k, n, p)
    axes[1].plot(k, pmf, 'o-', label=label, color=color, markersize=4, linewidth=1.5)
axes[1].set_title('二項分佈\n（n 次試驗中成功 k 次的機率）')
axes[1].set_xlabel('成功次數 k'); axes[1].set_ylabel('機率')
axes[1].legend(fontsize=9)

# ---- 3. 泊松分佈 ----
k2 = np.arange(0, 20)
for lam, color, label in [(2, 'steelblue', 'λ=2（低頻事件）'),
                            (5, 'coral',    'λ=5（中頻事件）'),
                            (10, 'mediumseagreen', 'λ=10（高頻事件）')]:
    pmf = stats.poisson.pmf(k2, lam)
    axes[2].plot(k2, pmf, 'o-', label=label, color=color, markersize=4, linewidth=1.5)
axes[2].set_title('泊松分佈\n（固定時間內事件發生次數）')
axes[2].set_xlabel('事件次數 k'); axes[2].set_ylabel('機率')
axes[2].legend(fontsize=9)

plt.suptitle('三種基礎分佈：直覺對照', fontsize=14, y=1.02)
plt.tight_layout(); plt.show()

# 中央極限定理示範
print("=== 中央極限定理示範 ===")
print("從右偏分佈（指數分佈）重複抽樣，樣本均值的分佈趨向常態：")
pop = np.random.exponential(scale=5, size=100_000)  # 右偏母體
fig2, axes2 = plt.subplots(1, 4, figsize=(14, 4))
axes2[0].hist(pop[:500], bins=30, color='coral', alpha=0.7)
axes2[0].set_title('原始分佈（右偏）')
for i, n in enumerate([5, 30, 100], 1):
    means = [np.random.choice(pop, n).mean() for _ in range(2000)]
    axes2[i].hist(means, bins=30, color='steelblue', alpha=0.7)
    axes2[i].set_title(f'樣本均值分佈（n={n}）')
plt.suptitle('中央極限定理：n 越大，樣本均值越接近常態', y=1.02)
plt.tight_layout(); plt.show()
```

### Slide 4：均值不夠用：為什麼要看分佈？

**核心訊息**：均值是一個點，分佈是完整的故事。兩組相同均值的資料，可以有完全不同的業務含義。

**講師講解要點**：

- **均值的陷阱**：比爾·蓋茲走進一間酒吧，酒吧裡所有人的「平均財富」立刻暴增——但沒有人變富了。均值被極端值拉偏
- **為什麼看分佈更重要**：
  - 分佈讓你看到「大多數人在哪裡」而不只是「中心在哪裡」
  - 雙峰分佈意味著存在兩個完全不同的用戶群體，用均值代表他們是荒唐的
  - 右偏分佈（收入、房價）中，均值 >> 中位數，用均值描述「典型用戶」會嚴重誤導
- **偏度（Skewness）的直覺**：
  - 正偏（右偏）：長尾在右，均值 > 中位數。收入、消費金額、留存天數
  - 負偏（左偏）：長尾在左，均值 < 中位數。考試成績（難度低時）
- **多種集中趨勢量數的對比**：
  - 均值：受極端值影響最大，代表「數學重心」
  - 中位數：不受極端值影響，代表「典型值」
  - 眾數：出現最多次的值，代表「最常見值」

**程式碼範例**：

```python
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from scipy import stats

sns.set_theme(style='whitegrid')
np.random.seed(42)

fig, axes = plt.subplots(2, 2, figsize=(13, 10))

# ---- 情境1：相同均值，不同分佈（Anscombe 精神）----
np.random.seed(42)
group_a = np.random.normal(500, 50, 500)                        # 單峰，中等分散
group_b = np.concatenate([np.random.normal(300, 30, 250),       # 雙峰
                           np.random.normal(700, 30, 250)])
# 強制讓兩組均值相同
group_b = group_b - group_b.mean() + group_a.mean()

axes[0,0].hist(group_a, bins=40, alpha=0.6, label=f'A 組（均值={group_a.mean():.0f}）', color='steelblue')
axes[0,0].hist(group_b, bins=40, alpha=0.6, label=f'B 組（均值={group_b.mean():.0f}）', color='coral')
axes[0,0].axvline(group_a.mean(), color='navy',    linestyle='--', linewidth=2)
axes[0,0].axvline(group_b.mean(), color='darkred', linestyle='--', linewidth=2)
axes[0,0].set_title('相同均值，完全不同的分佈\n（均值相同 ≠ 資料相同）')
axes[0,0].legend()

# ---- 情境2：右偏分佈 vs 均值 vs 中位數 ----
income = np.concatenate([np.random.lognormal(10, 0.5, 950),
                          np.random.uniform(5e6, 1e7, 50)])  # 超高收入者
mean_inc    = income.mean()
median_inc  = np.median(income)
mode_approx = stats.mode(income.astype(int))[0]
```

### Slide 5：信賴區間：「我有多確定？」

**核心訊息**：點估計只給你一個答案，信賴區間給你「答案有多可靠」的範圍。

**講師講解要點**：

- **為什麼需要信賴區間？**
  - 樣本均值 x̄ 只是一個點估計。每次重新抽樣都會得到不同的 x̄
  - 信賴區間告訴我們：「如果重複做 100 次這個研究，約有 95 次，真實的母體均值 μ 會落在這個區間內」
  - 它量化了我們的不確定性，讓決策更有根據

- **95% CI 的正確解讀**：
  - ✅ 正確：「在相同方法重複抽樣 100 次下，約有 95 個區間會包含真實母體均值」
  - ❌ 錯誤：「母體均值有 95% 的機率落在這個區間」（母體均值是固定的，不是隨機的！）
  - ❌ 錯誤：「95% 的資料落在這個區間」（這是說分佈範圍，不是說 CI）

- **CI 越窄越好嗎？**
  - 更窄的 CI = 更高的確定性，但需要更大的樣本數
  - 寬 CI + 不包含零 → 有差異但不精確
  - 窄 CI + 不包含零 → 有差異且很精確（最好的情況）
  - 任何 CI 包含零 → 無法排除「沒有差異」

- **CI 的計算直覺**：CI = x̄ ± z × SE，其中 SE = s / √n

**程式碼範例**：

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats
import seaborn as sns

sns.set_theme(style='whitegrid')
np.random.seed(42)

# ---- 示範1：信賴區間的構建邏輯 ----
population_mean = 500  # 假設我們知道這個（現實中不知道）
population = np.random.normal(population_mean, 80, 100_000)

n = 50
num_experiments = 100
contained = 0

fig, ax = plt.subplots(figsize=(10, 8))
for i in range(num_experiments):
    sample = np.random.choice(population, size=n)
    x_bar  = sample.mean()
    se     = sample.std() / np.sqrt(n)
    ci_low  = x_bar - 1.96 * se
    ci_high = x_bar + 1.96 * se

    color = 'steelblue' if ci_low <= population_mean <= ci_high else 'red'
    if color == 'steelblue':
        contained += 1
    ax.plot([ci_low, ci_high], [i, i], color=color, alpha=0.6, linewidth=1.5)
    ax.plot(x_bar, i, 'o', color=color, markersize=3)

ax.axvline(population_mean, color='black', linestyle='--', linewidth=2, label=f'真實母體均值 μ={population_mean}')
ax.set_xlabel('數值範圍')
ax.set_title(f'100 個 95% 信賴區間\n藍色={contained} 個包含了真實 μ，紅色={100-contained} 個未包含\n（理論上應有約 95 個包含）')
ax.legend(); plt.tight_layout(); plt.show()

# ---- 示範2：樣本數對 CI 寬度的影響 ----
sample_from_pop = np.random.choice(population, size=1000)  # 從母體抽取大樣本
print("=== 樣本數 vs 信賴區間寬度 ===")
for n_size in [10, 30, 100, 500, 1000]:
    sample_n = sample_from_pop[:n_size]
    ci = stats.t.interval(0.95, df=n_size-1,
                           loc=sample_n.mean(),
                           scale=stats.sem(sample_n))
    width = ci[1] - ci[0]
    print(f"  n={n_size:4d}: CI=[{ci[0]:.1f}, {ci[1]:.1f}], 寬度={width:.1f}, 樣本均值={sample_n.mean():.1f}")

# ---- 示範3：計算並視覺化單一樣本的 CI ----
sample = np.random.normal(520, 75, 80)
ci_95 = stats.t.interval(0.95, df=len(sample)-1, loc=sample.mean(), scale=stats.sem(sample))
ci_99 = stats.t.interval(0.99, df=len(sample)-1, loc=sample.mean(), scale=stats.sem(sample))
print(f"\n樣本（n={len(sample)}）：均值={sample.mean():.1f}, SE={stats.sem(sample):.1f}")
print(f"95% CI: [{ci_95[0]:.1f}, {ci_95[1]:.1f}]  寬度={ci_95[1]-ci_95[0]:.1f}")
print(f"99% CI: [{ci_99[0]:.1f}, {ci_99[1]:.1f}]  寬度={ci_99[1]-ci_99[0]:.1f}")
```

**視覺建議**：
重點展示 100 個 CI 的圖，大多數藍色（包含真實值），少數紅色（未包含），直覺理解「95% CI 的含義」。旁邊加一個「CI 寬度 vs 樣本數」的反比關係圖。

### Slide 6：假設檢定：差異可靠嗎？

**核心訊息**：假設檢定問的不是「有沒有差異」，而是「這個差異可靠到足以讓我行動嗎」。

**講師講解要點**：

- **假設檢定的核心邏輯（法庭類比）**：
  - H0（虛無假設）= 無罪推定：預設「沒有差異，你看到的差異只是抽樣噪聲」
  - H1（對立假設）= 起訴理由：「有差異，而且這個差異是真實的」
  - p-value = 反對無罪的證據強度：「如果 H0 是真的，看到這麼大差異的機率是多少？」
  - 顯著水準 α = 定罪門檻：通常設為 0.05
  - 結論：p < α → 拒絕 H0（定罪）；p ≥ α → 無法拒絕 H0（無罪釋放）

- **假設檢定的五步驟**：
  1. 設定 H0 和 H1（H0 永遠是「沒有效果」的假設）
  2. 選擇顯著水準 α（最常用 0.05，醫療或飛安用 0.01）
  3. 收集資料，計算檢定統計量（t 值、z 值、χ² 值）
  4. 計算 p-value
  5. 根據 p vs α 做決定

- **Type I Error vs Type II Error**：
  - Type I（α 錯誤，假陽性）：H0 是真的，但我們拒絕了它。「冤枉好人」
  - Type II（β 錯誤，假陰性）：H0 是假的，但我們沒有拒絕它。「放跑壞人」
  - 統計功效（Power）= 1 - β：正確拒絕假 H0 的機率，目標通常 ≥ 0.8

**假設設定練習**：

```
業務問題：「新的推薦演算法讓用戶停留時間更長嗎？」

H0：新演算法的平均停留時間 ≤ 舊演算法的平均停留時間
    （數學表示：μ_new ≤ μ_old，即差異 ≤ 0）

H1：新演算法的平均停留時間 > 舊演算法的平均停留時間
    （數學表示：μ_new > μ_old，即差異 > 0）

顯著水準：α = 0.05
判斷：如果 p < 0.05，我們有足夠證據說新演算法更好。
```

**視覺建議**：
流程圖：情境描述 → 設定 H0/H1 → 收集資料 → 計算統計量 → 計算 p-value → 與 α 比較 → 做決定。旁邊加「法庭類比」對照表，以及 Type I/II Error 的 2x2 矩陣（真實情況 × 我們的決定）。

### Slide 7：p-value 的正確解讀與常見陷阱

**核心訊息**：p < 0.05 不代表差異很大，不代表 H0 是假的，也不代表結果可以重現。

**講師講解要點**：

- **p-value 的正確定義**：
  「如果 H0 為真（真的沒有差異），那麼我們觀察到這麼大（或更大）差異的機率。」
  p = 0.03 意思是：「如果根本沒有差異，我有 3% 的機率看到這麼大的差異。」

- **常見誤解逐一破解**：

| 誤解 | 正確理解 |
|------|---------|
| p < 0.05 代表差異很大 | 不是。p 只說差異是否可靠，不說差異有多大 |
| p = 0.03 代表 H0 有 3% 機率是真的 | 不是。p 是在 H0 為真的條件下計算的，不是 H0 為真的機率 |
| p > 0.05 代表 H0 是真的 | 不是。只是證據不足以拒絕 H0，而不是證明 H0 |
| p < 0.05 代表結果一定能重現 | 不是。樣本波動仍然存在 |
| α = 0.05 是自然法則 | 不是。這是一個慣例，應根據決策代價調整 |

- **單尾 vs 雙尾檢定**：
  - 雙尾（two-tailed）：問「有沒有差異」（方向未知）
  - 單尾（one-tailed）：問「是不是比較大/小」（方向已知）
  - 建議：除非有強力理論依據，預設用雙尾檢定，更保守

**視覺建議**：
左側顯示常態分佈，標注臨界區域（拒絕 H0 的區域）和 p-value 的位置。右側顯示「誤解清單」，每個誤解用紅色 ✗ 標記，正確理解用綠色 ✓ 標記。底部加一個「p-value 單頁速查卡」。

### Slide 8：效果量：統計顯著 vs 實際顯著

**核心訊息**：百萬樣本能讓任何微小差異都「統計顯著」——但 0.001% 的轉換率提升值得全面改版嗎？

**講師講解要點**：

- **為什麼需要效果量（Effect Size）？**
  - 統計顯著性受樣本數影響極大：n 足夠大，任何差異都會 p < 0.05
  - 效果量衡量差異的「實際大小」，獨立於樣本數
  - 決策應該同時考慮：統計顯著（差異是真實的嗎？）AND 實際顯著（差異夠大嗎？）

- **Cohen's d（連續變數效果量）**：
  - d = (μ₁ - μ₂) / s_pooled
  - 解讀：d = 0.2（小效果），d = 0.5（中等效果），d = 0.8（大效果）
  - 直覺：「兩組均值差了幾個標準差？」

- **業務決策四象限**：

| | 統計顯著（p < 0.05）| 統計不顯著（p ≥ 0.05）|
|---|---|---|
| **實際顯著（大效果量）** | 上線！有力的證據支持大改善 | 延長測試，或增加樣本數 |
| **實際不顯著（小效果量）** | 差異真實但微小，需評估 ROI | 維持現狀，沒有證據也沒有意義 |

**程式碼範例**：

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats
import seaborn as sns

sns.set_theme(style='whitegrid')
np.random.seed(42)

# ---- 示範：樣本數如何讓微小差異「顯著」----
print("=== 樣本數對 p-value 的影響（效果量固定）===")
true_diff = 0.5   # 真實差異只有 0.5 元
std       = 50.0  # 標準差 50 元

for n in [30, 100, 500, 2000, 10000, 100000]:
    # 模擬兩組資料（差異很小）
    group_a = np.random.normal(500, std, n)
    group_b = np.random.normal(500 + true_diff, std, n)
    t_stat, p_val = stats.ttest_ind(group_a, group_b)
    cohens_d = true_diff / std  # 效果量
    print(f"  n={n:7d}: p={p_val:.4f} {'*** 顯著' if p_val < 0.05 else '    不顯著'}, "
          f"Cohen's d={cohens_d:.3f}（{'大' if cohens_d > 0.8 else '中' if cohens_d > 0.5 else '小' if cohens_d > 0.2 else '微小'}）")

# ---- 視覺化：相同效果量，不同樣本數的 p-value ----
fig, axes = plt.subplots(1, 2, figsize=(13, 5))

sample_sizes = [30, 50, 100, 200, 500, 1000, 5000, 10000]
p_values     = []
for n in sample_sizes:
    a = np.random.normal(500, 50, n)
    b = np.random.normal(501, 50, n)    # 效果量極小：d ≈ 0.02
    _, p = stats.ttest_ind(a, b)
    p_values.append(p)

axes[0].plot(sample_sizes, p_values, 'o-', color='steelblue', linewidth=2)
axes[0].axhline(0.05, color='red', linestyle='--', linewidth=1.5, label='α = 0.05')
axes[0].set_xscale('log')
axes[0].set_xlabel('樣本數（對數軸）'); axes[0].set_ylabel('p-value')
axes[0].set_title('效果量極小（d≈0.02）：\n樣本大到一定程度必然顯著')
axes[0].legend()

# Cohen's d 直覺視覺化
x = np.linspace(-4, 6, 300)
for d, label, color in [(0.2, "小效果 (d=0.2)", 'steelblue'),
                         (0.5, "中效果 (d=0.5)", 'coral'),
                         (0.8, "大效果 (d=0.8)", 'mediumseagreen')]:
    axes[1].plot(x, stats.norm.pdf(x, 0, 1), color=color, linewidth=2, linestyle='--', alpha=0.5)
    axes[1].plot(x, stats.norm.pdf(x, d, 1), color=color, linewidth=2, label=label)
axes[1].axvline(0, color='black', linestyle=':', linewidth=1)
axes[1].set_title("Cohen's d 直覺：兩組分佈的重疊程度")
axes[1].set_xlabel('標準差單位'); axes[1].legend()

plt.suptitle('統計顯著 vs 實際顯著', fontsize=14, y=1.01)
plt.tight_layout(); plt.show()
```

**視覺建議**：
左圖展示「樣本數對 p-value 的影響」（效果量固定但樣本越大越顯著），右圖展示 Cohen's d 的直覺（兩組分佈重疊程度）。底部放決策四象限表格。

### Slide 9：scipy.stats 實戰：三種常用檢定

**核心訊息**：用程式碼落地統計判斷，掌握 scipy.stats 的三個核心 API。

**講師講解要點**：

- **單樣本 t 檢定**：我的樣本均值和目標值（或歷史值）有沒有顯著差異？
  - 使用：`stats.ttest_1samp(sample, popmean=target)`
- **獨立雙樣本 t 檢定**：兩組樣本的均值有沒有顯著差異？
  - 使用：`stats.ttest_ind(group_a, group_b, equal_var=False)`
  - 注意：`equal_var=False` 使用 Welch's t-test，不假設兩組變異數相等（更保守，更常用）
- **卡方獨立性檢定**：兩個類別變數之間有沒有關聯？
  - 使用：`stats.chi2_contingency(contingency_table)`
  - 適合：點擊率 vs 設備類型、轉換率 vs 來源渠道

**完整程式碼範例**：

```python
import numpy as np
from scipy import stats
import pandas as pd
import warnings
warnings.filterwarnings('ignore')

np.random.seed(42)
print("=" * 60)
print("單樣本 t 檢定：本月 NPS 是否達到目標 70 分？")
print("=" * 60)
nps_scores = np.random.normal(72, 15, 80)  # 樣本 NPS
target_nps = 70
t_stat, p_val = stats.ttest_1samp(nps_scores, popmean=target_nps)
print(f"樣本均值: {nps_scores.mean():.2f}")
print(f"t 統計量: {t_stat:.3f}")
print(f"p-value:  {p_val:.4f}")
if p_val < 0.05:
    direction = "高於" if nps_scores.mean() > target_nps else "低於"
    print(f"結論: 顯著 → 樣本 NPS 顯著{direction}目標 {target_nps} 分（95% 信心水準）")
else:
    print(f"結論: 不顯著 → 無法確認樣本 NPS 與目標 {target_nps} 有顯著差異")

print("\n" + "=" * 60)
print("獨立雙樣本 t 檢定：桌機 vs 手機用戶的平均訂單金額")
print("=" * 60)
desktop_orders = np.random.normal(650, 120, 150)
mobile_orders  = np.random.normal(580, 160, 300)  # 手機樣本更多
t_stat, p_val = stats.ttest_ind(desktop_orders, mobile_orders, equal_var=False)
cohens_d = (desktop_orders.mean() - mobile_orders.mean()) / \
           np.sqrt((desktop_orders.std()**2 + mobile_orders.std()**2) / 2)
print(f"桌機均值: {desktop_orders.mean():.2f}（n={len(desktop_orders)}）")
print(f"手機均值: {mobile_orders.mean():.2f}（n={len(mobile_orders)}）")
print(f"差異: {desktop_orders.mean() - mobile_orders.mean():.2f}")
print(f"t 統計量: {t_stat:.3f},  p-value: {p_val:.4f}")
print(f"Cohen's d: {cohens_d:.3f} ({'大' if abs(cohens_d) > 0.8 else '中' if abs(cohens_d) > 0.5 else '小'}效果量)")
if p_val < 0.05:
    print("結論: 桌機和手機用戶的訂單金額有顯著差異")
    print(f"      業務意義：差異 {desktop_orders.mean()-mobile_orders.mean():.0f} 元，"
          f"效果量 {abs(cohens_d):.2f}，{'值得' if abs(cohens_d) > 0.5 else '需評估'}採取行動")
else:
    print("結論: 無法確認桌機和手機用戶的訂單金額有顯著差異")

print("\n" + "=" * 60)
print("卡方檢定：轉換率是否與獲客渠道有關聯？")
print("=" * 60)
# 列聯表：渠道（列）vs 是否轉換（欄）
contingency = pd.DataFrame({
    '未轉換': [850, 720, 680, 195],
    '已轉換': [150, 280, 120, 105]
}, index=['organic', 'paid', 'social', 'referral'])
print("列聯表：")
print(contingency)
print(f"\n各渠道轉換率：")
print((contingency['已轉換'] / contingency.sum(axis=1)).round(3))

chi2, p_val, dof, expected = stats.chi2_contingency(contingency)
print(f"\nχ² 統計量: {chi2:.3f},  df={dof}")
print(f"p-value: {p_val:.6f}")
if p_val < 0.05:
    print("結論: 轉換率與獲客渠道有顯著關聯（渠道質量確實不同）")
else:
    print("結論: 無法確認轉換率與獲客渠道有關聯")
```

### Slide 10：A/B Test 完整案例：從設計到決策

**核心訊息**：A/B Test 是用統計把產品決策從「感覺上比較好」變成「數據上可以確認更好」。

**業務情境**：
電商平台在考慮改版結帳頁面（簡化步驟），預期可提升轉換率。
目前轉換率：18%。希望能提升至 21%（效果量 +3 個百分點）。

**完整 A/B Test 流程**：

```python
import numpy as np
from scipy import stats
import matplotlib.pyplot as plt
import seaborn as sns

sns.set_theme(style='whitegrid')
np.random.seed(42)

# ========== 步驟 1：實驗設計 ==========
print("=" * 60)
print("步驟 1：實驗設計")
print("=" * 60)

p_control   = 0.18   # 控制組轉換率（舊版）
p_treatment = 0.21   # 目標：實驗組轉換率（新版）
alpha       = 0.05   # 顯著水準
power       = 0.80   # 統計功效（80% 的機率偵測到真實差異）

# 樣本數計算（using 近似公式）
def calc_sample_size(p1, p2, alpha=0.05, power=0.80):
    z_alpha2 = stats.norm.ppf(1 - alpha/2)  # 雙尾
    z_beta   = stats.norm.ppf(power)
    p_bar    = (p1 + p2) / 2
    n = ((z_alpha2 * np.sqrt(2 * p_bar * (1 - p_bar)) +
          z_beta  * np.sqrt(p1*(1-p1) + p2*(1-p2))) / (p2 - p1)) ** 2
    return int(np.ceil(n))

n_required = calc_sample_size(p_control, p_treatment, alpha, power)
print(f"控制組轉換率: {p_control:.0%}")
print(f"目標實驗組轉換率: {p_treatment:.0%}（最小可偵測效果 MDE = {p_treatment-p_control:.0%}）")
print(f"顯著水準 α: {alpha}")
print(f"統計功效: {power:.0%}")
print(f"每組所需樣本數: {n_required}")
print(f"測試總流量: {n_required * 2}（兩組各 {n_required}）")
print(f"若每日流量 5000，測試需 {n_required * 2 / 5000:.1f} 天")

# ========== 步驟 2：收集資料（模擬）==========
print("\n" + "=" * 60)
print("步驟 2：收集資料")
print("=" * 60)
n_actual  = n_required  # 達到設計樣本數
control   = np.random.binomial(1, p_control,   n_actual)
treatment = np.random.binomial(1, p_treatment + 0.005, n_actual)  # 加一點噪聲

conv_c = control.mean()
conv_t = treatment.mean()
print(f"控制組（A，舊版）: n={n_actual}, 轉換率={conv_c:.2%}")
print(f"實驗組（B，新版）: n={n_actual}, 轉換率={conv_t:.2%}")
print(f"觀察到的差異: {conv_t - conv_c:+.2%}")

# ========== 步驟 3：假設檢定 ==========
print("\n" + "=" * 60)
print("步驟 3：假設檢定")
print("=" * 60)
print("H0：實驗組轉換率 ≤ 控制組轉換率（新版沒有更好）")
print("H1：實驗組轉換率 > 控制組轉換率（新版更好）")

# 比例 z 檢定
from statsmodels.stats.proportion import proportions_ztest
count = np.array([treatment.sum(), control.sum()])
nobs  = np.array([n_actual, n_actual])
z_stat, p_val = proportions_ztest(count, nobs, alternative='larger')

print(f"\nz 統計量: {z_stat:.3f}")
print(f"p-value: {p_val:.4f}")

# 效果量
effect_size = conv_t - conv_c
relative_lift = (conv_t - conv_c) / conv_c
print(f"絕對提升: {effect_size:+.2%}")
print(f"相對提升: {relative_lift:+.1%}")

# ========== 步驟 4：信賴區間 ==========
print("\n" + "=" * 60)
print("步驟 4：95% 信賴區間（差異）")
print("=" * 60)
from statsmodels.stats.proportion import confint_proportions_2indep
ci = confint_proportions_2indep(treatment.sum(), n_actual,
                                 control.sum(),   n_actual,
                                 method='agresti-caffo')
print(f"差異的 95% CI: [{ci[0]:.3%}, {ci[1]:.3%}]")
if ci[0] > 0:
    print("CI 下界 > 0，差異方向確定（不含零）")
else:
    print("CI 包含零，差異方向不確定")

# ========== 步驟 5：業務決策 ==========
print("\n" + "=" * 60)
print("步驟 5：業務決策")
print("=" * 60)

stat_sig    = p_val < alpha
practical_sig = abs(effect_size) >= 0.02  # 業務要求：至少提升 2 個百分點

if stat_sig and practical_sig:
    decision = "上線新版本"
    reason   = f"統計顯著（p={p_val:.4f}<{alpha}）且業務顯著（提升{effect_size:+.2%} ≥ 2%門檻）"
elif stat_sig and not practical_sig:
    decision = "評估 ROI 後決定"
    reason   = f"統計顯著但業務效果微小（提升{effect_size:+.2%} < 2%門檻）"
elif not stat_sig and practical_sig:
    decision = "延長測試時間或增加樣本數"
    reason   = f"效果量夠大但統計不顯著（p={p_val:.4f}），可能需要更多資料"
else:
    decision = "維持舊版"
    reason   = f"統計不顯著（p={p_val:.4f}）且效果量小，無足夠證據改版"

print(f"決策：{decision}")
print(f"理由：{reason}")

# ========== 視覺化 ==========
fig, axes = plt.subplots(1, 2, figsize=(13, 5))

# 轉換率對比
bars = axes[0].bar(['控制組（舊版）', '實驗組（新版）'], [conv_c, conv_t],
                   color=['#3498db', '#2ecc71'], alpha=0.85, width=0.5)
axes[0].axhline(conv_c, color='blue', linestyle='--', alpha=0.5)
for bar, val in zip(bars, [conv_c, conv_t]):
    axes[0].text(bar.get_x() + bar.get_width()/2, val + 0.003, f'{val:.2%}',
                 ha='center', fontsize=12, fontweight='bold')
axes[0].set_title(f'A/B Test 結果\np={p_val:.4f}，差異={effect_size:+.2%}，{"顯著✓" if stat_sig else "不顯著✗"}')
axes[0].set_ylabel('轉換率'); axes[0].set_ylim(0, max(conv_c, conv_t) * 1.3)

# 決策矩陣
matrix_data = [['上線！\n強力支持', '延長測試\n增加樣本'],
               ['評估 ROI\n再決定',   '維持現狀\n無足夠證據']]
colors_matrix = [['#2ecc71', '#f39c12'], ['#3498db', '#95a5a6']]

axes[1].axis('off')
table = axes[1].table(cellText=matrix_data,
                       rowLabels=['實際顯著\n(大效果量)', '實際不顯著\n(小效果量)'],
                       colLabels=['統計顯著\n(p < 0.05)', '統計不顯著\n(p ≥ 0.05)'],
                       cellColours=colors_matrix,
                       loc='center', cellLoc='center')
table.scale(1.5, 3)
table.auto_set_font_size(False); table.set_fontsize(10)
axes[1].set_title('A/B Test 決策四象限', fontsize=12, pad=20)

# 標注當前位置
current_row = 0 if practical_sig else 1
current_col = 0 if stat_sig else 1
print(f"\n本案例落在決策矩陣的：第 {current_row+1} 列，第 {current_col+1} 欄")

plt.suptitle('A/B Test 完整流程：從設計到決策', fontsize=14, y=1.01)
plt.tight_layout(); plt.show()
```

上方顯示 A/B Test 全流程（五個步驟的標籤框），下方左側是轉換率柱狀圖（含誤差棒），右側是決策四象限矩陣，用顏色標注當前案例落在哪個象限。

### Slide 11：多重比較與相關性陷阱

**核心訊息**：同時測試 20 個假設，必然有一個「顯著」——這不是統計的成功，是多重比較的失敗。

**講師講解要點**：

**多重比較問題（Multiple Testing Problem）**：

- 單次檢定，α = 0.05：假陽性率 = 5%
- 同時做 20 個獨立檢定：至少一個假陽性的機率 = 1 - (0.95)^20 ≈ 64%
- 業務現場的常見陷阱：同時測試「點擊率、轉換率、客均價、停留時間、跳出率...」20 個指標
- 解決方案：**Bonferroni 校正**（最保守）：α' = α / m，其中 m 是檢定次數
  - 做 20 個檢定時，每個的顯著水準降為 0.05 / 20 = 0.0025
- 更實際的做法：**事前聲明主要指標**（pre-registration），只有主要指標需要嚴格校正

**相關性與因果關係的界限**：

- 觀察到 X 和 Y 相關，可能的原因：
  1. X 導致 Y（X → Y）
  2. Y 導致 X（反向因果）
  3. 有第三變數 Z 同時導致 X 和 Y（混淆變數）
  4. 純屬巧合（偽相關）
- 隨機對照實驗（RCT）才能建立因果關係，因為隨機分組消除了混淆變數
- A/B Test 是在網際網路公司最接近 RCT 的方法

**程式碼範例**：

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats
import seaborn as sns

sns.set_theme(style='whitegrid')
np.random.seed(42)

# ---- 多重比較問題模擬 ----
print("=== 多重比較問題：模擬 10000 次實驗 ===")
print("真實情況：A、B 兩組完全相同（H0 都是真的）")

num_simulations = 10000
num_tests       = 20  # 每次實驗同時做 20 個檢定
false_positive_any = 0

for _ in range(num_simulations):
    found_any_significant = False
    for _ in range(num_tests):
        a = np.random.normal(0, 1, 50)
        b = np.random.normal(0, 1, 50)  # 兩組來自同一分佈！
        _, p = stats.ttest_ind(a, b)
        if p < 0.05:
            found_any_significant = True
            break
    if found_any_significant:
        false_positive_any += 1

print(f"\n在 {num_simulations} 次實驗中：")
print(f"  至少有一個假陽性：{false_positive_any/num_simulations:.1%}（理論值 64%）")
print(f"  理論計算：1 - (0.95)^20 = {1-(0.95)**20:.1%}")

print("\n=== Bonferroni 校正效果 ===")
alpha_adjusted = 0.05 / num_tests
print(f"原始 α: 0.05")
print(f"校正後 α: {alpha_adjusted:.4f}（= 0.05 / {num_tests}）")
print(f"代價：更難偵測到真實的差異（Type II Error 增加）")

# ---- 視覺化多重比較問題 ----
fig, axes = plt.subplots(1, 2, figsize=(13, 5))

num_tests_range = range(1, 51)
fp_probs = [1 - (1 - 0.05)**k for k in num_tests_range]
axes[0].plot(num_tests_range, fp_probs, color='coral', linewidth=2)
axes[0].axhline(0.05, color='steelblue', linestyle='--', linewidth=1.5, label='單次檢定 5%')
axes[0].axhline(0.64, color='red',       linestyle=':', linewidth=1.5,  label='20 次檢定 64%')
axes[0].fill_between(num_tests_range, fp_probs, 0.05, alpha=0.2, color='coral')
axes[0].set_xlabel('同時進行的檢定次數')
axes[0].set_ylabel('至少一個假陽性的機率')
axes[0].set_title('多重比較問題：\n檢定次數越多，誤報風險越高')
axes[0].legend(); axes[0].set_ylim(0, 1)

# 偽相關（Spurious Correlation）示範
years = np.arange(1999, 2010)
# 真實偽相關例子（戲謔性）：使用相關但荒謬的資料
pool_drownings   = [109, 102, 102, 98, 85, 95, 96, 98, 123, 94, 102][:len(years)]
cage_movies      = [2, 2, 2, 3, 1, 1, 2, 3, 4, 1, 4][:len(years)]

ax2 = axes[1]
color_a, color_b = 'steelblue', 'coral'
ln1 = ax2.plot(years, pool_drownings, 'o-', color=color_a, linewidth=2, label='泳池溺水人數')
ax2.set_xlabel('年份'); ax2.set_ylabel('泳池溺水人數', color=color_a)
ax2_twin = ax2.twinx()
ln2 = ax2_twin.plot(years, cage_movies, 's--', color=color_b, linewidth=2, label='Nicolas Cage 電影數')
ax2_twin.set_ylabel('Nicolas Cage 電影數', color=color_b)

r, p = stats.pearsonr(pool_drownings[:len(years)], cage_movies[:len(years)])
ax2.set_title(f'偽相關示例\nr = {r:.2f}，p = {p:.3f}\n（「顯著相關」但完全不是因果關係）')

lns = ln1 + ln2
labels = [l.get_label() for l in lns]
ax2.legend(lns, labels, loc='upper left')

plt.suptitle('多重比較 + 相關性陷阱', fontsize=14, y=1.01)
plt.tight_layout(); plt.show()

# ---- 因果關係的判斷標準 ----
print("\n=== 從相關到因果：判斷清單 ===")
causal_criteria = {
    "時序性": "X 在 Y 之前發生？（因必在果之前）",
    "相關性": "X 和 Y 確實相關？",
    "排除混淆": "有無第三變數 Z 同時影響 X 和 Y？",
    "機制": "有合理的業務機制解釋 X 如何導致 Y？",
    "隨機化": "理想情況：X 是被隨機分配的（A/B Test）",
}
for k, v in causal_criteria.items():
    print(f"  [{k}] {v}")
```

**視覺建議**：
左圖展示「檢定次數 vs 假陽性率」的曲線（隨 m 增大快速上升），右圖展示一個著名的偽相關例子（Nicolas Cage 電影數 vs 泳池溺水人數），用幽默的方式強化「相關 ≠ 因果」的印象。

**講師標準答案要點**：
- 點擊率：統計顯著（p=0.021<0.05），但效果量 h=0.033 屬於微小效果
- 跳出率的 p=0.03 是 8 個次要指標同時比較的結果，需要 Bonferroni 校正後 α=0.00625，p=0.03 不再顯著
- 建議：需評估「0.9% 的點擊率提升」對業務的實際價值，若業務價值明確則可上線

## 模組總結

### 本模組的核心主線

統計不是數學課，是決策工具。
從「均值有差異」到「差異在統計上可靠」到「差異在業務上值得行動」——這三步缺一不可。

### 三個不能忘的原則

1. **均值不夠，看分佈**：相同均值可以有完全不同的故事
2. **p < 0.05 ≠ 差異很大**：統計顯著性和實際顯著性是兩件事，需要效果量才完整
3. **多重比較是陷阱**：同時監控 20 個指標，必然有一個「顯著」，要事前聲明主要指標

### 下一步：M7 ML 前導與學習路線

本模組建立的統計直覺（分佈、假設檢定、相關性）是機器學習的理論基礎。M7 將展示統計如何延伸成學習算法——從線性回歸的最小二乘法，到邏輯回歸的最大似然估計，到決策樹的信息增益，背後都有本模組的統計直覺在支撐。
