### S01 — 你以為的資料分析 vs. 實際在做什麼

**核心訊息：** 資料分析不是「畫圖表」，是清理 + 探索 + 推論的完整循環，圖表只是最後的呈現，不是工作的主體。

**講師講解要點：**
- 多數初學者以為資料分析 = 做 dashboard / 做折線圖。這是誤解。圖表是最末一步，前面有大量清理和判斷工作。
- 資料分析師的實際時間分配：業界調查顯示約 60-80% 時間花在資料清理，而非建模或視覺化。
- 三個核心動作定義：清理（讓資料可信）、探索（找出值得問的問題）、推論（用資料支撐決策）。
- 強調「推論」不必然是統計推論，也包括業務判斷：「這個指標下降，是因為活動結束還是有技術問題？」
- 這門課的 M1 到 M4 都在訓練這條鏈，從 Python 基礎出發，不是從圖表出發。

**視覺建議：** 兩欄對比圖。左欄「你以為的流程」：資料 -> 圖表 -> 結論（三步，箭頭乾淨）；右欄「實際流程」：原始資料 -> 清理（缺失/重複/型態）-> 探索（分布/相關/異常）-> 推論（假設/驗證/決策）-> 呈現，用多個循環箭頭暗示這是迭代而非線性。

**過渡語：** 「要能做這三件事，第一步是看得懂資料本身的結構。一份資料，不管是 CSV、資料庫還是 JSON，它背後都有五個元素。」

### S02 — 結構化資料的五個元素

**核心訊息：** 每一份結構化資料都可以用列（row）、欄（column）、綱要（schema）、型態（type）、鍵（key）來完整描述，理解這五個元素是所有分析工作的起點。

**講師講解要點：**
- 以一張真實感的訂單資料表為例（訂單 ID / 客戶名稱 / 金額 / 日期 / 狀態），帶出五個元素。
- **列（row）** = 一筆觀測值，代表「一件事發生了」。一列訂單 = 一筆交易發生。
- **欄（column）** = 一個變數，描述這件事的一個面向。金額是一個欄，日期是另一個欄。
- **綱要（schema）** = 所有欄名加上對應型態的完整定義，是資料的「藍圖」。換一份資料，先看 schema。
- **型態（type）** = 每個欄的資料類型，整數、浮點數、字串、日期、布林，型態決定能做哪些運算。
- **鍵（key）** = 可以唯一識別一筆記錄的欄位，通常是 ID 類欄位。沒有鍵會導致合併錯亂。
- 連結到後面 pandas：`df.dtypes` 看型態、`df.shape` 看列欄數、`df.head()` 看記錄。

**視覺建議：** 一張標記過的資料表格截圖，用不同顏色箭頭標出：橫向掃一列（標「一筆觀測值」）、縱向掃一欄（標「一個變數」）、整體標「schema」、某個 ID 欄標「key」、欄標題下方標型態。

**過渡語：** 「看得懂結構，下一步是判斷資料有沒有問題。大部分初學者拿到資料就開始分析，但乾淨的資料才是分析的前提。」

### S03 — 什麼叫做「乾淨的資料」

**核心訊息：** 污染資料有四種形式：缺失值、重複值、離群值、型態錯誤。識別這四類是清理工作的第一步，每一類都有不同的處理邏輯。

**講師講解要點：**
- **缺失值（missing value）**：欄位沒有值。在 Python/pandas 中通常是 `NaN`。注意空字串 `""` 和假值 `0` 在有些系統中也被用來表示「沒有資料」，但它們不是 `NaN`，要分開處理。
- **重複值（duplicate）**：完全重複（整列相同，通常是匯入錯誤）和邏輯重複（同一個客戶用不同拼法出現兩次）是不同問題，後者更難抓。
- **離群值（outlier）**：極端的數值。注意：離群值不一定是錯誤。一位真實的頂級客戶可能消費是平均值的 100 倍，這是真實的。需要先判斷「是資料問題」還是「是真實的業務現象」。
- **型態錯誤（type error）**：金額欄位被存成字串「$1,200」、日期被存成「2024/01/15」字串而非 datetime。Python 計算時會報錯，或更糟，默默產生錯誤結果。
- 清理的目的不是讓資料「好看」，是讓分析「可信」。

**視覺建議：** 四格圖，每格對應一種污染類型，用同一張訂單表格呈現，高亮出問題儲存格：缺失值用灰底、重複列用黃底標記兩行相同、離群值用紅底標記一個極端金額、型態錯誤用橘底標記字串格式的數字。

**過渡語：** 「知道什麼是乾淨資料之後，下一個問題是：用什麼工具來做清理？這裡就要談到一個很多人沒有意識到的工具選擇問題。」

### S04 — Excel 思維 vs. Python 思維

**核心訊息：** Excel 是「手動操作」，Python 是「可重現的工作流」。前者快速直觀，但每次做分析都是全新的一次操作；後者需要多寫一點，但腳本本身就是文件，也是自動化的起點。

**講師講解要點：**
- Excel 的核心問題不是「不好用」，而是「操作不留痕跡」。你按了幾十個步驟刪除了某些行、改了某些格式，這些動作沒有被記錄下來。下週來一批新資料，你必須重頭做一遍，而且你可能記不清自己上次做了什麼。
- Python 腳本的每一行都是一個可讀的決策：「把金額欄轉成 float」、「刪掉 NaN 超過一半的列」。這些決策是透明的，可以被審計，也可以被修改。
- 可重現性（reproducibility）是數據工程的核心要求之一：相同的輸入 + 相同的腳本 = 相同的輸出，永遠如此。
- 不是說「Excel 沒用」，而是：當資料超過一定規模、當分析需要重複執行、當結果需要被別人驗證，Python 是更可靠的工具。
- 這門課的定位：讓你建立 Python 思維，把分析流程寫成腳本，而不只是在 Notebook 裡跑一次就結束。

**視覺建議：** 左右對照時間軸圖。左側「Excel 工作流」：下載 CSV -> 打開 Excel -> 手動篩選 -> 手動刪行 -> 用公式轉換 -> 複製到 PowerPoint，標注「下次要全部重來」。右側「Python 工作流」：下載 CSV -> 執行 `clean.py` -> 執行 `analyze.py` -> 輸出報告，標注「下次只要換一行檔名」。

**過渡語：** 「Python 思維的最佳實踐環境，是 Jupyter Notebook。它不只是一個 IDE，它改變了資料分析工作的呈現方式。」

### S05 — 為什麼 Jupyter Notebook 成為標準

**核心訊息：** Jupyter Notebook 把文字說明、程式碼、執行結果、圖表全部放在同一個文件裡，讓分析過程本身可以被閱讀、被重現、被分享，這是它成為資料分析標準的根本原因。

**講師講解要點：**
- 傳統的分析流程：程式在 `.py` 檔、說明在 Word、圖表在 PowerPoint，三個地方分開，難以維護，更新一個地方其他兩個不會自動更新。
- Notebook 的設計哲學：分析過程 = 一篇可執行的論文。每個 cell 可以是 Markdown 說明，也可以是可執行的 Python 程式，執行結果直接顯示在下方。
- 四合一能力：文字（解釋為什麼這樣做）+ 程式（實際操作）+ 數字輸出（中間結果）+ 圖表（視覺呈現），全部在一個 `.ipynb` 文件。
- 為什麼說「資料故事」而不是「資料報告」：一個好的 Notebook 不只是程式加圖表，而是有推論脈絡的敘述，讀者能從頭到尾跟著你的思考走。
- 在實際工作中，Notebook 通常用於探索分析（EDA）和溝通；生產環境的執行腳本會是 `.py` 文件，這兩個工具不互斥。

**視覺建議：** 一個真實 Jupyter Notebook 截圖，用標記框分別圈出：Markdown 說明 cell（標「文字說明」）、程式碼 cell（標「可執行程式」）、數字輸出（標「中間結果」）、matplotlib 圖表輸出（標「視覺呈現」）。

**過渡語：** 「說了這麼多資料思維，現在要讓你親手做一次。打開你的 Notebook，我們來說第一個資料故事。」

### S07 — Python 型態與容器：只教分析會用到的

**核心訊息：** 不從語法表開始教，只教資料分析實際用到的六種型態與容器，並且每一個都對應到資料世界的具體使用場景。

**講師講解要點：**
- **`int`（整數）**：訂單數量、用戶 ID、計數。注意：除法 `7 / 2 = 3.5`，Python 3 的除法預設回傳 float。
- **`float`（浮點數）**：金額、比率、統計指標。注意浮點精度問題：`0.1 + 0.2` 不完全等於 `0.3`，在金融計算時要用 `decimal` 模組。
- **`str`（字串）**：商品名稱、類別標籤、日期（原始狀態）。重要操作：`.strip()` 去空格、`.lower()` 統一大小寫、`.replace()` 替換。
- **`list`（列表）**：一欄資料的原始形式，有順序、可重複、可修改。`[1, 2, 3, "a", True]` 可混型，但資料分析中通常同型。
- **`dict`（字典）**：鍵值對，一筆資料記錄的自然表達。`{"name": "Alice", "age": 30, "score": 95.5}` 對應一列資料。也是 JSON 的直接對映。
- **`tuple`（元組）**：不可修改的列表，常用於函式的多值回傳和不應被改動的設定值。`df.shape` 回傳的就是 tuple，例如 `(1000, 5)` 表示 1000 列 5 欄。

**視覺建議：** 三欄對照表：第一欄「型態」、第二欄「資料世界的例子」、第三欄「Python 寫法」。整張表一次呈現六個型態，用顏色區分基本型（int/float/str）和容器型（list/dict/tuple）。

**過渡語：** 「有了型態和容器，接下來要學的是怎麼對這些資料做判斷和迭代，也就是條件與迴圈。」

### S08 — 條件與迴圈：資料篩選的語言

**核心訊息：** `if / for / list comprehension` 是資料處理的基本語句結構，理解它們的場景才是重點，不是背語法。

**講師講解要點：**
- **`if / elif / else`**：用於單一記錄的條件判斷，或控制程式流程。資料分析中常見於定義新欄位的邏輯：金額 > 1000 標記為「高價值」，否則「標準」。

```python
def label_order(amount):
    if amount > 1000:
        return "high_value"
    elif amount > 200:
        return "standard"
    else:
        return "low_value"
```

- **`for` 迴圈**：對列表或任何可迭代物件逐元素處理。在 pandas 出現之前，for 迴圈是批量處理資料的主要方式；在 pandas 出現之後，大部分情況用 `.apply()` 或向量化操作取代，但 for 迴圈的思維邏輯仍然重要。

```python
# 對每一欄檢查缺失值數量
columns = ["name", "amount", "date", "category"]
for col in columns:
    missing = df[col].isna().sum()
    print(f"{col}: {missing} missing values")
```

- **List comprehension**：一行的轉換與篩選，是 Python 最具代表性的語法糖，也是初學者最值得學會的進階寫法。

```python
# 傳統 for 寫法
cleaned = []
for name in names:
    cleaned.append(name.strip().lower())

# list comprehension 寫法（等價，但更 Pythonic）
cleaned = [name.strip().lower() for name in names]

# 加上條件篩選
high_value = [x for x in amounts if x > 1000]
```

- 什麼時候用哪個：單一判斷用 `if`、批量轉換考慮 list comprehension、複雜邏輯用 `for` 加函式。

**視覺建議：** 程式碼對比框：左側「for 迴圈版本」，右側「list comprehension 版本」，用箭頭連接，標注「等價，但右側更 Pythonic」。底部加一行說明：「更大的資料量，應該用 pandas 向量化操作，M3 會教。」

**過渡語：** 「有了條件和迴圈，你已經可以對資料做基本的篩選和轉換。但如果這些步驟需要反覆執行，每次都要重寫一遍，這就需要函式。」

### S09 — 函式：把重複工作變成可呼叫的流程

**核心訊息：** 定義函式不是語法練習，而是把「我做了什麼」抽象成「任何時候任何人都能呼叫的流程」，這是讓分析腳本從一次性變成可重複使用的關鍵動作。

**講師講解要點：**
- 函式的本質：輸入 -> 處理 -> 輸出。一個好的函式只做一件事，做完就回傳，不依賴外部狀態。
- 資料分析中常見的函式化時機：清理某一欄（輸入是一個 Series，輸出是清理後的 Series）、計算某個業務指標（輸入是幾個參數，輸出是一個數字）、判斷記錄品質（輸入是一筆記錄，輸出是 True/False）。

```python
def clean_amount(raw: str) -> float:
    """
    Convert raw amount string like '$1,200.50' to float 1200.50.
    Returns 0.0 if conversion fails.
    """
    try:
        return float(raw.replace("$", "").replace(",", ""))
    except ValueError:
        return 0.0
```

- **預設參數**：讓函式在大多數情況下不需要傳入所有參數。`def summarize(df, top_n=10)` 預設看前十名，但可以覆蓋。
- **回傳多個值**：Python 的函式可以 `return` 一個 tuple，呼叫時用解構賦值接收。

```python
def get_stats(values: list) -> tuple:
    return min(values), max(values), sum(values) / len(values)

lo, hi, avg = get_stats([10, 20, 30, 40])
```

- 函式化是走向 OOP（M2）的前一步：M2 會把相關的函式和資料封裝成 class，但函式是基本單元。

**視覺建議：** 「函式三要素」圖：一個大方框代表函式，左邊是輸入箭頭（標「參數，決定這次要處理什麼」），右邊是輸出箭頭（標「回傳值，處理的結果」），框內是「處理邏輯（只做一件事）」。下方一行：「函式名稱 = 這個流程的名字」。

**過渡語：** 「函式讓你的程式碼可以重用。但 Python 真正強大的不只是語言本身，而是它身後的套件生態。最後一個概念：import。」

### S10 — import 與套件：Python 強在語言 + 生態

**核心訊息：** Python 語法本身只是起點，`import` 打開了套件生態的大門，資料分析 / 機器學習 / AI 的能力都來自這個生態，而不是來自語言本身。

**講師講解要點：**
- `import` 的本質：把別人寫好的程式碼引入你的工作空間，避免重新發明輪子。
- 三種 import 寫法與使用時機：

```python
import math                   # 用 math.sqrt(4) 呼叫，命名空間清楚
import numpy as np            # 給套件一個短別名，減少打字量，約定俗成
from datetime import datetime # 只引入需要的部分，減少命名污染
```

- 資料分析的核心套件地圖（簡短預告，M3/M4 詳細教）：
  - `pandas`：表格資料處理
  - `numpy`：數值計算和陣列操作
  - `matplotlib / seaborn`：視覺化
  - `scikit-learn`：機器學習（M7 前導）
- 為什麼說「Python 的強大 = 語言 + 生態」：如果只有語法，Python 和其他腳本語言差異不大。是 NumPy / pandas / PyTorch 這條生態鏈，讓 Python 變成 AI 和資料科學的共通語言。
- 套件安裝概念：`pip install` 和 `conda install`，以及為什麼需要虛擬環境（簡單帶過，M5 會深入）。

**視覺建議：** 圓心輻射圖。中心是「Python 語言核心」，往外第一圈是資料分析套件（pandas / numpy / matplotlib），第二圈是 ML/DL（scikit-learn / PyTorch），第三圈是系統與工程（fastapi / pyspark / docker sdk）。每個套件名稱下方標一行用途。

**過渡語：** 「現在你有了資料思維的框架，也有了 Python 的基本表達工具。是時候把這兩件事結合起來，做一個真正的練習。」

**Step 2：封裝欄位清理函式（8 分鐘）**

把「清理金額欄位」這個動作封裝起來：

```python
def clean_amount_column(series):
    """
    Convert a string-formatted amount column to float.
    Handles formats like '$1,200', '1200.5', '1,000'.
    """
    return (
        series
        .astype(str)
        .str.replace(r"[$,]", "", regex=True)
        .str.strip()
        .apply(lambda x: float(x) if x.replace(".", "").isdigit() else None)
    )

# 呼叫
df["amount_clean"] = clean_amount_column(df["amount"])
df[["amount", "amount_clean"]].head()
```

**Step 3：封裝類別統計函式（9 分鐘）**

```python
def category_summary(df, category_col: str, value_col: str, top_n: int = 5):
    """
    Return top N categories sorted by total value.
    """
    result = (
        df.groupby(category_col)[value_col]
        .sum()
        .sort_values(ascending=False)
        .head(top_n)
    )
    return result

# 呼叫
top_categories = category_summary(df, "category", "amount_clean", top_n=3)
print(top_categories)
```
