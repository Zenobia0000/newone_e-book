## 一、執行摘要（Executive Summary — Read This First）

### 一句話結論（Governing Thought）

> **「在 2026 年，不會用 Python 處理資料，就等於在 1995 年不會用 Excel。這不是選修，是基礎。而 80% 的資料工作者還卡在 Excel 思維。」**

### 三個支撐論點（Supporting Arguments）

1. **市場現實**：AI 浪潮下，每家企業都需要資料決策能力，但 Gartner 2017 指出 **85% 的資料專案失敗**，失敗原因 87% 是「資料準備不足」，而非演算法不夠好。
2. **學習投資報酬**：24 小時能把一位 Excel 使用者，升級到可執行 CRISP-DM 六階段前三階段（商業理解 → 資料理解 → 資料準備）的分析師層級。
3. **職涯槓桿**：此能力是 5 條職涯路線（統計 / ML / DL / 資料工程 / LLM 工程）的共通底盤，一次投資，五路受益。

### 課程承諾（The Promise）

完成本課程後，學員將：
- 停止把 **Excel 當成唯一工具**，開始用 Python 建立可重現分析流程
- 從 **被動執行報表**，升級到 **主動提出業務假設並驗證**
- 具備進入 ML / DL / 大數據工程的 **底盤語言與思維**

## 二、SCQA 情境框架（Why This Course, Why Now）

顧問報告的經典開場結構。

### Situation（現況）
2026 年，全球企業每 3 小時生成的資料量超過 1980 年代全年總和。生成式 AI 把「產出內容」的成本降到接近於零，但「判斷內容是否可信」的能力，反而比任何時代都珍貴。

### Complication（張力）
大部分企業資料工作者仍停留在「**Excel 層級**」——手動操作、無法重現、無法擴展、無法接上 AI 應用。這產生三個痛點：
- **時間黑洞**：資料工作者 60-80% 時間花在清理與整理（CrowdFlower 2016, Forbes 2023）
- **決策失真**：Anscombe's Quartet 證明統計量相同的資料可能呈現完全不同的故事
- **斷層風險**：AI 工具越強，「餵進去的資料品質」越決定產出價值

### Question（問題）
**一般業務工作者（非資工背景）如何在有限時間內，建立可與 AI 協作的資料分析能力？**

### Answer（本課程的回答）
**24 小時，8 模組，雙主線（資料思維 + 工具實作），從 Excel 到 ML 前導的完整鏈路。**

## 七、真實企業案例連結（Named Case Studies）

用具體名字讓抽象概念落地。

### Case 1：Airbnb 的 Knowledge Repo（對應 M1 Notebook 工作流）
Airbnb 資料團隊開發了 Knowledge Repo 開源工具，核心概念是：**每一份分析必須是一份可被他人閱讀、複現、質疑的 notebook**。這直接對應本課程 M1 強調的「Excel 思維 vs Python 思維」差異。
來源：[Airbnb Engineering Blog — Knowledge Repo](https://medium.com/airbnb-engineering/scaling-knowledge-at-airbnb-875d73eff091)

### Case 2：Netflix 的實驗平台（對應 M6 假設檢定）
Netflix 每年執行數千次 A/B Test，他們的內部平台 XP 強制每個實驗必須計算 **effect size** 與 **statistical power**，而非僅看 p-value。這正是 M6 強調的「統計顯著 ≠ 業務顯著」。
來源：Netflix Tech Blog — *It's All A/Bout Testing*

### Case 3：Spotify 的 Data Quality 警示系統（對應 M4 資料清理）
Spotify 的 資料平台團隊 2022 年分享，他們最大的痛點不是演算法，而是 **資料品質監控**。一個欄位型別改變就可能讓整條推薦管線退化 20%。這呼應 M4「型別錯誤是高影響、可偵測的資料問題」。

### Case 4：Uber 的 Michelangelo ML 平台（對應 M7 ML 前導）
Uber 公開 Michelangelo 架構論文時強調：**80% 的工作在資料準備，20% 在建模**。這直接驗證 CRISP-DM 的時間分配，也說明 M1-M6 是 M7 的必要前置。

### Case 5：台積電的製程分析（對應 M5 EDA）
台積電的工程師用 Python + pandas 分析晶圓製程良率，其中 EDA 四步驟（看全貌 → 抓異常 → 找關係 → 形成假設）是標準作業流程。這證明 EDA 不是資料科學家專利，是任何工程分析的基礎。

## 八、Pyramid Principle 重構的 8 模組敘事

每個模組用「**結論先行 + 三個支撐論點**」重新表達。

### M0 — Python 是 AI 時代的作業系統
> 不會 Python，等於 2026 年的數位文盲。

支撐：① Python 在 AI/ML/資料工程三棲；② 生態系已經鎖定（無替代方案）；③ 從學習到產業無縫銜接。

### M1 — 資料思維比 Python 語法更重要
> 你不是來學寫 code，是來學把現實翻譯成結構化資料。

支撐：① 結構化資料的五要素（row/column/schema/type/key）；② 乾淨資料的四類污染；③ Notebook 改變分析的呈現方式。

### M2 — Python 只學分析會用到的部分就夠
> 分析師的 Python，比工程師的 Python 窄但深。

支撐：① 型態 + 容器 + 流程 + 函式 = 90% 分析語法；② 錯誤閱讀是除錯的起點；③ import 打開整個生態系。

### M3 — NumPy 是所有模型的底層語言
> 看不懂 shape，就看不懂任何 ML 論文。

支撐：① ndarray 是計算引擎，不是資料容器；② broadcasting 消滅 90% for 迴圈；③ 向量化是效能與正確性的雙贏。

### M4 — DataFrame 是資料分析世界的工作台
> 沒有 pandas，資料分析工作無法啟動。

支撐：① DataFrame = 帶型別的計算物件（不是 Excel）；② groupby 是業務切片思維；③ Arrow backend 改變記憶體遊戲規則。

### M5 — 圖表是思考工具，不是美化裝飾
> Anscombe's Quartet 證明，數字會說謊，圖表揭真相。

支撐：① 四種圖表任務類型（分布/比較/關聯/時序）；② EDA 四步路徑（全貌/異常/關係/假設）；③ What → So What → Now What 洞察框架。

### M6 — 統計是從觀察升級到判斷的語言
> 不懂統計的分析師，只是描述者；懂統計的，才是決策支援者。

支撐：① 分布比均值更接近真相；② 假設檢定問的是「差異可不可信」；③ p-value 不是決策本身，是決策的輸入之一。

### M7 — ML 不是魔法，是函數逼近
> 你已經走完 CRISP-DM 的前 4 階段，M7 是打開第 5 階段的鑰匙。

支撐：① scikit-learn 的一致 API 是業界標準；② PyTorch 2.0 把 DL 推向編譯優化；③ 5 條職涯路線都從 M1-M6 起步。
