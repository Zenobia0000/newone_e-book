# 課程總計畫：Python 進階數據工程與分析

**版本：** 1.0  
**總時數：** 18 小時  
**模組數：** 4 模組 / 10 章節

---

## 一、課程定位

本課程是「**Python 入門完成後，邁向資料工程實戰**」的銜接課程。學員結業後應能：

1. **建立系統直覺**：理解 Python 程式碼背後的硬體與作業系統運作，避免寫出 OOM、I/O 阻塞、意外深拷貝等常見錯誤
2. **掌握 Python 進階機制**：能流暢使用 Generator、Lambda、Comprehension、`*args/**kwargs` 等特殊語法
3. **以 OOP 思維設計資料管線**：能用類別封裝清洗邏輯，使腳本可重用、可測試
4. **熟練資料工程三件套**：File I/O 例外處理、NumPy 向量化、Pandas 進階清洗與聚合
5. **完成端到端整合實作**：自訂類別讀檔 → 清洗 → 特徵工程 → 視覺化 → 匯出

**本課程不教**：scikit-learn / PyTorch / TensorFlow / 雲端 / Spark / Airflow（這些屬於本課之後的進階方向，於 Ch10 末段給出學習路徑導讀）。

## 三、教學理念（取捨原則）

1. **剔除進階機器學習**：18 小時內塞 sklearn 是稀釋，不是補強。專注資料底盤
2. **特殊函式 × 資料操作合流**：Lambda / Comprehension 不獨立教，全部對齊到 Pandas `apply` 與資料管線實戰
3. **OOP 為資料工程服務**：不講設計模式、不講多重繼承，只講「如何把清洗邏輯封裝成可重用類別」
4. **計概只講對 Python 有影響的部分**：不講匯流排、不講組合語言指令集，只講 RAM/OOM/I/O/GIL 對日後寫程式的衝擊
5. **教科書式編排 + 實用主義**：每章學習目標明確，但所有抽象概念都必須落地到一個可執行範例

## 七、與既有 24h 課程的差異化

本課程與 `python-data-ai-foundations`（24h M0–M7）並行，但定位不同：

| 維度 | 24h M0–M7 | 本 18h |
|------|-----------|--------|
| 對象 | 完全新手 → AI 工程師全景 | 已會基礎 → 資料工程實戰 |
| 廣度 | 涵蓋計概、ML/DL 前導、職涯地圖 | 聚焦 OOP + Pandas 工程化 |
| 開場 | 為什麼學 Python（動機） | 計概前導（直接打底層） |
| 收尾 | ML/DL/Big Data 學習路徑 | OOP × Pandas 端到端實作 |
| 風格 | 企業培訓概論 | 大學進階選修教科書 |
