## S1 · SILENT · Hero

## S2 · ASK · 為什麼第四次複製貼上

- 🖼️ 畫面：左側大問號與質問句，右側深綠資料卡顯示 61%。
- 📣 畫面上的字：主問句「為什麼同一段清洗邏輯，你已經複製貼上第四次？」；資料卡 `61% · 資料工程師每週維護 ≥ 3 條相似管線`。
- 🎙️ 講者這時說：腳本的缺點不是難寫，是難重用。每多一個資料源就多貼一次程式，bug 就多一個分身。OOP 的價值正是從這個痛點出發。
- Source：Kaggle State of Data Science 2024。

## S3 · VS · 腳本式 vs 類別式

- 🖼️ 畫面：左右兩欄對照。左欄灰底：5 個散落函式 + 10 個全域變數；右欄深綠框：1 個 DataPipeline 類別。
- 📣 畫面上的字：左標「腳本式」、右標「類別式」；底部收束句「差的不是行數，是狀態的去處」。
- 🎙️ 講者這時說：兩邊都 100 行，但左邊的狀態散在 global namespace，右邊的狀態收在 `self.xxx`。維護成本差 10 倍。

## S4 · MATRIX 1×3 · 三大效益

- 🖼️ 畫面：三格水平並列，每格一個效益 + 副文。
- 📣 畫面上的字：可重用（寫一次套 N 個資料集）/ 可測試（小單元獨立驗證）/ 可組合（管線各步驟可換）。
- 🎙️ 講者這時說：三個效益背後同一件事——降低耦合。

## S5 · FLOW · 反例三條

- 🖼️ 畫面：三節點橫向流程圖，標「一次性腳本 → 純轉換函式 → 實驗 notebook」，底部紅字警示（但我們用深綠）。
- 📣 畫面上的字：「不為 hello world 開類別」Linus 引言。
- 🎙️ 講者這時說：OOP 不是萬用解。一次性腳本、純函式轉換、探索性 notebook，這三種場景硬套 OOP 只會增加複雜度。

## S6 · ASK · Class vs Object

- 🖼️ 畫面：左側提問，右側待補藍圖 vs 房子示意圖 placeholder。
- 📣 畫面上的字：「如果 Class 是建築藍圖，Object 是什麼？」
- 🎙️ 講者這時說：記住這個比喻：一份藍圖可以蓋很多棟房子；每棟房子各自有地址、有住戶，但都長得像。

## S7 · CODE · 最小類別

- 🖼️ 畫面：單一 code panel，展示 class DataPipeline + __init__ + add_step。
- 📣 畫面上的字：右側 bullets「class 宣告藍圖 / __init__ 是誕生禮 / self 是身分證」。
- 🎙️ 講者這時說：三行關鍵字 + 兩個方法，就比 5 個散落函式好用。

## S8 · CODE · __init__ 與 self 解剖

- 🖼️ 畫面：code panel 展示 pipe = DataPipeline('ETL') 的實例化兩步。
- 📣 畫面上的字：標註「①呼叫類別 ②Python 自動呼 __init__(self=pipe, name='ETL')」。
- 🎙️ 講者這時說：self 不是關鍵字，是慣例名。Python 傳的是當前物件本身。

## S9 · MATRIX 1×3 · self 三問

- 🖼️ 畫面：三格並列。
- 📣 畫面上的字：Q1 誰傳進來？/ Q2 指到哪？/ Q3 不寫會怎樣？
- 🎙️ 講者這時說：背熟這三題，OOP 一半功課就完成了。

## S10 · CODE + IMAGE · 記憶體模型

- 🖼️ 畫面：左側待補記憶體示意圖（兩個 DataPipeline 實例各有自己的 steps list），右側 code panel 驗證。
- 📣 畫面上的字：code 展示 `p1.steps is p2.steps  # False`。
- 🎙️ 講者這時說：每次 __init__ 裡 self.steps = []，都是建一個新 list；兩個實例不會互相污染。

## S11 · ASK · items=[] 陷阱前哨

- 🖼️ 畫面：黑底大問號 + 程式碼片段 `class Foo: items = []`。
- 📣 畫面上的字：「如果把 items = [] 寫在類別頂端會怎樣？」
- 🎙️ 講者這時說：這是面試最常考、生產最常見的 bug。下一張直接拆給你看。

## S12 · VS-CODE · Class vs Instance Attribute

- 🖼️ 畫面：上下兩個 code panel，BAD（灰標）+ GOOD（深綠標）。
- 📣 畫面上的字：BAD — `class Bad: data = []`；GOOD — `def __init__(self): self.data = []`。
- 🎙️ 講者這時說：縮排差一格，語意差很多。可變物件絕對放 __init__。

## S13 · TABLE · 何時該用 Class Attribute

- 🖼️ 畫面：三欄 editorial table（用途 / 範例 / 警戒）。
- 📣 畫面上的字：常數、計數器、預設設定；絕不放 list/dict/set。
- 🎙️ 講者這時說：Class Attribute 是類別級共享狀態，只給不可變物件用。
