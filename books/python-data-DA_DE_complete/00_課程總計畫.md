# 課程總計畫：Python 資料分析與資料工程完整實戰

**版本：** 1.0  
**總時數：** 約 26 小時（基礎 14h + 實戰 12h）  
**階段數：** 2 階段 / 13 章節

---

## 一、課程定位

本課程整合「Python 資料分析與資料工程實戰（22h）」與「Python 進階數據工程與分析（18h）」兩套課程的精華，是給「**已學過 Python 基礎語法、非理工資訊背景**」學員的 **完整資料分析 / 資料工程銜接課**。

學員背景常為商管、人文、醫護、設計、行銷轉職或自學者，已會變數 / 條件 / 迴圈 / 函式，但：
- 沒有計算機概論 / OS 基礎（看到 OOM、路徑跨平台、I/O Bound 會卡）
- 沒有進階函式概念（Lambda / Comprehension / *args 看不懂）
- 沒有 OOP 概念（無法看懂業界開源程式碼、無法寫可重用的資料管線）
- 沒有檔案 I/O 與例外處理觀念（讀寫 CSV/JSON 出錯不知道如何排查）
- 只會「能跑就好」的腳本寫法（維護成本高、無法規模化）

結業後，學員應能：

1. 看懂進階開源資料科學程式碼（DataFrame / ndarray / class / apply）
2. 流暢使用 Lambda、Comprehension、Generator 等 Python 進階機制
3. 撰寫**模組化、可重用**的資料處理管線（DataCleaner 類別）
4. 熟練檔案 I/O、例外處理、分塊讀取大檔
5. 熟練 NumPy 向量化、Pandas 進階清洗、時序 EDA、視覺化呈報
6. 產出互動式 dashboard（Plotly）作為 capstone 專案
7. 具備系統直覺，知道大檔 OOM 怎麼處理、I/O 慢瓶頸在哪

**本課程不教**：scikit-learn / PyTorch / TensorFlow / 雲端 / Airflow（結業後的下一站路線，於 Ch07、Ch13 給出學習地圖）。

## 四、教學理念

### 1. 分階段：基礎 → 工具

F 階段不急著教工具，先把底盤鋪好（系統直覺 + 進階函式 + OOP 思維 + I/O 處理）。S 階段才進入 NumPy / Pandas / 視覺化，此時學員已有「為什麼要這樣寫」的理解力，不只學語法。

### 2. 進階函式為 Pandas 鋪路

Ch03 的 Lambda / Comprehension / `*args` 不是獨立教，而是全部對齊到後續 Pandas `apply` 與資料管線實戰，讓學員在 Ch09-Ch10 能直接上手。

### 3. 檔案 I/O 承接 OOP、銜接 NumPy

Ch06 放在 OOP 之後、NumPy 之前，學員已會寫 class 可以用在讀檔封裝，同時 try/except 和分塊讀取為後續大量資料處理奠基。

### 4. 教學型投影片設計（Teaching-Track）

全課程投影片套用 `shared/design_system/教學型投影片_黃金守則與泛式.md` 的 **T1–T12 黃金守則**與 **7 種教學原型**：

| 原型 | 用途 |
|------|------|
| MOTIVATION | 每章開場的痛點情境 |
| CONCEPT-CARD | 核心概念定義（左文右圖） |
| MECHANISM-FLOW | step-by-step 原理 |
| EXAMPLE-I/O | 明確標示 Input → Process → Output |
| PITFALL | 錯誤寫法 vs 正確寫法（紅綠對比） |
| PRACTICE-PROMPT | 章節中段練習題（非結尾） |
| CHECKPOINT | 每 5 張插一張檢核點 |

### 5. 節奏：四拍循環

```
概念（CONCEPT-CARD）
  ↓
例子（EXAMPLE-I/O）
  ↓
問題（ASK / PRACTICE）
  ↓
小結（SILENT / PYRAMID）
```

每 4-6 張完成一輪；一章 17-24 張 = 3-4 個循環。
