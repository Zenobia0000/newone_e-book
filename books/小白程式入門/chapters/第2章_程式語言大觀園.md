# 第2章：程式語言大觀園

> 「我應該學哪個程式語言？」
>
> 如果說上一章的問題是最常被問的，那這個問題絕對是第二名。而我的答案總是：「你想用它來做什麼？」

---

## 選擇的焦慮

記得小美第一次打開程式學習網站時的表情。

Python、JavaScript、Java、C++、Ruby、Go、Swift...螢幕上密密麻麻的選項讓她眼花撩亂。每個語言旁邊都寫著聽不懂的特色：「物件導向」、「函數式」、「強型別」、「直譯式」...

「這就像站在一家有200種口味的冰淇淋店前，」她苦笑著說，「選擇太多，反而不知道該選什麼。」

她花了整整一週研究「最好的程式語言」，看了無數篇比較文章，每一篇都說得頭頭是道，但看完後她更迷茫了。有人說Python最簡單，有人說JavaScript最實用，有人說Java找工作最容易...

直到有一天，她問了我那個關鍵的問題：「我應該學哪個？」

而我反問她：「你想解決什麼問題？」

---

## 語言，只是工具

讓我分享一個可能會讓你驚訝的事實：對於大部分的程式任務，用哪個語言其實沒那麼重要。

這就像問：「我該學開手排車還是自排車？」答案是——重要的不是車子的類型，而是你要去哪裡。

每個程式語言，本質上都在做同樣的事：
- 儲存資料
- 做判斷
- 重複動作
- 與外界溝通

差別只在於表達方式和擅長的領域。就像中文適合寫詩，英文適合寫科學論文，但兩者都能表達「我愛你」。

讓我用一個簡單的例子說明。假設我們要讓電腦說「Hello, 小美」：

**Python的方式：**
```python
name = "小美"
print(f"Hello, {name}")
```

**JavaScript的方式：**
```javascript
let name = "小美";
console.log(`Hello, ${name}`);
```

**Java的方式：**
```java
String name = "小美";
System.out.println("Hello, " + name);
```

看出來了嗎？概念完全一樣，只是「口音」不同。

---

## Python：像在寫詩的程式語言

如果程式語言是樂器，Python就是鋼琴——優雅、直觀、適合初學者，但也能演奏出大師級的作品。

我永遠記得小美第一次寫Python時的反應：「等等，就這樣？不用分號？不用大括號？這真的是程式嗎？」

```python
# 檢查今天是否該運動
體重 = 70
目標體重 = 65

if 體重 > 目標體重:
    print("該運動了！")
else:
    print("保持下去！")
```

Python的美在於它的簡潔。它去掉了所有不必要的符號，讓你專注在邏輯本身。這就像寫作時，你不需要先學習複雜的文法規則，而是直接開始表達想法。

**Python特別適合：**
- 數據分析（「這個月哪個產品賣最好？」）
- 自動化任務（「幫我整理這1000個檔案」）
- 人工智慧（「訓練一個認識貓的程式」）
- 網路爬蟲（「收集所有競爭對手的價格」）

小美最後選擇了Python，不是因為它「最好」，而是因為她的第一個目標是自動化Excel報表。三週後，她寫出了第一個實用程式——自動從20個分店的銷售檔案中提取數據，生成週報。原本要花她整個Monday早上的工作，現在只要按個按鈕。

## JavaScript：網頁的魔法師

如果說Python是鋼琴，那JavaScript就是吉他——隨處可見、容易上手、而且超級酷炫。

「你現在看到的每個網頁，背後都有JavaScript在運作，」我告訴小美的同事阿威，一個想要創業的設計師。

阿威原本是平面設計師，他想要的不是處理數據，而是讓他的設計「活起來」。當他第一次用JavaScript讓按鈕被點擊時改變顏色，他的眼睛亮了起來。

```javascript
// 當按鈕被點擊時
按鈕.addEventListener('click', function() {
    if (心情 === "開心") {
        背景.style.color = "黃色";
        播放音樂("快樂頌");
    } else {
        背景.style.color = "藍色";
        顯示訊息("需要一個擁抱嗎？");
    }
});
```

JavaScript的魔力在於「即時回饋」。你改一行程式碼，重新整理瀏覽器，立刻看到效果。這種即時的視覺回饋，對創意工作者來說特別有吸引力。

**JavaScript特別適合：**
- 網頁互動（「讓這個選單滑出來」）
- 前端開發（「做一個漂亮的購物網站」）
- 後端開發（「處理用戶登入」）——是的，JavaScript前後端通吃
- 手機App（React Native讓你用JavaScript寫App）

六個月後，阿威用JavaScript做出了他的第一個作品集網站。不是靜態的圖片展示，而是會動、會回應、會說故事的互動體驗。他說：「我終於能把腦中的創意完整呈現出來了。」

## 其他語言的故事

### Java：企業的可靠夥伴

我有個朋友老陳，在銀行工作。當他決定學程式時，毫不猶豫地選了Java。

「銀行系統不能出錯，」他說，「Java就像穿西裝的管家，規規矩矩，但絕對可靠。」

Java要求你明確定義每個變數的類型，就像銀行要求每筆交易都要有明確的記錄。這看似麻煩，但當系統變大時，這種嚴謹能救你一命。

### Go：速度與簡潔的平衡

小李是個系統工程師，他需要處理每秒數萬筆請求。他選擇了Go。

「Go就像瑞士軍刀，」他形容，「簡單、高效、專注於把事情做好。」

Go的並發處理能力驚人，就像一個超級協調者，能同時處理成千上萬件事而不會搞混。

### Swift：蘋果生態系的語言

如果你的夢想是做iOS App，Swift是唯一的選擇。就像要在法國當廚師，你最好會說法語。

## 如何選擇你的第一個語言

經過這麼多故事，你可能還是在想：「所以我該選哪個？」

讓我給你一個簡單的指南：

**如果你想要：**

**快速看到成果** → JavaScript
打開瀏覽器就能開始，改了就能看到效果。

**處理數據、自動化工作** → Python
簡潔優雅，大量現成工具，適合解決實際問題。

**找傳統大公司的工作** → Java
企業愛Java，工作機會多，薪水穩定。

**做iOS App** → Swift
別無選擇，但也是個優雅的語言。

**打好基礎、理解底層** → C
困難但值得，像學武功要先蹲馬步。

但最重要的建議是：**不要糾結太久**。

記住，程式語言不是終身伴侶，你不需要「從一而終」。大部分工程師都會3-5種語言，而且學了第一個之後，學第二個會快很多。

就像小美，她從Python開始，一年後因為想做網頁，很自然地學了JavaScript。她說：「第二個語言只花了一個月就上手，因為概念都是相通的。」

## 語言的偏見與真相

在程式世界裡，有很多偏見和迷思。讓我來打破幾個：

### 「Python太慢了」

是的，Python執行速度比C慢。但你知道嗎？Instagram用Python處理數億用戶的照片。速度很重要，但開發速度更重要。用Python兩天寫完的程式，C可能要兩週。

### 「JavaScript不是真正的程式語言」

Netflix、PayPal、Uber的後端都大量使用JavaScript。如果它不是「真正」的程式語言，這些公司怎麼敢把幾億美金的生意交給它？

### 「Java過時了」

全世界有900萬個Java開發者，Android App大部分用Java寫。過時？我看是穩如泰山。

### 「C++太難了」

遊戲產業愛死C++了。你玩的3A大作，底層引擎幾乎都是C++。難？是的。但值得？看你想做什麼。

---

## 超越語言的思考

讓我告訴你一個秘密：真正的程式高手，不是精通某個語言的人，而是能夠超越語言思考的人。

當你理解了「變數」的概念，你就能在任何語言中使用它。
當你學會了「迴圈」的邏輯，你就能用任何語法表達它。
當你掌握了「問題分解」的能力，你就能用任何工具解決它。

這就像一個真正的廚師，給他任何廚房、任何食材，他都能做出美味的料理。工具只是工具，創造力才是關鍵。

小美現在已經能用Python和JavaScript工作了。但更重要的是，她學會了「程式思維」。她看問題的角度變了，解決問題的方法變了，甚至思考的方式都變了。

「語言只是表達方式，」她說，「重要的是你想表達什麼。」

## 你的第一步

如果你讀到這裡還在猶豫，讓我給你一個最簡單的建議：

閉上眼睛，想像一個你想解決的問題。

是想要自動化無聊的工作？選Python。
是想要做個人網站？選JavaScript。
是想要開發App？看你用iPhone還是Android。

然後，今天就開始。

不要再研究了，不要再比較了，不要再等「最佳時機」了。

打開電腦，選一個語言，寫下你的第一行程式碼。哪怕只是讓電腦說「Hello」，那也是你改變的開始。

記住阿威的話：「我糾結了三個月該學什麼，但真正改變我的，是我終於開始寫的那一天。」

## 寫在第二章的最後

程式語言就像世界各地的語言，每種都有its own beauty，都有它的文化和哲學。

選擇哪一種開始，真的沒那麼重要。重要的是開始。

下一章，我們要談談更基礎的東西——如何像電腦一樣思考。因為無論你選擇什麼語言，邏輯思維都是相同的。

但在那之前，我想留給你一個思考：

如果程式語言是工具，那你想用這個工具創造什麼？

這個答案，比選擇哪個語言重要一萬倍。

---

*「The best programming language is the one you actually use.」*
*最好的程式語言，是你真正在用的那一個。*
