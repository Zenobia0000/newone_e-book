# 第7章：程式設計的職業道路

> 「如果我真的想轉職成為工程師，機會大嗎？」
>
> 小美完成記帳程式後，第一次認真思考這個問題。她35歲，沒有資工背景，但她已經能寫出實用的程式了。

---

## 咖啡廳裡的職涯討論

週六下午，我們約在常去的咖啡廳。

小美、阿威、老陳、小李都來了。還有一位新朋友——凱文，一位在科技公司工作五年的前端工程師。

「我也是轉職的，」凱文說，「28歲才開始學程式，之前是會計。」

大家的眼睛都亮了起來。

凱文繼續：「轉職不是問題，問題是你要知道自己在做什麼。工程師不是一種職業，而是很多種職業的統稱。」

他在餐巾紙上畫了一張圖...

---

## 前端工程師：網頁的藝術家

阿威最感興趣的是前端。

「前端工程師就是把設計變成真實網頁的人，」凱文解釋，「你要懂美感，也要懂程式。」

**日常工作：**
- 早上開 standup meeting，討論今天要做什麼
- 實作設計師的設計稿
- 處理使用者互動（點擊、滑動、輸入）
- 優化網頁載入速度
- 確保在不同瀏覽器都正常顯示
- 與後端工程師協調 API 介面

**需要的技能：**
```javascript
// 三劍客
HTML：網頁的骨架
CSS：網頁的外觀
JavaScript：網頁的行為

// 現代框架
React / Vue / Angular：其中一個

// 工具
Git：版本控制
Webpack：打包工具
npm：套件管理
```

**適合的個性：**
- 注重細節（1px 的差異都要在意）
- 有美感
- 喜歡即時看到成果
- 能接受快速變化（前端技術更新很快）

阿威聽完說：「這簡直是為我量身定做的！我本來就是設計師，會程式後就能自己實現設計了。」

---

## 後端工程師：系統的建築師

老陳對後端更有興趣。

「後端是處理商業邏輯的地方，」凱文說，「使用者看不到，但一切運作都靠它。」

**日常工作：**
- 設計資料庫結構
- 撰寫 API 介面
- 處理商業邏輯（訂單、付款、庫存）
- 優化查詢效能
- 確保資料安全
- 處理大流量（如果網站很熱門）

**需要的技能：**
```python
# 語言（選一個深入）
Python + Django/Flask
JavaScript + Node.js
Java + Spring
Go

# 資料庫
SQL：關聯式資料庫
NoSQL：MongoDB、Redis

# 概念
RESTful API
微服務架構
快取策略
資安基礎
```

**適合的個性：**
- 邏輯思維強
- 喜歡解決複雜問題
- 注重系統穩定性
- 能處理抽象概念

老陳點頭：「銀行系統就是大型的後端，我懂這個邏輯。」

---

## 全端工程師：瑞士刀

小李對全端有興趣。

「全端就是前後端都會，」凱文說，「通常在新創公司很受歡迎，因為一個人能做很多事。」

**優點：**
- 能獨立完成整個專案
- 了解系統全貌
- 轉職彈性大
- 新創公司愛用

**挑戰：**
- 要學的東西很多
- 難以在單一領域深入
- 技術更新壓力大

「但是，」凱文提醒，「不要一開始就想當全端。先專精一端，再學另一端。」

---

## 資料工程師：數據管道工

小美發現自己對數據特別有興趣。

「資料工程師負責確保資料能順暢流動，」凱文解釋，「從收集、清理、轉換到儲存。」

**日常工作：**
- 建立資料管道（ETL）
- 清理髒資料
- 設計資料倉儲
- 優化查詢效能
- 確保資料品質

**薪資往往很高，因為：**
- 需求大（大數據時代）
- 會的人相對少
- 直接影響商業決策

小美眼睛發亮：「我每天在Excel裡做的事，原來可以自動化成這樣！」

---

## 轉職的真實路徑

「好了，了解方向後，要怎麼轉職？」小美問。

凱文分享了他的經驗：

### 第0-3個月：打基礎
```
學習內容：
- 程式語言基礎
- 資料結構與演算法基礎
- Git 版本控制

每天投入：2-3小時
重點：建立程式思維
```

### 第4-6個月：選定方向
```
前端路線：
- 深入 JavaScript
- 學一個框架（React/Vue）
- 做3個小專案

後端路線：
- 深入一個後端語言
- 學資料庫
- 寫 RESTful API
```

### 第7-9個月：作品集
```
關鍵任務：
- 完成1-2個完整專案
- 放上 GitHub
- 寫技術部落格
- 參加技術社群
```

### 第10-12個月：求職準備
```
面試準備：
- 刷 LeetCode（基礎題就好）
- 準備作品集簡報
- 模擬面試
- 投遞履歷
```

「一年？」阿威有點驚訝。

「如果你全職學習，半年也可能。如果只能晚上學，可能要兩年。」凱文說，「重點是持續和累積。」

## 第一份工作的真相

「第一份工程師工作最難找，」凱文坦白說，「但一旦有了一年經驗，機會就完全不同了。」

**新手的機會在哪？**

### 1. 新創公司
- 願意給新手機會
- 學習曲線陡峭（學很快）
- 可能要一人多工
- 薪水可能較低

### 2. 接案/自由工作
- 累積作品集
- 建立人脈
- 收入不穩定
- 需要自律

### 3. 大公司的約聘
- 有機會轉正
- 能學到規範
- 競爭激烈

### 4. 內部轉職
- 在現有公司轉技術職
- 已有領域知識
- 風險較低

小美若有所思：「我可以先在公司內部推動數位轉型，邊做邊學...」

## 工程師的日常真相

「我想知道真實的一天是怎樣的。」老陳問。

凱文打開他的行事曆：

```
09:00 - 到公司，看昨天的 code review 評論
09:30 - 每日站會，15分鐘
09:45 - 寫程式（新功能開發）
11:00 - 幫同事 debug
12:00 - 午餐
13:30 - 寫程式（繼續早上的）
15:00 - 開會討論下週規劃
16:00 - Code review 其他人的程式
17:00 - 寫文件/測試
18:30 - 下班（有時候）

實際上：
- 30% 寫新程式
- 30% 改別人的程式/debug
- 20% 開會討論
- 20% 寫文件/測試
```

「不是一直在寫程式？」阿威驚訝。

「溝通和協作佔很大部分，」凱文說，「這就是為什麼軟實力也很重要。」

---

## 持續學習的宿命

「當工程師最大的挑戰，」凱文正色說，「是你永遠在學習。」

```
5年前的熱門技術 → 現在可能過時
現在的新技術 → 5年後可能成主流

例如：
jQuery（曾經必學）→ 現在很少用
React（5年前的新秀）→ 現在的主流
AI 輔助編程 → 未來的趨勢？
```

「這不是很累嗎？」小美擔心。

「換個角度想，」凱文笑了，「你永遠不會無聊。而且，基礎觀念不變，變的只是工具。」

---

## 不當工程師也很好

「我想說一件事，」凱文認真地說，「不是每個學程式的人都要當工程師。」

**程式能力 + 領域知識 = 獨特價值**

- 小美：行銷 + 程式 = 數據行銷專家
- 阿威：設計 + 程式 = 創意技術專家
- 老陳：金融 + 程式 = FinTech 專家
- 小李：業務 + 程式 = 銷售自動化專家

「會程式的行銷人員，比純工程師更了解行銷需求。」
「會程式的設計師，能創造別人做不出的互動體驗。」
「會程式的金融人，能開發別人想不到的交易策略。」

## 寫在第七章的最後

咖啡廳的討論持續到傍晚。

離開前，凱文說了一段話：

「五年前，我是個每天對著報表的會計。現在，我在開發影響百萬用戶的產品。不是因為我特別聰明，而是因為我開始了，並且堅持下去。」

「轉職不是終點，而是另一個起點。重要的不是你幾歲開始，而是你願意投入多少。」

「記住，科技業缺的不是會寫程式的人，而是能解決問題的人。你的過去不是包袱，而是你的獨特優勢。」

小美若有所思地說：「我不確定我會不會成為全職工程師，但我確定，學程式改變了我看問題的方式。」

阿威興奮地說：「我要開始準備作品集了！」

老陳淡定地說：「慢慢來，我先把基礎打好。」

小李已經在查線上課程了。

每個人都有自己的步調，自己的目標。

而這，正是最好的開始。
