## S01 · SILENT · 起

- 視覺：深藍滿版，白字 hero：「中級科目一的核心：你能否在 NLP、CV、GenAI、多模態之間選對技術。」下方四個 icon 分別代表 NLP（文字泡泡）、CV（眼睛）、GenAI（閃電）、多模態（拼圖）。
- 一句話：定錨——本章是四大技術領域的選擇指南。
- 講者腳本：中級 L211 考的就是這四個領域。不需要你會寫模型，但需要你能判斷「這個場景該用什麼技術」。接下來 120 分鐘，我們建一張技術地圖。

## S02 · ASK · 起

- 視覺：大哉問居中：「什麼時候該用 AI，什麼時候寫個 if-else 就夠了？」右側兩欄對比：AI 適用場景（模式複雜、資料量大、規則難以窮舉）vs Rule-based 適用場景（規則明確、場景有限、可解釋性要求高）。
- 一句話：不是所有問題都需要 AI，判斷「該不該用」是第一步。
- 講者腳本：Q1 考的就是這個。如果規則可以窮舉、場景固定，寫 if-else 更快更可靠。AI 的價值在於處理「人寫不出規則」的複雜模式。考試會給情境讓你判斷。

## S03 · MECHANISM_FLOW · 起

- 視覺：水平 Pipeline 流程圖：原始文字 → 前處理（清洗/正規化）→ Tokenization → 向量化（Word2Vec/TF-IDF/Embedding）→ 下游任務（分類/NER/情緒/摘要）。每階段上方標工具或方法名。
- 一句話：NLP 的核心是把文字變成數字，讓機器能處理。
- 講者腳本：機器不懂文字，只懂數字。NLP 全流程就是「文字→數字→任務」。Tokenization 是第一關，向量化是第二關，理解這條 pipeline 就能應對大部分 NLP 考題。

## S04 · TABLE · 起

- 視覺：editorial_table 三列比較：字元級 / 詞級 / 子詞級（BPE）。欄位：粒度 / 詞彙量 / OOV 問題 / 序列長度 / 代表系統。BPE 行用綠色高亮標示「現代主流」。
- 一句話：Tokenization 三種切法，BPE 是現在的主流。
- 講者腳本：字元級切太細、序列太長；詞級遇到沒見過的詞就卡住；BPE 子詞級是折衷，GPT 和 BERT 都用它。Q2 考 Tokenization 的定義——「把文字切成模型能處理的最小單位」，記住這句話。

## S05 · VS · 承

- 視覺：左欄「Word2Vec：稠密向量、300 維、捕捉語義、king-man+woman≈queen、不看文件頻率」；右欄「TF-IDF：稀疏向量、維度=詞彙量、衡量詞重要度、TF x IDF、不懂語義」。中間大紅分隔線標「本質不同」。
- 一句話：Word2Vec 懂語義，TF-IDF 懂重要性——兩個解決不同問題。
- 講者腳本：Q10 考 Word2Vec，關鍵字是「稠密向量」和「語義相似度」。TF-IDF 的關鍵字是「詞頻-逆文件頻率」和「文件中的重要性」。考題選項會故意把兩者的特性交叉放，你要能精準區分。

## S06 · TABLE · 承

- 視覺：editorial_table 四列：NLP 任務 / 定義 / 典型應用 / 相關技術。任務：情緒分析、NER、文本分類、文本摘要。每列附一個小 icon。
- 一句話：NLP 下游任務四大天王，考試就考你能不能配對。
- 講者腳本：情緒分析判斷正負面，NER 抽取實體名稱，文本分類歸類別，文本摘要壓縮重點。考題通常給一個商業場景問你用哪個任務。「分析客戶評論的情緒傾向」→ 情緒分析；「從合約中抽取甲乙方名稱」→ NER。

## S07 · PITFALL · 承

- 視覺：vs_two_col。左「常見錯誤：Word2Vec 是衡量詞頻重要性的方法 ✗」；右「正確理解：Word2Vec 是將詞映射為稠密向量，捕捉語義關係 ✓」。下方附 Word2Vec 向量空間示意（king/queen/man/woman 四點位置關係）。
- 一句話：Word2Vec 不管詞頻，它管的是語義——這個考試常搞混。
- 講者腳本：考題最愛把 Word2Vec 和 TF-IDF 的描述交叉放在選項裡。記住一個判斷法：看到「頻率/重要度」→ TF-IDF；看到「向量/語義/類比」→ Word2Vec。

## S08 · SILENT · 承

- 視覺：深藍滿版：「從文字到圖像——CV 的世界觀完全不同。NLP 處理的是序列，CV 處理的是像素矩陣。」
- 一句話：過渡——進入 CV 領域。
- 講者腳本：NLP 把文字變數字，CV 把圖像變特徵。接下來的四大任務是 CV 考題的核心，搞清楚輸入輸出就能做對大部分題目。

## S09 · MATRIX · 轉

- 視覺：2x2 矩陣。橫軸：整張圖 / 像素級。縱軸：單一類別 / 多個個體。四格：圖像分類（整張圖+單標籤）/ 物件偵測（整張圖+多 bbox）/ 語義分割（像素級+不分個體）/ 實例分割（像素級+分個體）。每格附一張示意小圖。
- 一句話：CV 四大任務的差異就在「粒度」和「是否分個體」。
- 講者腳本：Q7 考的就是這個矩陣。分類是最粗的——整張圖一個標籤。偵測找出位置（bbox）。語義分割每個像素標類別但不分個體。實例分割最細——每個像素標類別且分個體。考題通常給場景問你用哪個。

## S10 · MECHANISM_FLOW · 轉

- 視覺：時間軸演進圖，左到右：CNN 基礎（LeNet/AlexNet）→ 深層 CNN（VGG/ResNet）→ ViT（Vision Transformer）。每個節點標年份、核心創新、解決的問題。ViT 節點用虛線連接到 NLP 的 Transformer，標「跨領域遷移」。
- 一句話：CNN 奠定基礎，ResNet 解決深度問題，ViT 證明 Transformer 在 CV 也行。
- 講者腳本：CNN 用卷積提取局部特徵，ResNet 用殘差連接讓網路可以疊很深，ViT 把圖像切成 patch 當 token 丟進 Transformer。ViT 是理解「為什麼 Transformer 能跨領域」的關鍵。

## S11 · PITFALL · 轉

- 視覺：左右對比圖。左：語義分割——兩隻貓都標為橙色（同類同色）。右：實例分割——貓A藍色、貓B紅色（同類不同色）。下方大字：「語義不分個體，實例分個體」。
- 一句話：這張圖記住，考試就不會錯。
- 講者腳本：語義分割的「語義」是指類別語義——所有「貓」像素同色，不管有幾隻。實例分割的「實例」是指個體——每隻貓有自己的標識。自動駕駛需要知道「前面有三輛車各在哪」→ 實例分割。

## S12 · MECHANISM_FLOW · 轉

- 視覺：Transformer 架構簡化圖。左側：Input Embedding + Positional Encoding → Multi-Head Self-Attention → Feed Forward → Output。右側放大 Self-Attention 的 Q/K/V 計算流程。底部標：「Attention(Q,K,V) = softmax(QK^T/sqrt(d_k))V」。
- 一句話：Self-Attention 就是讓每個 token 看所有其他 token，決定該關注誰。
- 講者腳本：Transformer 的核心是 Self-Attention。每個 token 產生 Q（查詢）、K（鍵）、V（值），Q 和 K 算相似度決定注意力權重，再用權重加總 V。位置編碼補上順序資訊。這個機制讓 Transformer 能做 NLP、CV、多模態——因為它不假設資料結構，只看「誰跟誰相關」。

## S13 · TABLE · 轉

- 視覺：editorial_table 三列比較：知識蒸餾 / 剪枝 / 量化。欄位：原理 / 優點 / 缺點 / 精度損失 / 速度提升 / 典型應用。量化行額外標示「FP32 → FP16 → INT8 → INT4」的精度階梯。
- 一句話：壓縮三劍客各有長短，考試要你區分原理。
- 講者腳本：Q8 考模型壓縮。蒸餾 = 大模型教小模型（teacher-student）；剪枝 = 砍掉不重要的連接（像修剪樹枝）；量化 = 降低數字精度（從 32 位浮點降到 8 位整數）。記憶法：蒸「教」、剪「砍」、量「降」。

## S14 · MATRIX · 轉

- 視覺：三欄比較矩陣：早期融合 / 晚期融合 / 注意力機制融合。每欄包含：融合位置示意圖、優缺點、適用場景。早期融合畫在 pipeline 前端（特徵層合併），晚期畫在後端（決策層合併），注意力機制畫在中間（Cross-Attention）。
- 一句話：三種融合策略的差異就在「什麼時候合併」。
- 講者腳本：Q11 考 Early Fusion。早期融合 = 先合再處理（特徵層），好處是模態交互充分，壞處是維度爆炸。晚期融合 = 各做各的最後再合（決策層），好處是模組化，壞處是錯過低層交互。注意力機制融合用 Cross-Attention，是目前最先進的做法。

## S15 · VS · 轉

- 視覺：左欄「早期融合：在特徵層合併→模態交互深→維度高→計算量大→適合模態高度相關的場景」；右欄「晚期融合：在決策層合併→模組化→各自優化→錯過低層交互→適合模態可獨立處理的場景」。中間雙向箭頭標「注意力機制融合：折衷方案」。
- 一句話：早期和晚期的選擇取決於模態之間的相關程度。
- 講者腳本：如果影像和文字高度相關（如醫療影像+病歷），早期融合效果好但計算量大。如果模態可以獨立處理再合併（如語音辨識+文字分類），晚期融合更實際。考題看情境判斷。

## S16 · MATRIX · 合

- 視覺：多模態醫療案例全景圖。三個輸入源：醫學影像（X光/MRI）→ ViT 處理 / 電子病歷（文字）→ BERT 處理 / 基因數據（序列）→ 1D-CNN 處理。三路匯合處標「Cross-Attention 融合」→ 輸出：輔助診斷結果。右側標各環節的技術選擇理由。
- 一句話：多模態醫療是考題最愛的綜合案例，把 NLP+CV+融合策略全考一遍。
- 講者腳本：Q12 就是考這個。影像用 ViT（因為 Transformer 能處理 patch），文字用 BERT（NLP 專長），基因用 1D-CNN（序列局部特徵）。融合用 Cross-Attention 讓三個模態互相「看」。這題整合了本章所有知識點。

## S17 · PYRAMID · 合

- 視覺：四層金字塔，由底到頂：NLP（文字處理基礎）→ CV（圖像理解基礎）→ GenAI/Transformer（跨領域引擎）→ 多模態（融合應用）。每層標核心技術 2-3 個。頂端標「技術選擇力」。
- 一句話：四層堆疊，底層是基礎，頂層是整合。
- 講者腳本：NLP 和 CV 是兩個基礎領域，Transformer 是連接它們的引擎，多模態是最終的整合應用。考試的難題通常在頂端——多模態融合題需要你同時理解底下三層。

## S18 · EXAM_DRILL · 合

- 視覺：模擬考題畫面。左側 3 道 NLP 技術辨識題（含 Tokenization、Word2Vec、NER 相關），右側逐題解析。每題標考試範疇與歷屆對應。
- 一句話：NLP 考題實戰——抓住關鍵字就能選對。
- 講者腳本：第一題考 Tokenization 定義，第二題考 Word2Vec 特性辨識，第三題考 NER 應用配對。每題 60 秒，先自己選再看解析。NLP 題的訣竅是抓選項裡的關鍵字。

## S19 · EXAM_DRILL · 合

- 視覺：模擬考題畫面。左側 3 道題：CV 技術配對（1 題）+ 模型壓縮辨識（2 題）。右側逐題解析。附圖輔助（如分割示意圖）。
- 一句話：CV 和壓縮考題——圖記住了，題就對了。
- 講者腳本：CV 配對題看「輸出格式」——標籤/bbox/像素遮罩/像素遮罩+個體。壓縮題看「原理」——教/砍/降。這三題是歷屆高頻題型的變體。

## S20 · EXAM_DRILL · 合

- 視覺：模擬考題畫面。左側 3 道題：多模態融合策略（1 題）+ Transformer 應用（1 題）+ 綜合醫療案例（1 題）。右側逐題解析。
- 一句話：多模態和 Transformer 是中級的壓軸題型。
- 講者腳本：融合題看「在 pipeline 哪個位置合併」。Transformer 題記住它的跨領域通用性。醫療案例是綜合題，把 NLP+CV+融合策略串起來。能做對這三題，L211 基本穩了。

## S21 · PRACTICE_PROMPT · 合

## S22 · SILENT · 合

- 視覺：深藍滿版：「技術不是越新越好，是選對的那一個。NLP、CV、GenAI、多模態——考的不是你背了多少名詞，是你能不能在情境中選出最適合的技術。下一章：AI 導入規劃與系統部署。」
