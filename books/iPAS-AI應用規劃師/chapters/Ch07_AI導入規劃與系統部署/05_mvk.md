# Ch07 · Minimum Viable Knowledge（MVK）— 考前速查卡

> 若只記得 12 件事，這就是 Ch07 的精華。
> 評鑑範圍：中級 L212 + L213

---

## AI 導入五階段流程

```
需求分析 → 方案設計 → 數據準備 → 模型建構 → 系統部署
                                                ↑
                          監控更新 ←────────────┘
```

上線不是終點，監控發現問題後回到數據準備或模型建構重新迭代。

---

## 需求分析四步法

| 步驟 | 核心問題 | 考試關鍵字 |
|---|---|---|
| 1. 痛點識別 | 現有流程瓶頸在哪？（量化描述） | 錯誤率、人工成本、時效性 |
| 2. 可行性評估 | 能不能做到？（資料/技術/團隊） | 資料可得性、技術成熟度、團隊能力 |
| 3. ROI 估算 | 值不值得做？（成本 vs 效益） | 導入成本、維護成本、預期效益 |
| 4. 優先級排序 | 先做哪個？（影響力 x 可行性） | 高影響+高可行性 = 先做 |

## 技術方案三選一

| 方案 | 優勢 | 劣勢 | 適用場景 |
|---|---|---|---|
| **自建模型** | 高客製化、資料不外流 | 成本高、需專業團隊 | 核心業務、獨特資料 |
| **採購方案** | 快速導入、成本可控 | 客製化受限 | 通用場景、快速上線 |
| **AutoML** | 降低門檻、加速驗證 | 黑箱風險、仍需人工 | 原型驗證、團隊缺 ML 人才 |

**AutoML 考試重點**：自動化範圍 = 特徵選擇 + 模型選擇 + 超參數調整。不包含問題定義和資料收集。選項說「完全自動」就是錯的。

## 數據準備全流程

```
數據收集 → EDA（探索性分析）→ 清洗處理 → 特徵工程 → 特徵選擇
```

### 清洗處理關鍵步驟

| 步驟 | 方法 | 考試重點 |
|---|---|---|
| 缺值處理 | 刪除/填補（均值、中位數、模型）/插值 | MCAR 可刪、MAR 用模型填、MNAR 需調查 |
| 異常值偵測 | IQR（Q1-1.5*IQR ~ Q3+1.5*IQR）、Z-score（\|z\|>3） | Ch08 會深入 |
| 去重 | 精確比對/模糊比對 | — |
| 格式統一 | 日期/編碼/單位統一 | — |

### 特徵工程速查

| 類型 | 方法 | 何時用 |
|---|---|---|
| 標準化 (Z-score) | (x-mu)/sigma | 資料需符合常態分布假設時（SVM、LR） |
| 正規化 (Min-Max) | (x-min)/(max-min) | 需要壓縮到 [0,1] 時（神經網路） |
| One-Hot Encoding | 類別→二元向量 | 類別無序時（城市、顏色） |
| Label Encoding | 類別→整數 | 類別有序時（低/中/高） |

## 模型選擇速查表

| 場景 | 推薦模型 | 原因 |
|---|---|---|
| 資料量小 + 需可解釋性 | 決策樹 / 線性模型 | 簡單、透明 |
| 表格資料 + 精度優先 | XGBoost / 隨機森林 | Kaggle 冠軍常客 |
| 影像分類/偵測 | CNN（ResNet/YOLO） | 空間特徵擷取 |
| 文本/序列 | Transformer（BERT/GPT） | 注意力機制 |
| 聚類 | K-means / DBSCAN | K-means 球型、DBSCAN 任意形狀 |
| 降維 | PCA / t-SNE | PCA 線性、t-SNE 非線性 |

**選模型不能只看精度**：還要看推論延遲、成本、可解釋性、部署限制。

## 風險四象限

| 風險類型 | 具體風險 | 管控措施 |
|---|---|---|
| **技術風險** | 過擬合、資料漂移、概念漂移 | 交叉驗證、漂移偵測、觸發式重訓 |
| **資安風險** | 供應鏈攻擊、對抗攻擊、資料投毒 | SBOM、輸入驗證、安全掃描 |
| **倫理風險** | 偏見歧視、黑箱決策、隱私侵犯 | 公平性檢測、SHAP/LIME、PII 脫敏 |
| **法規風險** | 個資法/GDPR、EU AI Act | 法遵審查、資料治理、影響評估 |

### 供應鏈攻擊（Q9 考點）

- **攻擊方式**：透過第三方元件（預訓練模型/Python 套件/公開資料集）植入後門。
- **防範關鍵字**：SBOM（軟體物料清單）、可信任儲存庫、Model Card 驗證、定期安全掃描。

## 負責任 AI 五原則

| 原則 | 一句話定義 |
|---|---|
| 公平性 Fairness | 不因種族/性別/年齡等特徵歧視特定群體 |
| 可解釋性 Explainability | 決策過程能被利害關係人理解 |
| 透明性 Transparency | 訓練資料、方法、限制公開揭露 |
| 問責性 Accountability | 出錯時有明確責任歸屬 |
| 隱私保護 Privacy | 最小資料收集、匿名化、差分隱私 |

## 部署架構三選一

| 架構 | 適用場景 | 考試判斷依據 |
|---|---|---|
| **雲端** | 快速上線、彈性擴展、資料不敏感 | 題目沒提特殊限制 → 選雲端 |
| **地端** | 資料敏感、不可上傳、法規要求 | 題目提「資料不可離開」→ 選地端 |
| **Edge** | 即時推論、離線運行、設備端部署 | 題目提「低延遲+無法連網」→ 選 Edge |

**Edge 代價**：模型壓縮（量化/剪枝/蒸餾）→ 精度可能下降 + 模型更新不便。

## 容器化部署核心概念

| 技術 | 解決什麼問題 | 考試關鍵字 |
|---|---|---|
| Docker | 環境不一致（我的電腦上能跑） | 封裝、映像檔、環境一致性 |
| Kubernetes | 一台機器不夠用 | 編排、自動擴縮容、負載均衡 |
| CI/CD | 每次更新都要手動部署 | 自動測試、自動部署、持續整合 |
| MLflow/DVC | 模型版本管理混亂 | 版本追蹤、實驗紀錄、模型登記 |

## 系統測試四層（Q15 考點）

| 測試類型 | 測什麼 | 順序 |
|---|---|---|
| 單元測試 | 單一模組功能正確性 | 第 1（整合前） |
| **整合測試** | **子系統之間的介面與資料流** | **第 2（整合後最先做）** |
| 壓力測試 | 高併發承載能力 | 第 3 |
| A/B 測試 | 新舊版本效能對比 | 第 4（上線時） |

**Q15 直接考**：系統整合後最優先驗證 = 介面正確性（整合測試），不是效能、不是安全。

## 易混淆概念速查

| 概念 A | 概念 B | 一句話區分 |
|---|---|---|
| AutoML | MLOps | AutoML 自動化「建模」，MLOps 自動化「部署運維」 |
| Data Drift | Concept Drift | Data Drift 輸入分布變，Concept Drift 輸入輸出關係變 |
| 標準化 | 正規化 | 標準化 = Z-score（均值/標準差），正規化 = Min-Max（最小/最大） |
| 供應鏈攻擊 | 對抗攻擊 | 供應鏈 = 第三方元件植入後門，對抗 = 輸入加擾動騙模型 |
