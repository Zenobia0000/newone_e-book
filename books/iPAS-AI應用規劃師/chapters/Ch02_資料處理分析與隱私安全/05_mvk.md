# Ch02 · 考前速查卡（Minimum Viable Knowledge）

> 考前 30 分鐘快速複習用。對應 L112 全部考點。
> 核心判斷：**資料是 AI 的燃料——考試考的是你懂不懂資料從哪來、怎麼清、怎麼保護。**

---

## ① 大數據 5V 速查表（對應 LO1）

| V | 全名 | 一句話定義 | 考題場景 |
|---|------|---------|---------|
| Volume | 量 | 資料量大到傳統工具處理不了 | 「TB/PB 級資料」 |
| Velocity | 速 | 資料產生與處理速度快 | 「即時串流」「每秒數千筆」 |
| Variety | 多樣 | 資料格式多樣（不只表格） | 「文字+圖片+影音混合」 |
| Veracity | 真實性 | 資料品質有疑慮需驗證 | 「雜訊」「不一致」「品質」 |
| Value | 價值 | 從資料中萃取商業洞見 | 「商業價值」「決策支援」 |

**口訣**：量速多真值

**高頻陷阱**：Veracity（品質）≠ Value（價值），看到「品質」選 Veracity，看到「洞見」選 Value。

## ② 資料型態三分法（對應 LO2）

```
判斷流程：
  有固定 schema（固定欄位）？ → 結構化（SQL、CSV、Excel）
  有標籤但 schema 彈性？ → 半結構化（JSON、XML、HTML）
  完全沒格式？ → 非結構化（圖片、影音、自由文字、Email）

高頻陷阱：
  JSON → 半結構化（不是結構化！schema 彈性）
  Excel → 結構化（有固定欄位標題）
  Email → 非結構化（自由格式文字）
```

## ③ 資料處理五階段（對應 LO3）

```
收集 → 清理 → 轉換 → 分析 → 呈現
買菜 → 洗切 → 調味 → 下鍋 → 擺盤

各階段關鍵操作：
  收集：爬蟲、API、問卷、感測器、資料庫匯出
  清理：缺失值 + 異常值 + 重複值 + 格式統一 + 型態轉換
  轉換：特徵工程、One-hot 編碼、標準化/正規化、資料合併
  分析：描述統計、相關性分析、模型建構
  呈現：圖表、Dashboard、報告

判斷口訣：
  修品質 → 清理
  改格式 → 轉換
  取資料 → 收集
  出結果 → 分析
  做圖表 → 呈現
```

**高頻陷阱**：清理不只是刪 NULL，異常值處理也是清理！

## ④ 特徵工程「選建轉縮」（對應 LO4）

| 操作 | 做什麼 | 實例 |
|------|--------|------|
| 選取 Selection | 挑重要特徵、丟無關的 | 刪除相關性低的欄位 |
| 建立 Creation | 從現有欄位造新欄位 | 從日期取出「星期幾」 |
| 轉換 Transformation | 改變表示方式 | One-hot 編碼、Label 編碼 |
| 縮放 Scaling | 統一尺度 | 標準化（Z-score）/ 正規化（Min-Max） |

---

## ⑤ 標準化 vs 正規化（對應 LO5）

| | 標準化 Standardization | 正規化 Normalization |
|--|--|--|
| 公式 | **(X - mean) / std** | **(X - min) / (max - min)** |
| 結果 | 以 0 為中心，無固定範圍 | 縮放到 **[0, 1]** |
| 適用 | 常態分布、SVM、PCA | 固定範圍、NN、KNN |
| 記憶 | 標 = **Z**-score | 正 = **M**in-Max |

**終極記憶法**：
- 標準化的「標」→ Z（形狀像）→ Z-score
- 正規化的「正」→ 正數 0~1 → Min-Max

**高頻陷阱**：考題故意把名稱和公式交換，看到 [0,1] 一定是正規化！

## ⑥ 資料隱私三層防護（對應 LO6）

### 第一層：法規（個資法）
```
個資 = 可直接或間接識別個人的資料
核心原則：
  告知義務 — 蒐集前要告知當事人
  目的限制 — 只能用於蒐集時說明的目的
  最小蒐集 — 只蒐集必要的資料
  安全維護 — 要妥善保護蒐集到的個資
```

### 第二層：技術（去識別化）

| 技術 | 可逆性 | 做法 | 實例 |
|------|--------|------|------|
| 匿名化 | **不可逆** | 永久刪除所有識別資訊 | 刪除姓名+身分證+地址 |
| 假名化 | **可逆** | 用代號替換，保留對照表 | 姓名→隨機編號（有對照表） |
| 資料遮蔽 | 部分隱藏 | 遮蓋部分資訊 | A1234***89 |

**判斷口訣**：有對照表→假名化；永久刪除→匿名化；遮一半→資料遮蔽

### 第三層：原則（CIA）

| 原則 | 一句話 | 違反場景 |
|------|--------|---------|
| Confidentiality 機密性 | 不該看的人看不到 | 資料庫密碼外洩 |
| Integrity 完整性 | 資料沒被偷改 | 駭客竄改交易金額 |
| Availability 可用性 | 要用時用得到 | DDoS 攻擊系統癱瘓 |

**高頻陷阱**：Integrity 不是「資料完不完整」，是「資料有沒有被未授權竄改」！

## ⑦ 考前 5 分鐘終極速記

```
1. 5V：量速多真值（Veracity=品質 ≠ Value=價值）
2. 資料型態：固定schema=結構 / 有標籤但彈性=半結構 / 啥都沒=非結構
3. JSON 是半結構化！不是結構化！
4. 五階段：收清轉分呈（修品質=清理、改格式=轉換）
5. 清理 ≠ 只刪 NULL（還有異常值、重複值、格式、型態）
6. 標準化=Z-score（以0為中心）、正規化=Min-Max（0到1）
7. 匿名化不可逆、假名化可逆（有對照表）
8. CIA 的 Integrity = 沒被竄改（不是資料有沒有缺）
```
