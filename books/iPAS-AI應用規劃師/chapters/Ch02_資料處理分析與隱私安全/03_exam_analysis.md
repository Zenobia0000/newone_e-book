## 2. 考題類型分析

### 類型一：概念理解型（約佔 35%）
- 直接測試定義與分類知識
- 範例：「大數據 5V 特性中，哪一個 V 關注資料的品質與可信度？」
- 解題策略：記熟每個概念的一句話定義，精準配對

### 類型二：情境判斷型（約佔 45%）
- 給出一個資料處理場景，要求判斷所屬階段/操作/技術
- 範例：「某企業發現客戶資料中有重複紀錄和缺失值，進行修正。此操作屬於資料處理流程的哪個階段？」
- 解題策略：先看操作目的 → 品質問題=清理、格式問題=轉換、取得資料=收集

### 類型三：比較辨析型（約佔 20%）
- 比較兩個相近概念的差異
- 範例：「關於標準化與正規化，以下何者正確？」
- 解題策略：記住對照表，逐選項檢核

## 3. 模擬練習題

### 題目 1（概念理解型 — 大數據 5V）

關於大數據的 5V 特性，下列配對何者正確？

(A) Volume — 資料的產生速度與處理即時性
(B) Velocity — 資料的規模與儲存量
(C) Veracity — 資料的品質、可信度與準確性
(D) Value — 資料來源的多樣性

**答案：C**

**詳解**：Veracity 指的是資料的真實性、品質與可信度。(A) Volume 是量，不是速度——速度是 Velocity。(B) Velocity 是速度，不是規模——規模是 Volume。(D) Value 是價值，不是多樣性——多樣性是 Variety。最常見的錯誤是 Volume 和 Velocity 互換、Veracity 和 Value 互換。

---

### 題目 2（概念理解型 — 資料型態）

下列哪一項資料屬於「半結構化資料」？

(A) 關聯式資料庫中的客戶訂單表格
(B) 網站 API 回傳的 JSON 格式資料
(C) 數位相機拍攝的 JPEG 照片
(D) Microsoft Excel 的銷售報表

**答案：B**

**詳解**：JSON 具有 key-value 標籤結構但 schema 具彈性，屬於半結構化資料。(A) 關聯式資料庫表格有固定 schema，是結構化。(C) JPEG 照片沒有固定格式，是非結構化。(D) Excel 報表通常有固定欄位結構，是結構化。

---

### 題目 3（情境判斷型 — 資料處理流程）

某資料分析師發現資料集中，「年齡」欄位出現了 -3 和 250 這兩個值。他決定將這些值替換為該欄位的中位數。此操作屬於資料處理流程的哪個階段？

(A) 資料收集
(B) 資料清理
(C) 資料轉換
(D) 資料分析

**答案：B**

**詳解**：年齡出現 -3 和 250 明顯是異常值（不合理的值），將其替換為合理值是異常值處理，屬於資料清理階段。清理的目的是修正資料品質問題，包含缺失值處理、異常值處理、重複值移除、格式統一等。(C) 轉換是處理格式問題（如編碼、縮放），不是修正品質問題。

---

### 題目 4（情境判斷型 — 特徵工程）

某電商平台的資料科學家從「訂單日期」欄位中提取出「星期幾」和「是否為假日」兩個新欄位，以提升銷售預測模型的效果。此操作屬於特徵工程的哪一步驟？

(A) 特徵選取（Feature Selection）
(B) 特徵建立（Feature Creation）
(C) 特徵轉換（Feature Transformation）
(D) 特徵縮放（Feature Scaling）

**答案：B**

**詳解**：從現有欄位（訂單日期）衍生出新欄位（星期幾、是否假日）是特徵建立。(A) 特徵選取是從既有特徵中挑選重要的，不是創造新的。(C) 特徵轉換是改變既有特徵的表示方式（如 One-hot 編碼）。(D) 特徵縮放是統一尺度（如標準化/正規化）。

---

### 題目 5（比較辨析型 — 標準化 vs 正規化）

關於資料前處理中的標準化（Standardization）與正規化（Normalization），下列敘述何者正確？

(A) 標準化使用 Min-Max 公式，將資料縮放至 [0, 1] 區間
(B) 正規化使用 Z-score 公式，使資料均值為 0、標準差為 1
(C) 標準化後的資料以 0 為中心，正規化後的資料介於 [0, 1]
(D) 標準化與正規化的效果完全相同，可以互換使用

**答案：C**

**詳解**：標準化（Standardization）使用 Z-score 公式 (X-mean)/std，結果以 0 為中心。正規化（Normalization）使用 Min-Max 公式 (X-min)/(max-min)，結果縮放至 [0,1]。(A) 和 (B) 都把公式和名稱搞反了——這正是考題最愛的陷阱。(D) 兩者適用場景不同，不能互換。

---

### 題目 6（情境判斷型 — 去識別化）

某醫療研究機構需要使用病歷資料進行 AI 模型訓練。為保護患者隱私，該機構將所有患者姓名替換為隨機編號，並將編號與真實姓名的對照表儲存於加密系統中。此做法屬於下列何種去識別化技術？

(A) 匿名化（Anonymization）
(B) 假名化（Pseudonymization）
(C) 資料遮蔽（Data Masking）
(D) 資料加密（Data Encryption）

**答案：B**

**詳解**：以隨機編號替換真實姓名，且保留對照表（可還原），這是假名化的典型做法。(A) 匿名化是不可逆的——刪除所有可識別資訊且無法還原。(C) 資料遮蔽是部分隱藏（如身分證字號顯示 A***1234）。(D) 資料加密是用密鑰將資料轉為密文，不同於去識別化的概念。

### 題目 7（概念理解型 — CIA 三原則）

某公司的資料庫遭駭客入侵，部分客戶的交易金額被竄改。此資安事件最直接違反了資訊安全 CIA 三原則中的哪一項？

(A) 機密性（Confidentiality）
(B) 完整性（Integrity）
(C) 可用性（Availability）
(D) 可課責性（Accountability）

**答案：B**

**詳解**：資料被未經授權地竄改，這是完整性（Integrity）遭到破壞。(A) 機密性是「資料被未授權的人看到」。(C) 可用性是「需要時資料無法被存取」。(D) 可課責性不屬於 CIA 三原則（但屬於 AI 治理原則，見 Ch01）。注意：Integrity 是「資料未被竄改」，不是「資料有沒有缺失值」。

### 題目 8（情境判斷型 — 個資法）

某手機 App 在使用者未知情的情況下，持續蒐集使用者的 GPS 定位資料並傳送給廣告合作夥伴。此行為最主要違反了個人資料保護法的哪些原則？

(A) 僅違反安全維護義務
(B) 違反告知義務與目的限制原則
(C) 僅違反最小蒐集原則
(D) 不違反個資法，因為 GPS 資料不屬於個資

**答案：B**

**詳解**：GPS 定位資料可間接識別個人（知道一個人常出現的位置就能識別他），屬於個資。未告知使用者就蒐集→違反告知義務。將資料傳送給廣告夥伴（非原始蒐集目的）→違反目的限制原則。(D) 是錯誤的，GPS 資料是個資。(A) 和 (C) 不完整，主要違反的是告知義務和目的限制。

### 題目 9（情境判斷型 — 資料處理綜合）

以下四個資料處理操作，依正確的執行順序排列應為何？
甲：將「性別」欄位進行 One-hot 編碼
乙：移除資料集中的重複紀錄
丙：透過 API 從政府開放資料平台取得資料
丁：計算各特徵與目標變數的相關係數

(A) 丙→乙→甲→丁
(B) 乙→丙→丁→甲
(C) 丙→甲→乙→丁
(D) 丁→丙→乙→甲

**答案：A**

**詳解**：正確順序對應五階段流程：丙（收集：取得資料）→ 乙（清理：移除重複）→ 甲（轉換：One-hot 編碼）→ 丁（分析：計算相關係數）。(B) 清理在收集之前不合邏輯。(C) 轉換應在清理之後。(D) 分析在收集之前不合邏輯。

### 題目 10（綜合情境型 — 全章統整）

下列關於資料處理與隱私安全的敘述，何者最為正確？

(A) 大數據的 Veracity 特性是指從資料中挖掘出有價值的商業洞見
(B) XML 格式的資料因具有標籤結構，屬於結構化資料
(C) 資料清理僅包含處理缺失值，不包含異常值的偵測與處理
(D) 假名化與匿名化的主要差異在於：假名化可透過對照表還原原始資料，匿名化則不可逆

**答案：D**

**詳解**：(A) 錯誤——Veracity 是資料品質/可信度，不是商業價值（那是 Value）。(B) 錯誤——XML 有標籤但 schema 彈性，是半結構化，不是結構化。(C) 錯誤——清理包含缺失值處理、異常值處理、重複值移除、格式統一等。(D) 正確——假名化保留對照表可還原，匿名化刪除所有識別資訊且不可逆。

## 4. 備考策略建議

### 4.1 高效記憶法

1. **5V 口訣「量速多真值」**：Volume/Velocity/Variety/Veracity/Value
2. **型態判斷口訣「固定欄位→結構、有標籤→半結構、啥都沒→非結構」**
3. **五階段比喻「買洗調煮擺」**：收集/清理/轉換/分析/呈現
4. **特徵工程「選建轉縮」**：選取/建立/轉換/縮放
5. **標準化 vs 正規化「標Z正M」**：標準化=Z-score、正規化=Min-Max
6. **去識別化「匿不可逆、假可逆、遮部分」**
7. **CIA「機完用」**：機密性/完整性/可用性

### 4.2 答題技巧

1. **5V 題先排除最不像的**：Volume 和 Variety 通常不會搞混，先排除這兩個，再判斷 Velocity/Veracity/Value
2. **資料型態題看 schema**：有固定 schema→結構化，有標籤但彈性→半結構化，無格式→非結構化
3. **流程題看操作目的**：修品質→清理、改格式→轉換、取資料→收集、出結果→分析、做圖表→呈現
4. **標準化/正規化題看公式或結果範圍**：看到 Z-score 或「以 0 為中心」→標準化；看到 [0,1] 或 Min-Max→正規化
5. **隱私題先判斷有沒有對照表**：有→假名化；沒有且不可逆→匿名化；部分遮蓋→資料遮蔽

### 4.3 重點複習優先順序

1. **必拿分**：5V 配對、資料型態三分法、清理階段的操作範圍
2. **高分題**：標準化 vs 正規化差異、特徵工程四操作配對、去識別化技術比較
3. **易錯題**：JSON 是半結構化、Veracity ≠ Value、Integrity ≠ 資料完整性、清理 ≠ 只刪 NULL
