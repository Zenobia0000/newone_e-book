## S2 · CONCEPT-CARD — 大數據 5V 特性

- 🖼️ 畫面：白底 / 五個 V 排成五角形 / 每個 V 配圖示和一句話定義
- 📣 畫面上的字：
  - Volume（量）：資料量大到傳統工具處理不了 → TB/PB 級
  - Velocity（速）：資料產生速度快，需即時或近即時處理 → 串流資料
  - Variety（多樣）：資料格式多樣，不只表格 → 文字/圖片/影音/log
  - Veracity（真實性）：資料品質有疑慮，需驗證 → 雜訊、缺失、不一致
  - Value（價值）：從海量資料中萃取商業洞見 → 資料的最終目的
  - 記憶口訣：「量速多真值」
- 🎙️ 講者這時說：「大數據 5V 是基礎考點，考法是給你一個場景問你對應哪個 V。五個 V 各有一句話定義要記熟。最容易搞混的是 Veracity 和 Value——Veracity 問的是資料品質好不好、真不真實；Value 問的是資料有沒有商業價值、能不能產生洞見。口訣：量速多真值。」

## S3 · CONCEPT-CARD — 資料型態三分法

- 🖼️ 畫面：白底 / 三欄比較表 / 每欄上方有資料格式圖示
- 📣 畫面上的字：

  | | 結構化 | 半結構化 | 非結構化 |
  |--|--|--|--|
  | 特徵 | 固定 schema、欄位明確 | 有標籤/標記、schema 彈性 | 無固定格式 |
  | 格式 | SQL 表格、CSV、Excel | JSON、XML、HTML | 圖片、影音、文字、Email |
  | 儲存 | 關聯式資料庫 | NoSQL / 文件型資料庫 | 物件儲存 / 資料湖 |

  - 判斷口訣：「有固定欄位→結構 / 有標籤但彈性→半結構 / 什麼都沒有→非結構」
- 🎙️ 講者這時說：「資料型態分三種，判斷標準是有沒有固定的 schema。結構化就是你熟悉的表格——每一欄都有明確定義，每一列都遵守同樣格式。半結構化有標記但 schema 是彈性的——JSON 有 key-value 但不同筆資料可以有不同的 key。非結構化完全沒有格式——圖片、影音、自由文字。考試最愛的陷阱就是 JSON，很多人看到它有格式就選結構化——錯了，它是半結構化。」

## S5 · PITFALL (P1, P2) — 5V 混淆 + 半結構化陷阱

- 🖼️ 畫面：左橘錯 / 右綠對 / 底部解釋條
- 📣 畫面上的字：
  - 錯誤 1：「資料有雜訊代表沒有 Value」→ Veracity 是品質問題，Value 是價值問題，兩者獨立
  - 錯誤 2：「JSON 是結構化資料」→ JSON 有標籤但 schema 彈性，是半結構化
  - 錯誤 3：「Excel 是半結構化」→ Excel 表格有固定欄位，是結構化
  - 判斷反射：先問「有沒有固定 schema？」→ 有=結構化 / 有標籤但彈性=半結構 / 都沒有=非結構
- 🎙️ 講者這時說：「兩個高頻陷阱。第一，Veracity 和 Value 是獨立的——資料品質差（Veracity 低）不代表沒有價值（Value），你可以清理後再挖掘價值。第二，JSON 不是結構化——它有 key 但不同筆資料可以有不同的 key，schema 是彈性的，所以是半結構化。Excel 通常是結構化，因為有固定的欄位標題。」

## S7 · CONCEPT-CARD — 資料處理流程五階段

- 🖼️ 畫面：白底 / 橫向五節點 flow chart / 每個節點下方列出關鍵操作
- 📣 畫面上的字：
  - ① 收集（Collection）：爬蟲、API、問卷、感測器、資料庫匯出
  - ② 清理（Cleaning）：處理缺失值、移除重複、修正異常值、格式統一、型態轉換
  - ③ 轉換（Transformation）：特徵工程、編碼（One-hot/Label）、標準化/正規化、資料合併
  - ④ 分析（Analysis）：描述統計、視覺化探索、相關性分析、模型建構
  - ⑤ 呈現（Presentation）：圖表、儀表板、報告、摘要
  - 比喻：買食材→洗切→調味→下鍋→擺盤
- 🎙️ 講者這時說：「資料處理五階段，考題考的是你能不能把一個操作歸到正確的階段。用做菜比喻：收集是買食材，清理是洗菜切菜去掉壞的，轉換是調味備料讓食材可以下鍋，分析是下鍋烹煮，呈現是擺盤上桌。每個階段的操作列表要熟——尤其是清理和轉換，考題最愛在這兩個階段出題。」

## S9 · PITFALL (P3) — 資料清理不只是刪 NULL

- 🖼️ 畫面：左橘錯 / 右綠對 / 底部完整清理操作列表
- 📣 畫面上的字：
  - 錯：「資料清理 = 刪除缺失值」
  - 對：「資料清理 = 缺失值處理 + 異常值偵測 + 重複值移除 + 格式統一 + 型態轉換」
  - 缺失值處理方式：刪除整列 / 填入均值或中位數 / 用模型預測填補
  - 異常值處理方式：刪除 / 截斷（capping）/ 替換為合理值
  - 考題常問：「以下哪些操作屬於資料清理？」→ 全選上面五項
- 🎙️ 講者這時說：「很多考生以為清理就是 dropna()——刪掉缺失值就完事了。考題不會這麼簡單。清理包含五大操作：缺失值、異常值、重複值、格式、型態。考題可能問『以下何者不屬於資料清理？』，然後選項裡放一個轉換階段的操作當誘答。你要知道清理的完整範圍才不會被騙。」

## S11 · CONCEPT-CARD — 特徵工程四大操作

- 🖼️ 畫面：白底 / 四格卡片 / 每格一個操作 + 定義 + 實例
- 📣 畫面上的字：
  - 特徵選取（Selection）：挑出對模型有用的特徵，丟掉無關的 → 相關性分析、特徵重要性排序
  - 特徵建立（Creation）：從現有特徵組合出新特徵 → 從「生日」算出「年齡」、從「日期」取出「星期幾」
  - 特徵轉換（Transformation）：改變特徵的表示方式 → One-hot 編碼、Label 編碼、對數轉換
  - 特徵縮放（Scaling）：統一不同特徵的尺度 → 標準化（Z-score）、正規化（Min-Max）
  - 一句話：特徵工程 = 幫模型準備最好消化的食材
- 🎙️ 講者這時說：「特徵工程是把原始資料變成模型能有效學習的形式。四個操作各有明確定義。選取是挑好的、丟壞的。建立是從現有欄位造出新欄位。轉換是改格式。縮放是統一尺度。考題最常考的是『以下操作屬於特徵工程的哪個步驟？』——從日期取星期幾是建立、刪掉不重要的欄位是選取、One-hot 是轉換、Z-score 是縮放。」

## S12 · EXAMPLE-I/O — 標準化 vs 正規化：考題實戰

- 🖼️ 畫面：白底 / 上方數值範例 / 下方兩種轉換公式與結果對比
- 📣 畫面上的字：
  - 原始資料：[10, 20, 30]（mean=20, std=10, min=10, max=30）
  - 標準化 Z-score：(X - mean) / std → [-1, 0, 1]（以 0 為中心）
  - 正規化 Min-Max：(X - min) / (max - min) → [0, 0.5, 1]（縮放到 0-1）
  - 何時用標準化？→ 資料近似常態分布、用於 SVM/PCA 等對尺度敏感的演算法
  - 何時用正規化？→ 需要把值壓到固定範圍、用於神經網路/KNN
  - 考試不考計算，考的是概念和使用時機
- 🎙️ 講者這時說：「標準化和正規化是高頻考點，也是高頻陷阱。看這個例子：同樣的 [10,20,30]，標準化後變成 [-1,0,1]，正規化後變成 [0,0.5,1]。標準化的結果以 0 為中心，看的是每個值距離平均幾個標準差。正規化的結果在 0 到 1 之間，看的是每個值在最小到最大之間的相對位置。考試不會要你算，但會問你什麼時候用哪個、或者給你公式問你這是標準化還是正規化。」

## S13 · PITFALL (P4) — 標準化 vs 正規化分不清

- 🖼️ 畫面：左橘錯 / 右綠對 / 底部對照表
- 📣 畫面上的字：
  - 錯：「Min-Max 是標準化」
  - 對：「Min-Max 是正規化（Normalization），Z-score 才是標準化（Standardization）」

  | | 標準化 Standardization | 正規化 Normalization |
  |--|--|--|
  | 公式 | (X-mean)/std | (X-min)/(max-min) |
  | 結果範圍 | 無固定範圍（以 0 為中心） | [0, 1] |
  | 適用情境 | 常態分布資料、SVM/PCA | 固定範圍需求、NN/KNN |

  - 記憶法：「標=Z、正=MinMax」
- 🎙️ 講者這時說：「這張對照表必須記住。標準化=Z-score=以平均為中心。正規化=Min-Max=壓到 0-1。考題可能用中文名、英文名或公式出題，你都要能對應。記憶法：標準化的『標』長得像 Z，所以標=Z-score。正規化的『正』像 0 到 1 之間的正數，所以正=Min-Max。」

## S15 · CONCEPT-CARD — 資料隱私：個資法 + 去識別化技術

- 🖼️ 畫面：白底 / 上方個資法核心概念 / 下方三種去識別化技術比較
- 📣 畫面上的字：
  - 個資定義：可直接或間接識別特定個人的資料（姓名/身分證字號/地址/生物特徵...）
  - 個資法核心原則：告知義務、目的限制、最小蒐集、安全維護
  - 去識別化三技術：
    - 匿名化（Anonymization）：不可逆，刪除所有可識別資訊
    - 假名化（Pseudonymization）：可逆，用代號替換但保留對照表
    - 資料遮蔽（Data Masking）：部分隱藏，如身分證字號顯示 A***1234
  - 考試重點：匿名化是不可逆的，假名化是可逆的
- 🎙️ 講者這時說：「資料隱私在考試中出現的頻率越來越高。首先，個資的定義要記——只要能直接或間接識別一個人，就是個資。然後是去識別化三技術：匿名化把所有識別資訊刪到無法還原，假名化用代號替換但保留對照表所以可以還原，資料遮蔽是部分隱藏。考題最常考的是匿名化和假名化的差異——核心在可不可逆。」

## S16 · PRACTICE-PROMPT — 隱私安全情境判斷

- 🖼️ 畫面：白底 / 五個情境卡片 / 選擇對應概念
- 📣 畫面上的字：
  - 情境 1：醫院將病歷中姓名替換為隨機編號，保留對照表 → 假名化
  - 情境 2：研究機構將資料中所有可識別欄位永久刪除 → 匿名化
  - 情境 3：電商收集用戶位置資訊但未告知用戶 → 違反告知義務
  - 情境 4：公司將員工身分證字號顯示為 A1234***89 → 資料遮蔽
  - 情境 5：行銷部門將客戶資料分享給未經授權的第三方 → 違反目的限制 + 安全維護
- 🎙️ 講者這時說：「五個情境配對。關鍵判斷：有保留對照表→假名化（可逆）；永久刪除→匿名化（不可逆）；部分遮蓋→資料遮蔽。情境 3 和 5 是個資法考題——沒告知就蒐集違反告知義務，把資料給不該拿的人違反目的限制和安全維護。」

## S17 · CONCEPT-CARD — 資料安全 CIA 三原則

- 🖼️ 畫面：白底 / 三根柱子 / 每根柱子配定義和違反場景
- 📣 畫面上的字：
  - Confidentiality（機密性）：只有授權的人能存取資料 → 違反：資料庫密碼外洩
  - Integrity（完整性）：資料未被未經授權地竄改 → 違反：駭客修改交易金額
  - Availability（可用性）：需要時資料能被正常存取 → 違反：DDoS 攻擊導致系統癱瘓
  - 注意：Integrity 不是「資料完不完整」，是「資料有沒有被非法竄改」
  - 記憶口訣：CIA = 機完用（機密/完整/可用）
- 🎙️ 講者這時說：「CIA 是資訊安全的三大支柱。機密性——不該看的人看不到。完整性——資料沒有被偷偷改過。可用性——需要用的時候用得到。最大的陷阱是 Integrity——它不是問你資料有沒有缺失值，是問資料有沒有被未經授權地竄改。這個區別考題一定會考。」
