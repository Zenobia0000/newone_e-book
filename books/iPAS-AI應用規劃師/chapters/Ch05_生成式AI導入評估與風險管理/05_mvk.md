## 導入評估三維度（L1 核心）

| 維度 | 核心問題 | 子項目 |
|---|---|---|
| **技術可行性** | 能不能做？ | 資料量充足性、基礎設施、模型效能可達標 |
| **商業價值** | 值不值得做？ | 降低成本、提升效率、創造新營收 |
| **組織準備度** | 有沒有能力做？ | AI 人才、管理層支持、員工接受度 |

**考試口訣**：技術問「能不能」，商業問「值不值」，組織問「有沒有」。三維缺一不可。

## 模型效能指標-場景配對

| 指標 | 定義 | 適用場景 |
|---|---|---|
| **Accuracy** | 整體預測正確比例 | 類別平衡的通用分類 |
| **Precision** | 預測為正的真正比例 | 垃圾郵件過濾（不能誤殺正常信） |
| **Recall** | 實際為正被找出的比例 | 醫療診斷（不能漏診） |
| **F1 Score** | Precision 與 Recall 的調和平均 | 兩者都重要的折衷場景 |
| **Latency** | 單次回應時間 | 即時互動系統 |
| **Cost per query** | 單次請求成本 | 大量呼叫的成本控制 |

**考試陷阱**：「準確率最高 = 模型最好」是錯的。類別不平衡時 Accuracy 會誤導。

## 解決方案三路比較

| | 自建模型 | API 調用 | 開源微調 |
|---|---|---|---|
| **成本** | 高（GPU + 人力） | 低（按量計費） | 中（GPU + 少量人力） |
| **彈性** | 最高 | 最低 | 中高 |
| **上線速度** | 慢（月級） | 快（天級） | 中（週級） |
| **技術門檻** | 高 | 低 | 中 |
| **資料隱私** | 資料留內部 | 資料送出 | 資料留內部 |

**考試判斷**：看情境的「預算/時間/隱私/人才」四條件，逐一比對。

## 成本效益分析

### ROI（投資報酬率）
- 公式：ROI = (收益 - 成本) / 成本
- 是**比率**，衡量「每投入一元能回收多少」

### TCO（總擁有成本）
- 是**金額**，衡量「從買到退場總共花多少」
- 組成項目（必背 8 項）：
  1. 硬體採購與雲端服務
  2. 軟體授權與 API 費用
  3. AI 人力成本
  4. 資料清洗與標註
  5. 員工教育訓練
  6. 系統維運與模型再訓練
  7. 法遵與稽核成本
  8. 退場與資料處理成本

**考試重點**：最常被遺漏的是 4（標註）、5（訓練）、8（退場）。

## 經濟部《AI導入指引》五階段

```
需求確認 → 可行性評估 → PoC → 系統開發 → 部署維運
```

| 階段 | 核心任務 | 關鍵產出 |
|---|---|---|
| 1. 需求確認 | 定義問題、釐清期望 | 需求規格書 |
| 2. 可行性評估 | 技術/商業/組織三維評估 | 評估報告 |
| 3. PoC | 小規模驗證、概念證明 | PoC 結果報告 |
| 4. 系統開發 | 完整系統建置、整合測試 | 系統文件 |
| 5. 部署維運 | 上線、監控、持續改善 | SLA 與維運計畫 |

**考試送分題**：順序不可錯，需求確認永遠是第一步。

## 導入規劃五面向（L2 核心）

1. **目標設置**：用 SMART 原則，定量化 KPI
2. **資源分配**：人力、預算、時間、資料、基礎設施
3. **導入策略**：漸進式（試點→擴展）vs 全面式
4. **因應措施**：風險預案、變更管理、利害關係人溝通
5. **測試**：功能測試、壓力測試、使用者驗收測試（UAT）

**記憶口訣**：目資策因測（目標、資源、策略、因應、測試）。

## AI Agent vs 傳統 AI 工具

| | 傳統 AI 工具 | AI Agent |
|---|---|---|
| **互動模式** | 被動回應 | 主動規劃 |
| **任務範圍** | 單次執行 | 多步驟決策 |
| **環境互動** | 固定輸入輸出 | 動態環境感知 |
| **可預測性** | 高 | 較低（行為可能超預期） |
| **核心架構** | 模型 + API | 感知 → 推理 → 行動 → 回饋循環 |

**考試判斷**：「任務固定、流程明確」→ 傳統工具。「需動態規劃、多步驟推理」→ Agent。

## GenAI 三大風險維度（L3 核心）

### 1. 倫理風險
- 偏見與歧視：AI 對特定群體不公平對待
- 透明度不足：決策過程無法解釋
- 問責模糊：出錯時責任歸屬不清
- 深偽（Deepfake）：生成虛假影音內容

### 2. 資料安全與隱私
- 資料外洩：訓練資料或使用者資料被取得
- 模型竊取：模型參數或結構被逆向工程
- PII 暴露：個人可識別資訊被 AI 系統洩漏

### 3. 合規性
- 個人資料保護法：AI 處理個資的法律義務
- 著作權法：AI 生成內容的智財權爭議
- AI 基本法草案：台灣正在推動的 AI 法規框架
- 歐盟 AI Act：跨國業務需注意的域外效力

## 風險影響矩陣

|  | 低機率 | 中機率 | 高機率 |
|---|---|---|---|
| **高衝擊** | 緩解 | 緩解+轉移 | **迴避** |
| **中衝擊** | 監控 | 緩解 | 緩解+轉移 |
| **低衝擊** | 接受 | 監控 | 緩解 |

**考試速解**：看到「高-高」選迴避或緩解，看到「低-低」選接受。

---

## 風險緩解四層策略

| 層級 | 階段 | 具體措施 |
|---|---|---|
| **預防** | 設計階段 | 倫理審查、資料脫敏、存取控制、偏見檢測 |
| **偵測** | 運行階段 | 持續監控、異常偵測、定期稽核、使用者回報 |
| **回應** | 事件發生 | 事件應變計畫、通報機制、緊急停機、公關處理 |
| **復原** | 事後改善 | 影響評估、回復計畫、根因分析、改善措施 |

**考試重點**：「最優先」策略通常是預防，成本效益比最高。
