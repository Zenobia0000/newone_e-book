> 20 張內容投影片設計規格（視覺 / 一句話 / 講者腳本）

## S01 · SILENT · 起

- 視覺：深藍滿版，白字 hero：「導入 AI 不是裝一個工具，是做一個決策。考試考的是你的規劃判斷力。」
- 一句話：定錨——本章三大關鍵字：評估、規劃、風險。
- 講者腳本：這章涵蓋初級 L1/L2/L3 三個考試範疇，是初級考試的核心章節。接下來的 120 分鐘，你會學到怎麼評估該不該導入、怎麼規劃導入、導入後怎麼管風險。

## S02 · ASK · 起

- 視覺：大哉問居中：「你的公司說要導入 AI，第一步該做什麼？」右下資料卡：stat「87%」caption「企業 AI 專案未達預期效益（McKinsey 2024）」。
- 一句話：不是「選工具」，是「評估值不值得做」。
- 講者腳本：大部分人的第一反應是開始找工具、找模型。但考試要你回答的第一步是「需求確認」和「可行性評估」。這是規劃師和工程師的思維差異。

## S03 · MATRIX 2x2 · 起

- 視覺：三維評估矩陣（以三個互鎖圓環呈現）：技術可行性 / 商業價值 / 組織準備度。每個圓環內列 3 個子項目。交集區域標「Go / No-Go 決策點」。
- 一句話：導入評估三維度，缺一不可。
- 講者腳本：技術可行性問「能不能做」，商業價值問「值不值得做」，組織準備度問「有沒有能力做」。考題最愛出「以下何者不屬於導入評估範疇」，答案通常是那個你沒想到的維度。

## S04 · TABLE · 起

- 視覺：editorial_table 六列：指標名 / 適用任務 / 公式或定義 / 使用場景 / 考試陷阱。指標：Accuracy、Precision、Recall、F1、Latency、Cost per query。
- 一句話：效能指標不是越高越好，要看場景選對的那一個。
- 講者腳本：醫療診斷要高 Recall（不漏診），垃圾郵件要高 Precision（不誤殺）。考試常出「某場景應優先考慮哪個指標」，記住場景-指標配對。Latency 和 Cost 是實務導入不可忽略的。

## S05 · VS · 承

- 視覺：三欄比較表：左「自建模型」/ 中「API 調用」/ 右「開源微調」。每欄列：成本、彈性、時間、技術門檻、資料隱私。底部用紅黃綠標示各維度優劣。
- 一句話：三條路各有代價，考試要你判斷哪條路適合哪個情境。
- 講者腳本：自建最貴最彈性，API 最快最便宜但資料送出去了，開源微調折衷但需要 GPU 和人才。考題情境通常給你一家公司的條件，要你選最適合的方案。

## S06 · TABLE · 承

- 視覺：成本效益分析框架表。上半部：ROI 公式 + 計算範例。下半部：TCO 分解圖（冰山圖——水面上是硬體軟體，水面下是人力、維運、標註、法遵、退場）。
- 一句話：ROI 看報酬率，TCO 看總花費——方向不同，考題不同。
- 講者腳本：ROI = (收益 - 成本) / 成本，是比率。TCO 是從買到退場的所有成本加總。考題最愛考「TCO 包含哪些」，答案永遠比你想的多——別忘了資料標註、員工訓練、法遵成本。

## S07 · MECHANISM_FLOW · 承

- 視覺：水平五階段流程圖：需求確認 → 可行性評估 → PoC → 系統開發 → 部署維運。每階段上方標關鍵產出（需求規格書 / 評估報告 / PoC 結果 / 系統文件 / SLA），下方標核心角色。
- 一句話：經濟部《AI導入指引》五階段，背起來就是送分題。
- 講者腳本：這五個階段的順序和各階段的核心產出是高頻考題。需求確認一定在最前面——不是可行性評估。每個階段該做什麼、產出什麼、誰負責，考前默寫一遍。

## S08 · PITFALL · 承

- 視覺：vs_two_col。左「錯誤：只做技術評估就決定導入 → 上線後發現員工不會用、老闆不買單 → 專案失敗」；右「正確：技術+商業+組織三維評估 → 確認 Go → 漸進導入 → 成功擴展」。
- 一句話：只評估技術不評估組織，導入必敗。
- 講者腳本：最常見的考試陷阱：選項裡只列技術面的評估看起來很專業，但缺了商業面或組織面就不完整。記住三維缺一不可。

## S09 · MATRIX · 轉

- 視覺：五宮格矩陣：目標設置 / 資源分配 / 導入策略 / 因應措施 / 測試。每格列 2-3 個子項目和一個考試關鍵字。中央標「導入規劃五面向」。
- 一句話：規劃不是只有時程表，是五個面向的系統思考。
- 講者腳本：目標設置用 SMART 原則，資源分配包含人力預算時間資料基礎設施，導入策略分漸進式和全面式，因應措施是風險預案和變更管理，測試包含功能測試壓力測試和 UAT。考題會從任一面向出情境題。

## S10 · MECHANISM_FLOW · 轉

- 視覺：決策樹流程圖。起點「需要自主多步驟決策？」→ Yes → AI Agent → 評估自主權邊界 → 設計監控機制。No → 傳統 AI 工具 → 評估 API/自建/微調。右側附 AI Agent 架構圖：感知 → 推理 → 行動 → 環境回饋循環。
- 一句話：AI Agent 不只是更強的 ChatGPT，是能自主規劃和行動的系統。
- 講者腳本：AI Agent 是 2025-2026 考試新增方向。它的核心是感知-推理-行動的循環，能拆解複雜任務並自主執行。導入 Agent 的關鍵考量：自主權邊界在哪、人類何時介入、出錯怎麼回滾。

## S11 · VS · 轉

- 視覺：左欄「傳統 AI 工具：被動回應、單次執行、明確輸入輸出、可預測性高」；右欄「AI Agent：主動規劃、多步驟決策、動態環境互動、行為可能超預期」。底部標考試出題角度。
- 一句話：考試會問你什麼時候該用 Agent、什麼時候傳統工具就夠了。
- 講者腳本：如果任務是固定流程、輸入輸出明確，傳統工具就夠。如果需要動態規劃、多步驟推理、工具串接，才需要 Agent。考題會給情境讓你判斷。

## S12 · SILENT · 轉

- 視覺：深藍滿版：「導入是規劃問題，風險管理是存活問題。接下來講的是：AI 專案可能死在哪裡。」
- 一句話：過渡——從規劃進入風險管理。
- 講者腳本：前半段講怎麼評估和規劃，後半段講怎麼不翻車。L3 風險管理在考試中佔比 25-35%，而且題目通常是情境判斷題，不能只靠背。

## S13 · MATRIX · 轉

- 視覺：三欄矩陣：倫理風險（偏見/歧視、透明度、問責、Deepfake）/ 資安隱私（資料外洩、模型竊取、PII 暴露）/ 合規性（個資法、著作權法、AI 基本法草案、EU AI Act）。每欄用不同色系區隔。
- 一句話：GenAI 風險不只是資安，倫理和合規一樣會考。
- 講者腳本：考生最常見的盲區是把風險管理等於資安。但考題會出偏見歧視的情境、著作權爭議的案例、跨國合規的判斷。三個維度都要準備。

## S14 · TABLE · 轉

- 視覺：九宮格風險影響矩陣。橫軸：發生機率（低/中/高）；縱軸：衝擊程度（低/中/高）。九格分別標應對策略（接受/監控/緩解/轉移/迴避），並在對應格中放入具體風險範例。
- 一句話：風險不是非黑即白，是機率與衝擊的組合判斷。
- 講者腳本：考題會給你一個風險情境，問你應採取什麼策略。關鍵是先判斷機率和衝擊，再對應策略。高機率高衝擊 = 迴避或緩解，低機率低衝擊 = 接受。不要反射性選「全部迴避」。

## S15 · PITFALL · 轉

- 視覺：vs_two_col。左「錯誤：風險管理 = 買防火牆 + 加密 → 倫理爭議爆發 → 無法回應」；右「正確：資安 + 倫理審查 + 合規檢核 三管齊下 → 全面防護」。
- 一句話：只做資安防護是風險管理的一條腿，你需要三條腿才站得穩。
- 講者腳本：2024 年有公司的 AI 客服回答歧視性內容上新聞，問題不在資安，在倫理審查沒做。考試也會出這種情境題。

## S16 · PYRAMID · 合

- 視覺：四層金字塔，由底到頂：預防（設計階段）→ 偵測（運行階段）→ 回應（事件發生）→ 復原（事後改善）。每層標具體措施 2-3 項。
- 一句話：風險緩解四層策略，從預防到復原一條龍。
- 講者腳本：預防是最有效的——設計階段就做倫理審查、資料脫敏、存取控制。偵測是持續監控和定期稽核。回應是事件應變計畫和通報機制。復原是影響評估和改善措施。考題問「最優先採取的策略」，通常答案是預防。

## S17 · EXAM_DRILL · 合

- 視覺：模擬考題畫面。左側題目區顯示 3 道導入評估/規劃情境題，右側解析區逐題展開答案與解題思路。底部標計時器。
- 一句話：考題長什麼樣？現在就練。
- 講者腳本：我們來實戰三題。第一題考導入評估三維度，第二題考《AI導入指引》階段順序，第三題考成本效益分析。每題 90 秒，先自己選再看解析。

## S18 · EXAM_DRILL · 合

- 視覺：模擬考題畫面。左側 3 道風險管理判斷題，右側逐題解析。每題標示出題範疇（L3）和難度等級。
- 一句話：風險管理考的是情境判斷，不是背條文。
- 講者腳本：這三題分別考倫理風險辨識、風險影響矩陣應用、合規性判斷。注意讀題要抓「情境關鍵字」，不要被技術術語帶偏。

## S19 · PRACTICE_PROMPT · 合

- 視覺：練習題居中：「情境：一間 200 人的中型製造業要導入 AI 品檢系統。請完成：(1) 三維評估摘要 (2) 導入規劃五面向各一句話 (3) 列出前三大風險及緩解措施」。下方留 5 分鐘倒數。
- 一句話：能寫出來才算真的會。
- 講者腳本：這題整合了 L1/L2/L3 三個範疇，就是考試的綜合情境題型。五分鐘寫完，寫不出來的地方就是你的備考弱點。

## S20 · SILENT · 合

- 視覺：深藍滿版：「考試考的不是你會不會用 AI，是你敢不敢為這個決策負責。導入評估看判斷力，風險管理看責任感——這就是規劃師的核心能力。下一章：NLP/CV/多模態技術辨識。」
- 一句話：課程結語 + Ch06 預告。
- 講者腳本：Ch05 講的是決策和管理，Ch06 要進入技術層——NLP、CV、生成式 AI、多模態。考試不只考你的規劃能力，也考你的技術辨識力。準備好了，我們繼續。
