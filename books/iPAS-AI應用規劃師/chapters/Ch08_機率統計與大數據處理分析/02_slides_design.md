## S05 · MECHANISM_FLOW · 承

- 視覺：水平流程圖——「原始值 X → 減去平均值(X-mu) → 除以標準差(/sigma) → Z-score」。右側接標準常態分佈鐘形曲線，標示 68/95/99.7 的區間。下方計算範例：mu=72, sigma=8, X=88 → Z=(88-72)/8=2.0。
- 一句話：Z-score 就是「距離平均值幾個標準差」——公式一步到位。
- 講者腳本：Q10 直接考 Z-score 計算。公式：Z = (X - mu) / sigma。意義：Z=2 表示這個值高於平均 2 個標準差，在常態分佈下大約在前 2.5%。Z=0 就是平均值。|Z|>3 通常視為異常值。68-95-99.7 法則：1 個標準差內 68%、2 個 95%、3 個 99.7%。

## S06 · CODE · 承

- 視覺：左側 code panel 展示 Python Z-score 計算程式碼（numpy + scipy.stats.zscore），右側展示輸出：原始資料、Z-score 值、標記異常值（|Z|>3）。程式碼用語法高亮。
- 一句話：Python 程式題考的就是這種——讀懂 code、預測 output。
- 講者腳本：程式碼重點：`np.mean(data)` 算均值、`np.std(data)` 算標準差、`(data - mean) / std` 算 Z-score。scipy 有現成的 `zscore()` 函式。考試不需要你背 code，但要看得懂每一行在做什麼。注意：`np.std()` 預設是母體標準差（除以 N），加 `ddof=1` 才是樣本標準差（除以 N-1）。

## S07 · TABLE · 承

- 視覺：editorial_table——步驟化呈現 IQR 計算。上方：資料排序圖 + Q1/Q2/Q3 標示。下方表格：Q1=20, Q3=40, IQR=20, 下界=20-30=-10, 上界=40+30=70。右側：箱型圖（box plot）示意。
- 一句話：IQR = Q3 - Q1，異常值界線 = Q1-1.5*IQR 到 Q3+1.5*IQR。
- 講者腳本：Q7 考 IQR 異常值判斷。步驟：排序 → 找 Q1 和 Q3 → 算 IQR → 乘 1.5 → 算上下界。重點中的重點：一定要乘 1.5！考試會故意放一個「不乘 1.5」的選項。以 Q1=20, Q3=40 為例：IQR=20，下界=20-30=-10，上界=40+30=70。65 在範圍內，不是異常值；75 超出上界，是異常值。

## S09 · MATRIX · 承

## S10 · VS · 承

- 視覺：決策樹流程圖——根節點「你要比較什麼？」→ 左支「兩組平均值 → t 檢定」→ 中支「三組以上平均值 → F 檢定(ANOVA)」→ 右支「類別頻次 → 卡方檢定」。t 檢定再分「獨立兩組」和「配對」。標示 Q1 考點。
- 一句話：這張決策樹記住，Q1 直接解。
- 講者腳本：Q1 問的就是「給一個場景，你該用哪種檢定」。規則超簡單：比兩組均值 → t 檢定。比三組以上均值 → F 檢定（ANOVA）。比類別頻次 → 卡方。為什麼三組不能用 t 檢定做三次？因為多重比較會膨脹 Type I error——做三次 t 檢定，犯錯機率不是 5% 而是接近 14%。

## S12 · MECHANISM_FLOW · 轉

- 視覺：四步驟流程圖——「標準化 → 計算協方差矩陣 → 特徵值分解 → 選前 k 個主成分」。右側加一個 explained variance 的長條圖，累積到 85% 處畫紅線。標示 Q9 考點。
- 一句話：PCA 找的是「資料變異最大的方向」——降維不是丟資料，是壓縮資訊。
- 講者腳本：Q9 考 PCA。核心觀念：原始資料有 100 個特徵，但可能只有 10 個方向包含了大部分資訊。PCA 就是找到這些方向（主成分），把 100 維壓到 10 維。關鍵指標：explained_variance_ratio_——第一個主成分解釋 40%、第二個 25%、第三個 15%，累積 80%。通常累積到 85% 以上就夠了。記住：PCA 是「無監督」的——不看標籤。

## S15 · MECHANISM_FLOW · 轉

- 視覺：左側——ROC 曲線圖（X: FPR, Y: TPR），標示完美分類器（左上角）、隨機猜測（對角線）、實際模型（曲線）。AUC 用陰影面積表示。右側——混淆矩陣圖示（TP/FP/TN/FN）+ TPR 和 FPR 公式。標示 Q2 考點。
- 一句話：ROC 曲線越靠左上角越好，AUC 越接近 1 越好——0.5 是隨機猜。
- 講者腳本：Q2 考 ROC 曲線。先搞懂四個數字：TP（真陽）、FP（假陽）、TN（真陰）、FN（假陰）。TPR = TP/(TP+FN)，就是 Recall。FPR = FP/(FP+TN)。ROC 曲線就是在不同 threshold 下把 (FPR, TPR) 畫出來。AUC 是曲線下面積。記住：AUC=0.5 對角線=沒用，AUC=0.9 以上=很好，AUC=1.0=完美。

## S18 · VS · 轉

- 視覺：左右對比——左「epsilon 小（如 0.1）：噪音大、隱私保護強、資料可用性低」；右「epsilon 大（如 10）：噪音小、隱私保護弱、資料可用性高」。中間天秤圖示：隱私 vs 可用性。標示 Q8 考點。
- 一句話：差分隱私的 epsilon 是隱私和可用性的調節旋鈕——越小越安全。
- 講者腳本：Q8 考差分隱私。核心概念：查詢資料庫時，在結果加隨機噪音，讓攻擊者無法判斷某個人是否在資料集中。epsilon 是控制噪音大小的旋鈕。epsilon 越小 → 噪音越大 → 隱私越好 → 但查詢結果越不準確。考試必考題：「epsilon 減小會怎樣？」→ 隱私保護增強但資料可用性降低。

## S21 · PITFALL · 合

- 視覺：四格錯誤展示——(1) shape 搞混：把 (50000, 32, 32, 3) 的 50000 當維度。(2) axis 搞混：axis=0 是列(向下)不是行。(3) dtype 沒轉：整數除法 255 得 0，要除 255.0。(4) train_test_split 忘 random_state：結果不可重現。
- 一句話：Python 程式題四大陷阱——shape/axis/dtype/random_state。
- 講者腳本：這四個是科目2 Python 題最常見的出題點。(1) `x_train.shape` 的第一個數字是樣本數，不是維度。(2) `np.mean(X, axis=0)` 是沿列方向算每個特徵的均值，不是每個樣本的均值。(3) Python 2 時代 `255/255=1`，Python 3 `255/255=1.0`，但考試可能考 NumPy 整數陣列除法。(4) `random_state=42` 確保結果可重現——沒設的話每次跑結果不同。
