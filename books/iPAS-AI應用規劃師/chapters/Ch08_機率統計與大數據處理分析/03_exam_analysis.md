## 二、考題類型分析

### 類型 1：公式計算題（約 25%）

- 考法：給數據，要你計算 Z-score / IQR / 異常值判斷。
- 特點：數字通常可以心算，但步驟不能省。
- 範例：「某資料集 Q1=25, Q3=75，求異常值上下界」。
- 對策：記住公式 + 步驟，不要跳步。

### 類型 2：方法選擇題（約 25%）

- 考法：給場景，問該用什麼方法/檢定/分析技術。
- 範例：「比較三個品牌的顧客滿意度是否有差異」→ F 檢定（ANOVA）。
- 對策：建立決策樹，記住「場景→方法」的配對。

### 類型 3：概念解讀題（約 25%）

- 考法：給分析結果（ROC 曲線、PCA 結果、p-value），問意義。
- 範例：「模型 A 的 AUC=0.85，模型 B 的 AUC=0.72，何者較優？」。
- 對策：記住各指標的判讀規則和臨界值。

### 類型 4：Python 程式題（約 25%）

- 考法：給程式碼片段，問輸出/錯誤/填空。
- 範例：給 PCA code，問 X_pca.shape 或 explained_variance_ratio_ 加總。
- 對策：熟悉 NumPy/Pandas/sklearn/matplotlib 常用 API。

## 三、模擬試題（10 題，含 Python 程式題）

### 題 1（方法選擇 / L221）

**研究者想了解「性別」與「是否購買某產品」之間是否有關聯，最適合的檢定方法是？**

(A) 獨立樣本 t 檢定
(B) 配對樣本 t 檢定
(C) 單因子變異數分析（ANOVA）
(D) 卡方獨立性檢定

**答案：D**

**詳解**：兩個變數都是類別變數（性別：男/女；購買：是/否），要檢驗兩個類別變數是否獨立，用卡方獨立性檢定。t 檢定 (A)(B) 用於比較連續變數的均值，ANOVA (C) 用於三組以上連續變數均值的比較，都不適用於兩個類別變數的關聯性檢驗。記住：兩個類別變數的關聯 → 卡方。

### 題 2（概念解讀 / L223）

**關於 ROC 曲線與 AUC，以下敘述何者錯誤？**

(A) ROC 曲線的 X 軸是 False Positive Rate，Y 軸是 True Positive Rate
(B) AUC = 0.5 代表模型的分類能力等同於隨機猜測
(C) AUC 值越接近 1.0，模型的分類能力越好
(D) AUC 只能用於二元分類，無法用於多類別分類

**答案：D**

**詳解**：AUC 可以透過 One-vs-Rest（OVR）或 One-vs-One（OVO）策略擴展到多類別分類問題。(A) 正確：ROC 曲線的軸定義。(B) 正確：AUC=0.5 對應 ROC 曲線的對角線，等同隨機。(C) 正確：AUC 越大越好。sklearn 的 roc_auc_score 支援 multi_class 參數。

### 題 3（公式計算 / L221 + L223）

**某班 30 位學生的考試成績，平均分 75，標準差 10。學生 A 考了 95 分，其 Z-score 為何？在常態分佈假設下，約有多少比例的學生成績低於學生 A？**

(A) Z=1.5，約 93.3%
(B) Z=2.0，約 97.7%
(C) Z=2.0，約 95.0%
(D) Z=2.5，約 99.4%

**答案：B**

**詳解**：Z = (95 - 75) / 10 = 2.0。在標準常態分佈下，Z=2.0 對應的累積機率約為 0.977（97.7%），即約 97.7% 的學生成績低於 95 分。(C) 的 95% 是 Z 約 1.645 的值，不是 Z=2.0。記住 68-95-99.7 法則：2 個標準差內涵蓋 95%，所以超過 2 個標準差的在單側約 2.3%，低於的約 97.7%。

### 題 4（公式計算 / L223）

**一組資料的五數摘要為：Min=10, Q1=25, Median=40, Q3=55, Max=100。根據 IQR 法則，以下哪個值被判定為異常值？**

(A) 5
(B) 70
(C) 95
(D) 105

**答案：D**

**詳解**：IQR = Q3 - Q1 = 55 - 25 = 30。下界 = Q1 - 1.5 * IQR = 25 - 45 = -20。上界 = Q3 + 1.5 * IQR = 55 + 45 = 100。異常值 = 超出 [-20, 100] 範圍。(A) 5 在範圍內。(B) 70 在範圍內。(C) 95 在範圍內。(D) 105 > 100，是異常值。注意 Max=100 剛好在上界，不算異常；105 才超出。

### 題 5（概念解讀 / L223）

**對一個 50 維的資料集進行 PCA，前 5 個主成分的 explained variance ratio 分別為 0.35, 0.20, 0.15, 0.10, 0.05。若要保留至少 80% 的總變異量，最少需要幾個主成分？**

(A) 3 個
(B) 4 個
(C) 5 個
(D) 6 個

**答案：B**

**詳解**：累積 explained variance：PC1=35%, PC1+PC2=55%, PC1+PC2+PC3=70%, PC1+PC2+PC3+PC4=80%。4 個主成分累積 80%，剛好達標。3 個只有 70%，不夠。5 個有 85%，超出但不是「最少」。注意：題目問的是「至少 80%」，所以 80% 剛好達標。

### 題 6（方法選擇 / L222）

**某電商平台需要即時偵測使用者的異常交易行為，資料以每秒數千筆的速度持續流入。最適合的處理方式是？**

(A) 每日排程批次處理，使用 Hadoop MapReduce
(B) 串流處理，使用 Apache Kafka + Spark Streaming
(C) 將所有資料存入 PostgreSQL 後批次查詢
(D) 使用 Excel 匯入資料後手動分析

**答案：B**

**詳解**：關鍵條件：「即時偵測」+「每秒數千筆持續流入」。這是典型的串流處理場景。(A) 批次處理延遲太高，無法即時偵測。(C) 先存再查也有延遲。(D) 完全不適用於大量即時資料。Kafka 負責資料流的接收和緩衝，Spark Streaming 負責即時運算。

### 題 7（概念解讀 / L224）

**關於差分隱私（Differential Privacy），以下敘述何者正確？**

(A) epsilon 越大，隱私保護越強
(B) 差分隱私透過加密資料來保護隱私
(C) epsilon 越小，加入的噪音越大，隱私保護越強
(D) 差分隱私只適用於醫療資料

**答案：C**

**詳解**：差分隱私的核心是在查詢結果中「加噪音」（不是加密），讓攻擊者無法判斷某個人是否在資料集中。epsilon 是隱私預算參數：epsilon 越小 → 噪音越大 → 隱私越好 → 但可用性越低。(A) 反了。(B) 不是加密，是加噪音。(D) 適用於所有需要隱私保護的場景。

### 題 8（Python 程式題 / L223）

**以下 Python 程式碼的輸出是什麼？**

```python
import numpy as np

data = np.array([10, 20, 30, 40, 50])
mean = np.mean(data)
std = np.std(data)
z_scores = (data - mean) / std

print(np.round(mean, 1))
print(np.round(z_scores[0], 2))
```

(A) 30.0 和 -1.41
(B) 30.0 和 -2.00
(C) 25.0 和 -1.41
(D) 30.0 和 -1.00

**答案：A**

**詳解**：mean = (10+20+30+40+50)/5 = 30.0。std = sqrt(((10-30)^2 + (20-30)^2 + (30-30)^2 + (40-30)^2 + (50-30)^2) / 5) = sqrt((400+100+0+100+400)/5) = sqrt(200) = 14.14。z_scores[0] = (10-30)/14.14 = -20/14.14 = -1.41。注意：np.std() 預設計算母體標準差（除以 N=5），不是樣本標準差（除以 N-1=4）。如果用 ddof=1，std 會是 15.81，z_score[0] 會是 -1.26。考試要看清楚用的是哪個。

### 題 9（Python 程式題 / L223 + L224）

**以下程式碼執行後，`X_pca.shape` 的輸出為何？**

```python
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
import numpy as np

X = np.random.randn(200, 10)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

pca = PCA(n_components=0.95)
X_pca = pca.fit_transform(X_scaled)
print(X_pca.shape)
```

(A) (200, 10)
(B) (200, 0.95)
(C) (10, 200)
(D) 取決於資料，保留累積 explained variance >= 95% 所需的主成分數

**答案：D**

**詳解**：當 `n_components` 設為 0 到 1 之間的浮點數時，PCA 會自動選擇使累積 explained variance ratio >= 該值所需的最少主成分數。所以 `n_components=0.95` 不是固定保留某個數量，而是保留到累積解釋 95% 變異量為止。實際主成分數取決於資料本身。(A) 10 個特徵全保留就不需要 PCA 了。(B) 0.95 不是維度數。(C) 行列搞反。

### 題 10（Python 程式題 / L224）

**以下程式碼用於載入 CIFAR-10 資料集，哪一行的註解描述是正確的？**

```python
from tensorflow.keras.datasets import cifar10

(x_train, y_train), (x_test, y_test) = cifar10.load_data()  # 行1
print(x_train.shape)                                          # 行2：輸出 (50000, 32, 32, 3)
x_train = x_train.astype('float32') / 255.0                   # 行3：正規化到 [0, 1]
print(y_train[:5])                                             # 行4：輸出前5筆的類別標籤
```

(A) 行2 的輸出應為 (50000, 3, 32, 32)
(B) 行3 的目的是將像素值標準化為均值 0、標準差 1
(C) 行2 的輸出正確，行3 的描述正確
(D) 行4 會輸出類別名稱（如 'airplane', 'cat'）

**答案：C**

**詳解**：(A) CIFAR-10 在 Keras 中的格式是 (samples, height, width, channels)=(50000, 32, 32, 3)，不是 PyTorch 的 (channels, height, width) 格式。(B) 除以 255.0 是正規化到 [0,1]，不是標準化到均值 0 標準差 1（那需要 StandardScaler）。(C) 正確：shape 是 (50000, 32, 32, 3)，除以 255.0 確實是正規化到 [0,1]。(D) y_train 存的是整數標籤（0-9），不是文字名稱。

### 公式速查（考前必背）

| 公式 | 定義 | 常用值 |
|---|---|---|
| Z = (X - mu) / sigma | 距離平均值幾個標準差 | \|Z\|>3 為異常 |
| IQR = Q3 - Q1 | 四分位距 | — |
| 異常值下界 = Q1 - 1.5*IQR | IQR 法則下界 | — |
| 異常值上界 = Q3 + 1.5*IQR | IQR 法則上界 | — |
| TPR = TP / (TP + FN) | 真陽率 = Recall | — |
| FPR = FP / (FP + TN) | 假陽率 | — |
| AUC = ROC 曲線下面積 | 分類器總體能力 | 0.5=隨機, 1.0=完美 |

### 答題技巧

1. **計算題先寫公式**：即使心算也先寫，避免跳步出錯。
2. **檢定題看變數類型**：連續 vs 連續 → 相關/迴歸。連續 vs 類別 → t/F 檢定。類別 vs 類別 → 卡方。
3. **Python 題先看 shape**：任何 NumPy/sklearn 題目，先確認 input 和 output 的 shape。
4. **差分隱私記反比**：epsilon 小 = 隱私大 = 噪音大 = 可用性低。

### 易混淆概念

| 概念 A | 概念 B | 區分方式 |
|---|---|---|
| Z-score 標準化 | Min-Max 正規化 | Z-score 用 mu/sigma，Min-Max 用最小/最大值 |
| 母體標準差 | 樣本標準差 | 母體除 N，樣本除 N-1（np.std 預設母體） |
| t 檢定 | F 檢定 | t 比兩組均值，F 比三組以上均值 |
| 卡方適合度 | 卡方獨立性 | 適合度：一個變數觀察 vs 期望。獨立性：兩個變數是否獨立 |
| K-means | DBSCAN | K-means 需預設 k + 球型，DBSCAN 自動群數 + 任意形狀 |
| 批次處理 | 串流處理 | 批次=固定排程+完整資料，串流=即時+持續流入 |
| 正規化 /255 | 標準化 Z-score | /255 壓到 [0,1]，Z-score 轉成均值0標準差1 |
