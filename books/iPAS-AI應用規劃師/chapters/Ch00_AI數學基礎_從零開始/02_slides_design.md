## S3 · CONCEPT-CARD — 隨機變數：離散 vs 連續

- 🖼️ 畫面：左右對比。左欄「離散」：骰子圖示 + 長條圖（1-6 各 1/6）。右欄「連續」：溫度計圖示 + 鐘形曲線。中間分隔線。
- 📣 畫面上的字：
  - 離散隨機變數：值可列舉（骰子 1-6、硬幣正反）
  - P(X = x)，所有加總 = 1
  - AI 應用：分類模型的輸出 = 離散
  - 連續隨機變數：值不可列舉（身高 170.32...）
  - 用密度函數 f(x)，P(a < X < b) = 面積
  - AI 應用：迴歸模型的輸出 = 連續
- 🎙️ 講者這時說：「第一個概念：隨機變數。分兩種。離散的，像骰子，你可以把所有可能性列出來——1, 2, 3, 4, 5, 6。每個值有一個機率。連續的，像身高，你不能問某人身高『恰好 170 公分』的機率是多少——那個機率是零。你只能問一個範圍的機率。這個區分很重要，因為 AI 的分類模型輸出離散值，迴歸模型輸出連續值。」

## S4 · MECHANISM-FLOW — 條件機率 P(A|B)

- 🖼️ 畫面：上半部公式卡。下半部文氏圖（Venn Diagram）：兩個重疊圓圈 A 和 B，重疊部分標示 A∩B，箭頭指向 P(A|B) = 重疊面積 / B 的面積。右側警告框。
- 📣 畫面上的字：
  - 公式：P(A|B) = P(A ∩ B) / P(B)
  - 文字：「已知 B 發生了，A 也發生的機率是多少？」
  - ⚠️ 警告：P(A|B) ≠ P(B|A)！方向很重要
  - 例：P(有病|陽性) ≠ P(陽性|有病)
- 🎙️ 講者這時說：「條件機率是考試最愛考的陷阱。P(A 給定 B) 是已經知道 B 發生了，再問 A 的機率。看這個 Venn 圖——B 已經發生，所以你的世界縮小到 B 這個圓圈裡面。A 在 B 裡面佔多少比例，就是 P(A|B)。最重要的一句話：方向不能反。P(有病|陽性)——你檢測陽性了，真的有病的機率——和 P(陽性|有病)——你有病的情況下檢測陽性的機率——是完全不同的數字。」

## S5 · EXAMPLE-I/O — 貝氏定理 + 醫療篩檢

- 🖼️ 畫面：上方貝氏公式。中央為樹狀圖（Tree Diagram）：1000 人起始 → 有病 10 人 / 沒病 990 人 → 各分陽性/陰性。底部結論框。右側真圖佔位：貝氏定理示意圖。
- 📣 畫面上的字：
  - 公式：P(A|B) = P(B|A) × P(A) / P(B)
  - 數據設定：盛行率 1%、靈敏度 99%、特異度 95%
  - 樹狀圖數字：1000 人 → 10 有病(9.9 陽性 / 0.1 陰性) + 990 沒病(49.5 陽性 / 940.5 陰性)
  - 總陽性 = 9.9 + 49.5 = 59.4 人
  - P(有病|陽性) = 9.9 / 59.4 ≈ 17%
  - 結論框：「檢測很準(99%)但陽性結果只有 17% 是真的——因為盛行率太低」
- 🎙️ 講者這時說：「這是貝氏定理最經典的案例。假設有個疾病盛行率只有 1%——每 100 個人只有 1 個有病。檢測的靈敏度 99%——有病的人 99% 會被抓到。特異度 95%——沒病的人 95% 會正確顯示陰性。現在你檢測陽性了，你真的有病的機率是多少？很多人直覺說 99%。錯。答案只有 17%。為什麼？看這棵樹。1000 個人裡面只有 10 個有病，其中 9.9 個陽性。但 990 個沒病的人裡面有 49.5 個假陽性。所以 59.4 個陽性裡面只有 9.9 個是真的。這就是為什麼 COVID 要做二次篩檢。」

## S6 · CONCEPT-CARD — 期望值 E(X) 與變異數 Var(X)

- 🖼️ 畫面：左側公式區。右側視覺：兩個常態分佈重疊——同樣均值但不同寬度，窄的標示「Var 小=穩定」，寬的標示「Var 大=不穩定」。
- 📣 畫面上的字：
  - E(X) = Σ x_i × P(x_i) = 加權平均
  - 例：骰子 E(X) = (1+2+3+4+5+6)/6 = 3.5
  - AI 應用：損失函數 = 最小化預測誤差的期望值
  - Var(X) = E[(X - μ)²] = 離散程度
  - AI 應用：模型 variance 高 = overfitting
  - 直覺：E(X) = 中心位置 / Var(X) = 寬度
- 🎙️ 講者這時說：「期望值就是加權平均，你高中學過的。骰子的期望值是 3.5——雖然骰子不可能骰出 3.5，但平均下來就是這個數字。AI 裡面，損失函數做的事情就是算預測誤差的期望值，然後想辦法把它最小化。變異數衡量資料有多分散。右邊這張圖，兩個分佈的中心一樣，但寬度不同。窄的代表穩定，寬的代表不穩定。AI 裡面如果模型的 variance 很高，就是 overfitting——它在訓練資料上表現很好但換一批資料就不行。」

## S7 · CONCEPT-CARD — 中央極限定理 CLT

- 🖼️ 畫面：四行視覺（真圖佔位）。第一行：均勻分佈 → 抽樣均值分佈 ≈ 常態。第二行：指數分佈 → 抽樣均值分佈 ≈ 常態。第三行：雙峰分佈 → 抽樣均值分佈 ≈ 常態。底部結論。
- 📣 畫面上的字：
  - 母體什麼分佈不重要
  - 只要 n ≥ 30，抽樣平均數的分佈 → 常態分佈
  - 這就是假設檢定能用常態分佈的原因
  - 考試不考 CLT 推導，只考「為什麼假設檢定成立」
- 🎙️ 講者這時說：「中央極限定理只需要記住一件事：不管你的原始資料長什麼樣——歪的、平的、有兩個峰的——只要你抽夠多的樣本去算平均值，那些平均值的分佈一定會趨近常態分佈。這就是為什麼假設檢定可以假設常態分佈——不是因為你的資料是常態，而是因為你用的統計量（比如平均數）的分佈是常態。考試不會要你推導 CLT，但可能會問：為什麼 t 檢定可以假設常態分佈？答案就是 CLT。」

## S8 · PITFALL — Type I/II 錯誤 + α + p-value

- 🖼️ 畫面：2x2 表格（現實 vs 判斷），左上角真陽性 / 右上假陰性(Type II) / 左下假陽性(Type I) / 右下真陰性。下方兩個比喻框。底部 p-value 判斷流程。
- 📣 畫面上的字：
  - 2x2 表格：
  | | H0 為真 (沒事) | H0 為假 (有事) |
  | 拒絕 H0 | ❌ Type I (α) | ✅ 正確拒絕 |
  | 不拒絕 H0 | ✅ 正確 | ❌ Type II (β) |
  - Type I = 喊「狼來了」但沒有狼 (假陽性)
  - Type II = 狼來了但你沒喊 (假陰性)
  - α = 容忍 Type I 的上限（通常 0.05）
  - p-value < α → 拒絕 H0
  - ⚠️ 考試直接考：給情境判斷是 Type I 還是 Type II
- 🎙️ 講者這時說：「這張表你必須背下來。Type I 錯誤是你冤枉了好人——事實上沒事但你說有事。比喻：喊狼來了但其實沒有狼。alpha 就是你能忍受多少比例的冤枉。Type II 是你放走了壞人——事實上有問題但你沒發現。比喻：狼真的來了但你沒喊。p-value 的判斷很簡單：p 比 alpha 小就拒絕 H0，p 比 alpha 大就不拒絕。注意：不拒絕不等於接受——你只是沒有足夠的證據拒絕。考試會給你一個情境問你這是 Type I 還是 Type II，用這個表格對照就行。」

## S9 · CHECKPOINT — Part A 機率統計小測驗

- 🖼️ 畫面：白底三題 + 答案摺疊
- 📣 畫面上的字：
  - Q1：P(A|B) = 0.6，P(B|A) 一定也是 0.6 嗎？（否，方向不同）
  - Q2：某疾病盛行率 0.1%，篩檢陽性結果可靠嗎？（不可靠，盛行率太低 → 假陽性多）
  - Q3：p-value = 0.03，α = 0.05，你的結論？（p < α → 拒絕 H0 → 差異顯著）
- 🎙️ 講者這時說：「暫停 1 分鐘自己做。Q1 是條件機率方向——P(A|B) 和 P(B|A) 沒有任何保證相等。Q2 是貝氏定理的直覺——盛行率越低，假陽性比例越高。Q3 是 p-value 判斷的基本功——0.03 小於 0.05，拒絕虛無假設。Part A 結束，恭喜你已經具備機率統計的基礎語言了。」

## S10 · CONCEPT-CARD — 向量 = 特徵清單

- 🖼️ 畫面：左側試算表示意（一行資料 = 一個向量）。右側 2D 座標平面，向量 [3, 4] 畫成從原點出發的箭頭（真圖佔位）。
- 📣 畫面上的字：
  - x = [x₁, x₂, ..., x_d]
  - 一個樣本的 d 個特徵 = d 維向量
  - 例：房屋 = [坪數, 樓層, 屋齡] = [30, 5, 10]
  - 2D 視覺：向量 [3, 4] = 從原點到 (3, 4) 的箭頭
  - AI 應用：ML 把每一筆資料變成向量
- 🎙️ 講者這時說：「現在進入 Part B 線性代數。別緊張，從最直覺的開始。向量就是一串數字排成一列。你在 Excel 裡面一行資料就是一個向量。比如一間房子，坪數 30、樓層 5、屋齡 10 年——這三個數字排成一列 [30, 5, 10] 就是這間房子的向量。如果只有兩個特徵，你可以在 2D 平面上畫成一支箭頭。機器學習做的第一步就是把所有資料變成向量。」

## S11 · MECHANISM-FLOW — 內積 = 相似度

- 🖼️ 畫面：上方公式。中央 2D 視覺：兩個向量 a 和 b，標示夾角 theta，虛線標示投影（真圖佔位）。下方手算框。
- 📣 畫面上的字：
  - 公式：a · b = Σ aᵢ × bᵢ = |a| |b| cos θ
  - cos θ ≈ 1 → 方向相同 → 很相似
  - cos θ ≈ 0 → 垂直 → 不相關
  - cos θ ≈ -1 → 方向相反 → 很不像
  - 手算：[1, 2, 3] · [4, 5, 6] = 4 + 10 + 18 = 32
  - AI 應用：線性回歸 ŷ = w · x + b = 內積
- 🎙️ 講者這時說：「內積是線性代數裡最重要的運算之一。兩個向量的內積就是把對應位置相乘再加起來。它的幾何意義是衡量兩個向量有多『同方向』。cosine similarity 就是歸一化的內積。在 AI 裡面，線性回歸的預測值 y hat = w 點乘 x 加 b，就是一個內積運算。來做一題手算：[1, 2, 3] 點乘 [4, 5, 6]，就是 1 乘 4 加 2 乘 5 加 3 乘 6 等於 32。」

## S12 · CONCEPT-CARD — L2 範數 = 向量長度

- 🖼️ 畫面：左側公式。右側 2D 視覺：向量 [3, 4] 標示長度 = 5（畢氏定理）。底部連結框。
- 📣 畫面上的字：
  - 公式：||x||₂ = √(x₁² + x₂² + ... + x_d²)
  - 例：||[3, 4]||₂ = √(9 + 16) = √25 = 5
  - 就是畢氏定理的高維推廣
  - → 連結 Part C：L2 正則化 penalty = ||w||₂² = 限制權重的長度
- 🎙️ 講者這時說：「L2 範數就是向量的長度。你高中學過畢氏定理：3, 4, 5 直角三角形。向量 [3, 4] 的長度就是根號(9+16) = 5。推廣到高維也一樣，就是所有元素平方加起來再開根號。記住這個概念，因為等一下 Part C 講正則化的時候，L2 正則化做的事情就是限制權重向量的長度不能太大。」

## S13 · CONCEPT-CARD — 矩陣 = 一批向量

- 🖼️ 畫面：左側 Excel 試算表截圖風格（5 行 3 列，標示行=樣本、列=特徵）。右側對應的矩陣符號 X(5,3)。（真圖佔位）

## S14 · PRACTICE-PROMPT — 矩陣乘法手算

- 🖼️ 畫面：上方維度規則大字。中央完整手算範例，左矩陣 A(2,3) 和右矩陣 B(3,2) 用顏色對應標示 dot product 過程。四個結果格子依次高亮。（真圖佔位）
- 📣 畫面上的字：
  - 維度規則：(m, n) × (n, p) = (m, p)
  - ⚠️ 中間的 n 必須一樣！否則不能乘
  - A = [[1,2,3],[4,5,6]]  shape(2,3)
  - B = [[7,8],[9,10],[11,12]]  shape(3,2)
  - C = A × B = shape(2,2)
  - C[0,0] = 1×7 + 2×9 + 3×11 = 58
  - C[0,1] = 1×8 + 2×10 + 3×12 = 64
  - C[1,0] = 4×7 + 5×9 + 6×11 = 139
  - C[1,1] = 4×8 + 5×10 + 6×12 = 154
  - 考試天花板：只考 2×3 × 3×2，不會考更大的
- 🎙️ 講者這時說：「矩陣乘法是考試直接考的。規則只有一個：(m,n) 乘 (n,p) 等於 (m,p)，中間的 n 必須一樣。結果的每個位置怎麼算？第一個矩陣的某一行，和第二個矩陣的某一列，做內積。我們來算 C[0,0]：A 的第一行 [1,2,3] 和 B 的第一列 [7,9,11] 做內積 = 7+18+33 = 58。現在你自己算 C[1,1] 試試看。4 乘 8 加 5 乘 10 加 6 乘 12 等於 154。樣題不會考比這更大的矩陣，所以練熟這個 2x3 乘 3x2 就夠了。」

## S15 · MECHANISM-FLOW — 前向傳播 = Wx + b

- 🖼️ 畫面：神經網路單層示意。左邊輸入節點 x(3維)，中間權重矩陣 W(3,2)，右邊輸出節點 z(2維)，加上偏差 b(2維)。箭頭標示 z = Wx + b → activation(z)。維度標註在每個環節旁。
- 📣 畫面上的字：
  - z = W × x + b → a = activation(z)
  - 維度：輸入(batch, d_in) × 權重(d_in, d_out) = 輸出(batch, d_out)
  - 例：3 個輸入特徵 → 2 個輸出神經元
  - W 的 shape = (3, 2)，x 的 shape = (1, 3)
  - → Ch09 S17 會完整手算
- 🎙️ 講者這時說：「這就是為什麼我們花時間學矩陣乘法。神經網路的前向傳播核心就是一個矩陣乘法加上偏差再套一個激活函數。你的輸入有 3 個特徵，你要輸出 2 個值，那權重矩陣 W 的 shape 就是 (3, 2)。然後 x 乘 W 加 b 就得到 z，再套 ReLU 或 Sigmoid 就完成一層。Ch09 會帶你做完整的手算，但概念就是這麼簡單——前向傳播 = 矩陣乘法。」

## S16 · CONCEPT-CARD — 特徵值/特徵向量 = PCA 原理

- 🖼️ 畫面：左側 2D 示意（真圖佔位）：一個橢圓形資料雲，兩個箭頭標示第一主成分(長軸)和第二主成分(短軸)。右側公式與文字。
- 📣 畫面上的字：
  - Av = λv
  - 矩陣 A 作用在特徵向量 v → 只改變長度(λ)，不改變方向
  - λ = 拉伸倍率 = 特徵值
  - v = 被拉伸的方向 = 特徵向量
  - PCA = 找協方差矩陣最大特徵值的方向 = 資料變異最大的方向
  - 考試不考計算，只考概念
- 🎙️ 講者這時說：「最後一個線代概念：特徵值和特徵向量。矩陣乘以一個普通向量，方向和長度都會改變。但有些特殊向量，矩陣乘上去只會改變長度，方向不變——這些就是特徵向量，長度改變的倍率就是特徵值。PCA 的原理就是找資料的協方差矩陣的最大特徵值方向——那就是資料變異最大的方向。沿著那個方向投影就能用最少的維度保留最多的資訊。考試不考計算，只會問你 PCA 為什麼能降維。」

## S17 · CONCEPT-CARD — 損失函數 MSE vs Cross-Entropy

- 🖼️ 畫面：左右對比。左欄 MSE：公式 + 散點圖上預測線與真實值的垂直距離。右欄 Cross-Entropy：公式 + 機率輸出越偏離真實標籤懲罰越重的曲線。
- 📣 畫面上的字：
  - MSE = (1/n) Σ(y - ŷ)² → 迴歸
  - 直覺：預測偏差的平方平均
  - Cross-Entropy = -Σ yᵢ log(ŷᵢ) → 分類
  - 直覺：預測機率越偏離真實標籤，懲罰越重（log 放大錯誤）
  - ⚠️ 考試直接考：什麼任務用什麼損失函數
  - 迴歸 → MSE / 分類 → Cross-Entropy
- 🎙️ 講者這時說：「損失函數是 AI 的成績單——數字越小代表預測越準。兩大損失函數一定要分清楚。MSE 用在迴歸：你預測房價 100 萬但實際 120 萬，差距 20 萬，平方後就是 400 萬——差越多懲罰越重。Cross-Entropy 用在分類：你說這封信是垃圾郵件的機率是 0.9 但其實它不是——log 函數會把這個錯誤放大。考試常考的陷阱是把兩個搞反，記住口訣：迴歸用 MSE，分類用 CE。」

## S18 · MECHANISM-FLOW — 導數/偏微分 → 梯度

- 🖼️ 畫面：左側 2D 曲線 f(x) = x² 標示斜率線（高中複習）。右側 3D 曲面 loss(w1, w2) 標示沿 w1 方向和 w2 方向的偏微分切線。底部梯度向量箭頭（真圖佔位）。
- 📣 畫面上的字：
  - 導數 = 某點的斜率 = f'(x)（高中複習）
  - 偏微分 = 只看一個方向的斜率 ∂L/∂w₁
  - 梯度 = 所有偏微分組成的向量 ∇L = [∂L/∂w₁, ∂L/∂w₂, ...]
  - 梯度指向最陡上坡方向
  - → 沿負梯度走 = 最陡下坡 = 梯度下降
- 🎙️ 講者這時說：「導數就是斜率，高中學過。x 平方的導數是 2x，在 x=3 的斜率是 6。偏微分多了一個『偏』字，意思是函數有很多變數，你只看其中一個的變化率，其他的當作常數。在 AI 裡面，你的 loss 函數有幾千個權重，你對每一個權重算偏微分，把結果排成一個向量——這個向量就叫梯度。梯度指向最陡的上坡方向，所以反過來——沿負梯度走——就是最陡的下坡。」

## S19 · PITFALL — Learning Rate 三面板 + 更新規則

- 🖼️ 畫面：上方更新規則公式框。下方三面板並排（真圖佔位）。左面板「lr 太大」：loss 曲線上下震盪。中面板「lr 太小」：loss 曲線緩慢下降，箭頭標示「要訓練很久」。右面板「lr 剛好」：loss 曲線穩定下降到底。
- 📣 畫面上的字：
  - 更新規則：w = w - lr × ∇L
  - lr 太大 → 震盪跳過最低點 → 不收斂
  - lr 太小 → 收斂極慢 → 可能卡在局部最小
  - lr 剛好 → 穩定下降 → 到達最低點
  - ⚠️ 考試直接考：看圖判斷 lr 的問題
  - → Ch09 Adam/Adagrad 就是在自動調 lr
- 🎙️ 講者這時說：「w = w 減去 learning rate 乘以梯度。這是整個 AI 學習的核心等式。learning rate 是你每一步走多遠。走太大——你會左右震盪跳來跳去，永遠到不了谷底。走太小——你確實在往下走，但速度太慢可能訓練一週都不夠。考試常常會給你三張 loss 曲線的圖問你哪個是 lr 太大。看到震盪就是太大，看到幾乎水平就是太小。Ch09 的 Adam 和 Adagrad 做的事情就是自動調整 learning rate。」

## S20 · PRACTICE-PROMPT — 鏈式法則 + 凸性 + L1 vs L2 正則化

- 🖼️ 畫面：三段式版面。上段：鏈式法則概念圖（loss → z → w 的反傳箭頭）。中段：碗(凸) vs 崎嶇地形(非凸) 對比。下段：L1 鑽石 vs L2 圓形的約束區域圖（真圖佔位）。
- 📣 畫面上的字：
  - 鏈式法則：dL/dw = dL/dz × dz/dw → 反向傳播的數學基礎
  - 凸函數(碗)：一定找到全域最佳（線性回歸）
  - 非凸函數(崎嶇)：可能卡在局部最小（神經網路）
  - L1 正則化(Lasso)：penalty = λΣ|wᵢ| → 稀疏（很多 w=0）→ 鑽石形
  - L2 正則化(Ridge)：penalty = λΣwᵢ² → 小權重（w 接近 0 但不為 0）→ 圓形
  - ⚠️ 考試直接考 L1/L2 差異
- 🎙️ 講者這時說：「最後三個概念打包。鏈式法則：高中學過的連鎖律，反向傳播就是用它一層一層把梯度算回去。考試不考推導，只要知道反向傳播的數學基礎是鏈式法則。凸性：線性回歸的 loss 是碗形——只有一個最低點，一定找得到。神經網路的 loss 是崎嶇地形——可能卡在局部最小。正則化考試直接考。L1 的約束區域是鑽石形，角在座標軸上，所以解容易落在軸上——某些權重直接變成 0——這就是稀疏。L2 的約束區域是圓形，解不太會在軸上——權重會縮小但不會變 0。口訣：L1 鑽石稀疏、L2 圓形小。」
