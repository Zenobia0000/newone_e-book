## 2. 目標 (Learning Objectives)

完成本節後，學員應能：

1. 說明 ML 的定義、目的與核心流程（資料→訓練→泛化），能區分 ML 與傳統規則式程式。
2. 辨別監督式、非監督式、半監督式、強化式學習的特性與適用場景。
3. 說明深度學習與多模態學習的基本概念，知道它們在 ML 家族中的定位。
4. 對比鑑別式 AI 與生成式 AI 的核心差異（目標函數、輸入輸出、應用場景）。
5. 列舉 CV、語音、NLP、生成技術的典型應用場域，能判斷場景該用哪類技術。
6. 面對考題時能快速判斷 regression vs classification、supervised vs unsupervised、鑑別 vs 生成。

## 4. 教學重點 (Key Teaching Points)

1. **ML 是家族不是個體**：整章最重要的心智模型是「ML 是一個家族樹」。每種學習類型有不同的前提、適用場景、代表模型。考試考的不是數學推導，而是你能不能在場景描述中辨認出該用哪種。

2. **三層判斷流程**：(1) 有沒有 label → supervised/unsupervised；(2) label 是數字還是類別 → regression/classification；(3) 任務是判斷還是創造 → 鑑別式/生成式。這三層要讓學員練到變反射。

3. **鑑別式 vs 生成式用方向感來記**：鑑別式是「看了東西做判斷」，生成式是「憑空創造新東西」。數學上：鑑別式學 P(Y|X)，生成式學 P(X) 或 P(X|Y)。考題不會考公式，但會考方向。

4. **AI ⊃ ML ⊃ DL 同心圓**：這是最容易被混淆的階層關係。DL 是 ML 的子集，ML 是 AI 的子集。考題會出「以下何者正確」的包含關係題。

5. **應用場域要能配對**：考題常見形式是「以下哪個場景最適合用 X 技術」。CV/語音/NLP/生成各有典型應用場域，要讓學員建立配對表。

6. **強化式學習常被忽略**：多數考生只記得 supervised 和 unsupervised，但 RL 也是必考點。用 AlphaGo 當錨點，延伸到推薦系統、自動駕駛。

## 5. 考題陷阱 (Exam Traps)

- **DL = ML 混淆題**：「深度學習是機器學習的同義詞」→ 錯，是子集。
- **非監督 = 不需要資料**：「非監督式學習不需要訓練資料」→ 錯，不需要的是 label。
- **生成式 = ChatGPT 限縮題**：「以下何者為生成式 AI」→ GAN、VAE、Diffusion 都是，不只 LLM。
- **鑑別 vs 生成方向反轉**：「生成式 AI 的核心是學會分辨資料的類別」→ 錯，那是鑑別式。
- **regression vs classification 場景混淆**：「預測客戶滿意度（1-5 分）」→ 可以是 regression 也可以是 classification，取決於你把它當連續還是離散。考題會用模糊場景測試判斷力。
- **半監督式 vs 遷移學習混淆**：兩者都利用少量標注資料，但機制不同。

## 8. 常見 Q&A

**Q1：考試會考 ML 的數學公式嗎？**
A：初級不考公式推導。考的是概念理解和場景判斷——知道什麼是什麼、什麼時候用什麼。中級科目三才會涉及較多數學。

**Q2：半監督式學習在考試中常出嗎？**
A：出現頻率不高，但每次出都是鑑別度題。重點記住：少量有 label + 大量無 label = 半監督。典型場景是標注成本很高的領域（醫療、法律）。

**Q3：鑑別式和生成式可以結合嗎？**
A：可以，這正是整合應用的考點。例如 GAN 裡面的 Discriminator 就是鑑別式，Generator 就是生成式。再例如 RAG（下一章會教）就是結合檢索（鑑別）和生成。

**Q4：多模態學習會怎麼考？**
A：通常考定義和應用場景。例如「以下何者為多模態 AI 的應用？」答案會是「同時分析圖片和文字的系統」。不會考多模態的架構細節。
