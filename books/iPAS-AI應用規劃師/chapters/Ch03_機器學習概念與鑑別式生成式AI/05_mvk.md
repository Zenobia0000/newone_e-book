## ① ML 定義與核心流程（對應 LO1）

**ML vs 傳統程式**：

| | 傳統程式 | 機器學習 |
|--|---------|---------|
| 輸入 | 規則 + 資料 | 資料 + 答案 |
| 輸出 | 答案 | 規則（模型）|
| 核心 | 人寫規則 | 電腦學規律 |

**核心流程**：資料收集 → 特徵工程 → 模型訓練 → 驗證評估 → 部署上線

**關鍵術語**：
- **訓練 (Training)** = 讓模型從歷史資料學規律
- **泛化 (Generalization)** = 學到的規律能用在沒看過的新資料
- **過擬合 (Overfitting)** = 背答案，訓練好但新資料差

**考題口訣**：ML 的核心目的是泛化，不是把訓練資料記住。

## ② 四種學習類型速查（對應 LO2）

| 學習類型 | 有沒有 label | 核心特徵 | 關鍵字 | 典型場景 |
|---------|-------------|---------|--------|---------|
| 監督式 Supervised | 有 | 從 feature→label 學對應 | 預測、分類、有答案 | 房價預測、垃圾郵件、信用評分 |
| 非監督式 Unsupervised | 無 | 自己找資料結構 | 分群、探索、找結構 | 客戶分群、降維視覺化、異常偵測 |
| 半監督式 Semi-supervised | 少量有+大量無 | 借力互補 | 標注成本高、少量標注 | 醫療影像、法律文件分類 |
| 強化式 RL | 無 label（有獎勵） | Agent 環境試錯 | Agent、獎勵、策略、試錯 | AlphaGo、自動駕駛、推薦系統 |

**Supervised 再分兩種**：

| Label 類型 | 任務 | 範例 |
|-----------|------|------|
| 連續數字 | Regression | 房價、銷售額、溫度 |
| 離散類別 | Classification | 流失/不流失、垃圾/正常 |

**一句口訣**：有沒有答案 → 答案是數字還是類別 → 三秒判斷。

## ③ AI / ML / DL 包含關係（對應 LO3）

```
┌─────────────────────────────┐
│  AI（人工智慧）               │
│  ┌───────────────────────┐  │
│  │  ML（機器學習）          │  │
│  │  ┌─────────────────┐  │  │
│  │  │  DL（深度學習）    │  │  │
│  │  └─────────────────┘  │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
```

- AI ⊃ ML ⊃ DL
- DL 是 ML 的**子集**，不是同義詞
- DL 的核心價值：**自動特徵提取**（不用手動做特徵工程）
- DL 的代價：**大量資料 + 大量運算**
- DL 擅長：影像（CNN）、序列（RNN/Transformer）、生成（GAN/GPT）

**多模態學習**：同時處理多種資料類型（文字+圖片+語音）。範例：GPT-4V 看圖回答。

## ④ 鑑別式 vs 生成式 AI（對應 LO4）— 必背對比表

| 比較維度 | 鑑別式 AI | 生成式 AI |
|---------|----------|----------|
| 核心目標 | 畫邊界，做判斷 | 學分佈，做創造 |
| 數學直覺 | P(Y\|X) | P(X) 或 P(X\|Y) |
| 輸入→輸出 | 資料 → 類別/分數 | 條件 → 新內容 |
| 代表模型 | LogReg / SVM / RF / CNN | GAN / VAE / Diffusion / GPT |
| 典型應用 | 分類/辨識/偵測 | 生成/創作/合成 |
| 記憶法 | 法官做判斷 | 藝術家做創造 |

**方向判斷口訣**：
- 看到「辨識/偵測/分類/判斷」→ 鑑別式
- 看到「生成/創作/合成/產生新內容」→ 生成式

**易錯題**：
- 語音轉文字 (ASR) → **鑑別式**（辨識任務，不是生成）
- 語音合成 (TTS) → **生成式**（從文字創造語音）
- GAN 裡有 Discriminator → **整體是生成式**（D 只是組件）

---

## ⑤ 生成式 AI 家族成員（對應 LO4）

| 模型類型 | 全名 | 代表產品/應用 |
|---------|------|-------------|
| GAN | 生成對抗網路 | 圖片生成、風格轉換、DeepFake |
| VAE | 變分自編碼器 | 資料增強、異常偵測 |
| Diffusion | 擴散模型 | Stable Diffusion、DALL-E |
| LLM | 大型語言模型 | ChatGPT、Gemini、Claude |

**考試重點**：生成式 AI 不只有 ChatGPT！GAN/VAE/Diffusion 都是生成式。

## ⑥ 應用場域配對卡（對應 LO5）

| 電腦視覺 CV | 語音技術 | 生成技術 |
|------------|---------|---------|
| 人臉辨識 | 語音助理 (Siri/Alexa) | 文字生成 (ChatGPT) |
| 自動駕駛（物件偵測）| 語音轉文字 ASR | 圖片生成 (Midjourney) |
| 醫療影像判讀 | 聲紋辨識 | 程式碼生成 (Copilot) |
| 工廠品質檢測 | 語音合成 TTS | 藥物分子設計 |
| OCR 文字辨識 | 情緒語音分析 | 數據增強/合成資料 |

**易混淆**：
- 自動駕駛核心 → **CV**（不是 NLP）
- 語音轉文字 → **語音技術 + 鑑別式**（不是生成式）
- 聊天機器人 → **NLP + 生成式**（不是語音技術，除非有語音介面）

## ⑦ 三層判斷法（考場武器）

```
Step 1：有沒有 label？
         ├── 有 → Supervised
         │      ├── Label 是數字 → Regression
         │      └── Label 是類別 → Classification
         ├── 無 → Unsupervised
         │      ├── 找群組 → Clustering
         │      ├── 壓縮維度 → Dimensionality Reduction
         │      └── 找異類 → Anomaly Detection
         ├── 少量有 + 大量無 → Semi-supervised
         └── Agent + 獎勵 → Reinforcement Learning

Step 2：任務是判斷還是創造？
         ├── 判斷/分類/辨識 → 鑑別式 AI
         └── 生成/創作/合成 → 生成式 AI

Step 3：配對技術場域
         ├── 涉及圖片/影像 → CV
         ├── 涉及聲音 → 語音技術
         ├── 涉及文字理解/生成 → NLP / LLM
         └── 涉及多種資料 → 多模態
```

## 五個必踩地雷（考前必背）

| # | 地雷 | 錯 | 對 |
|---|------|----|----|
| P1 | DL = ML | 深度學習就是機器學習 | DL 是 ML 的子集 |
| P2 | 非監督不需要資料 | 非監督式不需要訓練資料 | 不需要 label，仍需大量資料 |
| P3 | 生成式 = ChatGPT | 生成式 AI 就是 ChatGPT | GAN/VAE/Diffusion/LLM 都是 |
| P4 | 鑑別生成方向反 | 生成式是做分類的 | 鑑別=判斷，生成=創造 |
| P5 | RL 只能下棋 | 強化式學習只用在遊戲 | 推薦系統/自動駕駛/資源排程都是 |
