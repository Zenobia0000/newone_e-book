## S4 · CONCEPT-CARD — ML 的定義與目的

- 🖼️ 畫面：左右對比 vs_two_col
- 📣 畫面上的字：
  - 左：傳統程式 — 人寫規則 → 電腦照做 · 輸入：規則 + 資料 → 輸出：答案
  - 右：機器學習 — 人給資料 → 電腦自己學 · 輸入：資料 + 答案 → 輸出：規則（模型）
  - 底部：「ML 的核心目的：泛化——用舊資料學到的規律，去預測新資料」
- 🎙️ 講者這時說：「這張對比圖要印在腦子裡。傳統程式是你告訴電腦怎麼做，ML 是你給電腦看很多例子讓它自己學。考題會問『以下何者為機器學習的特性？』，答案就是『從資料中自動學習規律』。」

## S5 · CONCEPT-CARD — 模型訓練與泛化

- 🖼️ 畫面：橫向流程圖五步驟
- 📣 畫面上的字：
  - 流程：資料收集 → 特徵工程 → 模型訓練 → 驗證評估 → 部署上線
  - 關鍵詞框：訓練 = 從歷史資料學規律 · 泛化 = 規律能用在新資料
  - 警告條：Overfitting = 背答案，訓練很好但遇到新題就崩
- 🎙️ 講者這時說：「泛化是 ML 的終極目標。就像你念書不是為了背考古題，而是為了遇到新題也能答對。Overfitting 就是只會背考古題的學生——考古題滿分，新題零分。考試會考：『泛化能力不足的模型具有什麼特性？』答案是 overfitting。」

## S7 · CONCEPT-CARD — 監督式學習

- 🖼️ 畫面：上方定義框 + 下方左右二分

- 🖼️ 畫面：白底五題列表 + 底部答案行

  - Q3 預測下個月銷售額 → Regression（連續數字）

## S9 · CONCEPT-CARD — 非監督式學習

- 🖼️ 畫面：上方定義框 + 下方三欄任務卡
- 📣 畫面上的字：
  - 定義：資料沒有 label · 模型自己找資料中的結構與模式
  - 左欄：分群 Clustering · K-Means / DBSCAN · 客戶分群、市場區隔
  - 中欄：降維 Dimensionality Reduction · PCA / t-SNE · 資料視覺化、特徵壓縮
  - 右欄：異常偵測 Anomaly Detection · Isolation Forest · 設備故障、網路入侵
  - 底部口訣：「沒答案 = unsupervised · 找結構 = 分群 · 找異類 = 異常偵測」
- 🎙️ 講者這時說：「unsupervised 的關鍵字是『沒有 label』和『找結構』。你不是在預測什麼，而是在探索資料本身有什麼模式。考題看到『分群』『區隔』『探索性分析』，就選 unsupervised。」

## S10 · PITFALL (P2) — 非監督式不代表不需要資料

- 🖼️ 畫面：左紅錯 / 右綠對 / 底部 why 綠條
- 📣 畫面上的字：
  - 錯：「非監督式學習不需要訓練資料」
  - 對：「非監督式不需要 label，但仍然需要大量高品質的資料」
  - Why：分群要有足夠資料才能找到有意義的群組結構；降維要有足夠維度才值得降
- 🎙️ 講者這時說：「這題考的是咬文嚼字。『不需要 label』和『不需要資料』差一個字差千里。非監督只是不用標注答案，但資料量可能比 supervised 還大——因為沒有答案引導，它需要更多例子才能自己找到模式。」

## S11 · CONCEPT-CARD — 半監督式與強化式學習

- 🖼️ 畫面：上下二分
- 📣 畫面上的字：
  - 上：半監督式 Semi-supervised · 少量有 label + 大量無 label · 場景：標注成本高（醫療影像、法律文件）· 方法：用少量標注資料訓練初始模型，再用未標注資料擴展
  - 下：強化式 Reinforcement Learning · Agent 在環境中試錯 · 透過獎勵/懲罰學最佳策略 · 場景：AlphaGo、自動駕駛、推薦系統、機器人控制
  - 底部比較：半監督「借力」，強化式「試錯」
- 🎙️ 講者這時說：「半監督的關鍵字是『標注成本高』加『少量 label』。強化式的關鍵字是『Agent』『環境互動』『獎勵』。考題會用場景包裝，但只要抓到這些關鍵字就能判斷。強化式不只是下棋——推薦系統推薦給你看的內容，你點不點就是獎勵信號。」

## S12 · CHECKPOINT — 四種學習類型配對場景

- 🖼️ 畫面：左四種學習類型 / 右四個場景 / 連線配對
- 📣 畫面上的字：

  - 場景 A：客戶分群做精準行銷 → 非監督式
  - 場景 B：垃圾郵件偵測（大量標注資料）→ 監督式
  - 場景 C：醫療影像分類（只有 200 張標注、10000 張未標注）→ 半監督式
  - 場景 D：自動駕駛車輛路徑規劃 → 強化式
- 🎙️ 講者這時說：「這就是考試的原型題。場景 C 最容易錯——你有一些 label 但不多，同時有大量未標注資料，這就是半監督式的典型場景。記住：label 少 + 未標注多 = 半監督。」

## S13 · CONCEPT-CARD — 深度學習：自動學特徵的多層神經網路

- 🖼️ 畫面：左側簡化神經網路架構圖 + 右側三個關鍵詞框
- 📣 畫面上的字：
  - 左：多層神經網路示意（輸入層 → 隱藏層 x N → 輸出層）
  - 右上：優勢：自動從原始資料提取特徵，不需手動特徵工程
  - 右中：代價：需要大量資料 + 大量運算資源（GPU）
  - 右下：擅長：影像（CNN）、序列（RNN/Transformer）、生成（GAN/GPT）
  - 底部：「DL 是 ML 的子集，專門處理非結構化資料」
- 🎙️ 講者這時說：「深度學習的核心價值就一句話：自動學特徵。傳統 ML 你要手動做特徵工程，DL 丟原始資料進去它自己學。代價是你需要很多資料和很多 GPU。考題常問：深度學習跟傳統 ML 最大的差異是什麼？答案就是自動特徵提取。」

## S14 · PITFALL (P3) — 生成式 AI 不只有 ChatGPT

- 🖼️ 畫面：左紅錯 / 右綠對 / 下方四格生成式家族卡
- 📣 畫面上的字：
  - 錯：「生成式 AI 就是 ChatGPT」
  - 對：「ChatGPT 是生成式 AI 家族的一員（LLM），但家族還有很多成員」

## S16 · CONCEPT-CARD — 鑑別式 AI：學會分辨的專家

- 🖼️ 畫面：上方定義框 + 下方應用卡列
- 📣 畫面上的字：
  - 定義：鑑別式 AI 學的是「邊界」— 給一個輸入，判斷它屬於哪一類
  - 數學直覺：學 P(Y|X) — 給定 X，Y 的機率是多少
  - 代表模型：Logistic Regression · SVM · Random Forest · CNN（分類用途）
  - 應用：垃圾郵件分類 / 人臉辨識 / 醫療影像判讀 / 信用評分 / 語音指令辨識
  - 一句話：「看了東西做判斷」
- 🎙️ 講者這時說：「鑑別式的核心就是『判斷』。你給它一張照片，它告訴你是貓是狗。你給它一封郵件，它告訴你是不是垃圾信。它不會自己創造新照片或新郵件——它只做判斷。考題看到『辨識』『偵測』『分類』『判斷』這些動詞，就是鑑別式。」

---

## S17 · CONCEPT-CARD — 生成式 AI：學會創造的藝術家

- 🖼️ 畫面：上方定義框 + 下方應用卡列
- 📣 畫面上的字：
  - 定義：生成式 AI 學的是「分佈」— 學會資料整體長什麼樣，就能生成新的
  - 數學直覺：學 P(X) 或 P(X|Y) — 資料的分佈，或給定條件下生成資料
  - 代表模型：GAN · VAE · Diffusion · Transformer(GPT) · LLM
  - 應用：文字生成 / 圖片生成 / 程式碼生成 / 語音合成 / 影片生成 / 藥物分子設計
  - 一句話：「憑條件創造新東西」
- 🎙️ 講者這時說：「生成式的核心就是『創造』。你給它一段文字描述，它畫出一張圖。你給它一個問題，它寫出一篇文章。它不只是做判斷，而是從無到有產生新內容。考題看到『生成』『創作』『合成』『產生新內容』，就是生成式。」

---

## S18 · CONCEPT-CARD — 鑑別式 vs 生成式：一張表搞定考題

- 🖼️ 畫面：中央大對比表（真圖佔位）
- 📣 畫面上的字：

  | 比較維度 | 鑑別式 AI | 生成式 AI |
  |---------|----------|----------|
  | 核心目標 | 畫邊界，做判斷 | 學分佈，做創造 |
  | 數學直覺 | P(Y\|X) | P(X) 或 P(X\|Y) |
  | 輸入→輸出 | 資料→類別/分數 | 條件→新內容 |
  | 代表模型 | LogReg/SVM/RF/CNN | GAN/VAE/Diffusion/GPT |
  | 典型應用 | 分類/辨識/偵測 | 生成/創作/合成 |
  | 一句話 | 看了東西做判斷 | 憑條件創造新東西 |

- 🎙️ 講者這時說：「這張表就是你的考場武器。碰到任何鑑別 vs 生成的題目，腦子裡浮現這六行對比。最好記的方式：鑑別式是法官（做判斷），生成式是藝術家（做創造）。」

## S19 · CHECKPOINT — 鑑別式 vs 生成式場景判斷

- 🖼️ 畫面：白底五題列表 + 鑑別/生成標記
- 📣 畫面上的字：
  - Q1 信用卡詐騙偵測 → 鑑別式（判斷是否詐騙）
  - Q2 自動生成行銷文案 → 生成式（創造新文字）
  - Q3 手機人臉解鎖 → 鑑別式（判斷是否為機主）
  - Q4 AI 繪圖（文字→圖片）→ 生成式（創造新圖片）
  - Q5 語音轉文字 → 鑑別式（辨識語音內容，輸出對應文字）
  - 陷阱提醒：Q5 容易被「轉」字誤導以為是生成。語音轉文字是辨識任務，不是從無生成。
- 🎙️ 講者這時說：「第五題是經典陷阱。語音轉文字看起來好像在『生成』文字，但本質是『辨識』——它在判斷語音對應的是什麼字。真正的生成式語音任務是語音合成，像 TTS（Text-to-Speech）——從文字生成語音。方向反過來才是生成。」
