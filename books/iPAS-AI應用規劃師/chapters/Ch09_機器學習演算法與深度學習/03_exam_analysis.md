## 四、模擬試題（10 題）

### 模擬題 1（類型 A — 演算法選型）

某電商公司想根據客戶的購買紀錄、瀏覽行為等特徵，將客戶自動分成不同族群以進行差異化行銷。已知群集形狀不規則且有雜訊資料。下列哪種演算法最適合？

(A) Linear Regression
(B) K-means
(C) DBSCAN
(D) Logistic Regression

**答案：(C)**

**詳解**：這是一個非監督式的分群任務（沒有預先定義的類別標籤）。群集形狀不規則排除了 K-means（假設球狀群集），且有雜訊資料需要自動排除。DBSCAN 基於密度的分群方法能找到任意形狀的群集，且會自動將低密度區域的點標記為噪音。Linear Regression 和 Logistic Regression 是監督式演算法，不適用於分群任務。

### 模擬題 2（類型 B — 架構辨識）

下列關於 CNN 經典架構的描述，何者正確？

(A) VGG 的核心創新是在同一層使用不同大小的卷積核並行處理
(B) Inception (GoogLeNet) 的設計哲學是「加寬不加深」，使用多尺度卷積
(C) ResNet 透過增加更多的全連接層來解決梯度消失問題
(D) VGG 使用殘差連接（skip connection）來訓練超深網路

**答案：(B)**

**詳解**：Inception 的核心設計是在同一層放置 1x1、3x3、5x5 不同大小的卷積核並行處理，再合併結果——這就是「加寬不加深」。(A) 描述的是 Inception 而非 VGG。VGG 的特色是用 3x3 小卷積核簡單堆疊。(C) ResNet 是用殘差連接（skip connection）而非全連接層。(D) 殘差連接是 ResNet 的特色，不是 VGG。

### 模擬題 3（類型 C — R² 計算）

一個 Linear Regression 模型的 R² = 0.85，下列解讀何者最正確？

(A) 模型的預測準確率為 85%
(B) 模型解釋了目標變數 85% 的變異
(C) 模型的誤差率為 15%
(D) 模型能正確分類 85% 的樣本

**答案：(B)**

**詳解**：R²（判定係數）表示模型解釋了目標變數多少百分比的變異。R² = 0.85 表示模型解釋了 85% 的變異，剩下 15% 的變異是模型無法捕捉的。(A) R² 不等於準確率。(C) R² 不直接等於誤差率。(D) R² 用於迴歸任務，不是分類任務的指標。

### 模擬題 4（類型 B — 激活函數）

建構一個卷積神經網路（CNN）進行手寫數字辨識（0-9 共 10 類），下列關於激活函數的配置何者最合適？

(A) 隱藏層使用 Softmax，輸出層使用 ReLU
(B) 隱藏層使用 Sigmoid，輸出層使用 Sigmoid
(C) 隱藏層使用 ReLU，輸出層使用 Softmax
(D) 隱藏層使用 Tanh，輸出層使用 ReLU

**答案：(C)**

**詳解**：隱藏層用 ReLU 是標準做法——計算快速且能有效緩解梯度消失問題。輸出層的 10 類分類任務用 Softmax——它會輸出每個類別的機率，加總為 1。(A) Softmax 不適合隱藏層。(B) Sigmoid 在隱藏層會導致梯度消失，且 Sigmoid 輸出 0-1 的單一數值，不適合多類分類。(D) ReLU 不適合作為分類任務的輸出層。

### 模擬題 5（類型 C — 矩陣乘法前向傳播）

一個單層神經網路的輸入 X = [2, 3]，權重矩陣 W = [[0.5, -0.1], [0.3, 0.2]]，偏差 b = [0.1, -0.2]，激活函數為 ReLU。請問輸出為何？

(A) [1.5, 0.2]
(B) [2.0, 0.4]
(C) [2.0, 0.2]
(D) [1.5, 0.0]

**答案：(C)**

**詳解**：
- Z = X · W + b
- Z[0] = 2 × 0.5 + 3 × 0.3 + 0.1 = 1.0 + 0.9 + 0.1 = 2.0
- Z[1] = 2 × (-0.1) + 3 × 0.2 + (-0.2) = -0.2 + 0.6 - 0.2 = 0.2
- ReLU(2.0) = 2.0, ReLU(0.2) = 0.2
- 輸出 = [2.0, 0.2]

### 模擬題 6（類型 D — Python 程式題）

以下 Python 程式碼的執行結果最接近什麼含義？

```python
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
pca = PCA(n_components=0.95)
X_reduced = pca.fit_transform(X_scaled)
print(f"原始維度: {X.shape[1]}, 降維後: {X_reduced.shape[1]}")
```

(A) 將資料隨機挑選 95% 的特徵
(B) 將資料標準化後，用 PCA 降維至保留 95% 累積變異的最少維度數
(C) 將資料壓縮到原始維度的 95%
(D) 隨機保留 95% 的樣本數量

**答案：(B)**

**詳解**：`PCA(n_components=0.95)` 中的 0.95 是一個小數，代表保留 95% 的累積解釋變異量，PCA 會自動決定需要多少主成分才能達到這個門檻。程式先用 StandardScaler 標準化（PCA 前必須做），再進行降維。不是隨機挑特徵，也不是固定壓縮到 95% 維度。

### 模擬題 7（類型 A — 演算法特性）

下列關於 K-means 和 DBSCAN 分群演算法的比較，何者正確？

(A) K-means 不需要事先指定群數，DBSCAN 需要
(B) K-means 能找到任意形狀的群集，DBSCAN 只能找球狀群集
(C) DBSCAN 能自動標記噪音點，K-means 會將所有點歸入某群
(D) 兩者對初始參數的設定都不敏感

**答案：(C)**

**詳解**：DBSCAN 基於密度分群，不屬於任何群集核心區域的點會被標記為噪音（outlier），而 K-means 會強制將每個點分配到最近的群心。(A) 反了——K-means 需要事先指定 K，DBSCAN 不需要。(B) 反了——K-means 假設球狀群集，DBSCAN 能找任意形狀。(D) K-means 對初始群心位置敏感，DBSCAN 對 eps 和 min_samples 敏感。

### 模擬題 8（類型 B — 優化器）

下列關於深度學習優化器的描述，何者正確？

(A) SGD 會自動調整每個參數的學習率
(B) Adagrad 的學習率會持續增加，導致訓練後期震盪
(C) Adam 結合了動量（Momentum）和自適應學習率的特性，是目前最常用的優化器
(D) 所有優化器的效果相同，只是計算速度不同

**答案：(C)**

**詳解**：Adam（Adaptive Moment Estimation）結合了 Momentum 的動量效果和 Adagrad/RMSprop 的自適應學習率，是目前深度學習中最常用的預設優化器。(A) SGD 的學習率是固定的，需要手動設定。(B) Adagrad 的學習率會持續「衰減」（不是增加），導致後期幾乎停止更新。(D) 不同優化器在不同場景下效果差異很大。

### 模擬題 9（類型 D — Python 程式題）

以下程式碼的 `???` 處應填入什麼，才能完成 K-means 分群並取得每個樣本的群集標籤？

```python
from sklearn.cluster import KMeans
import numpy as np

X = np.array([[1, 2], [1, 4], [1, 0],
              [10, 2], [10, 4], [10, 0]])

model = KMeans(n_clusters=2, random_state=42)
labels = model.???(X)
print(labels)
```

(A) predict(X)
(B) fit_predict(X)
(C) transform(X)
(D) fit_transform(X)

**答案：(B)**

**詳解**：`fit_predict(X)` 會同時進行訓練（fit）和預測（predict），回傳每個樣本的群集標籤。(A) `predict(X)` 需要先 `fit` 才能用，單獨使用會報錯。(C) `transform(X)` 會回傳每個樣本到各群心的距離，不是群集標籤。(D) `fit_transform(X)` 也是回傳距離矩陣。在 K-means 中，要直接取得標籤用 `fit_predict` 或先 `fit` 再存取 `model.labels_`。

### 模擬題 10（類型 B — 跨概念整合）

一個深度學習模型在訓練集上的 loss 持續下降趨近於 0，但在驗證集上的 loss 先下降後上升。下列處理方式何者最不適合？

(A) 加入 Dropout 層
(B) 增加更多的訓練資料
(C) 增加模型的層數和神經元數量
(D) 使用 Early Stopping 在驗證集 loss 最低點停止訓練

**答案：(C)**

**詳解**：題目描述的是典型的 Overfitting——訓練集表現越來越好但驗證集表現先好後差。(A) Dropout 是 regularization 的一種，隨機關閉神經元防止過度依賴，適合。(B) 更多資料能增加模型的泛化能力，適合。(D) Early Stopping 在最佳點停止，直接阻止 overfitting，適合。(C) 增加層數和神經元會讓模型更複雜，反而加劇 overfitting，是最不適合的。
