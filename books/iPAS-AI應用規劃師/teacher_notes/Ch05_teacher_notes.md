## 1. 本節目標 (Learning Objectives)

完成本節後，學員應能：

1. 運用「技術可行性 × 商業價值 × 組織準備度」三維框架評估 AI 導入方案。
2. 熟悉模型效能評估指標（Accuracy / F1 / Latency / Cost），能在考題中正確配對。
3. 掌握成本效益分析（ROI / TCO）與隱性成本辨識，不被表面數字誤導。
4. 背熟經濟部《AI導入指引》五階段流程，這是送分題。
5. 區分 AI Agent 與傳統 AI 工具的差異，應對代理相關新題型。
6. 辨識 GenAI 三大風險維度（倫理/資安隱私/合規性），能判斷風險影響層級。

## 3. 關鍵教學點 (Key Teaching Points)

1. **導入評估三維度缺一不可**：技術可行性（現有技術能否達標？資料量足夠？）、商業價值（降多少成本？提多少效率？）、組織準備度（有沒有 AI 人才？管理層支持？）。考題常用「以下何者不屬於導入評估範疇」出題。

2. **模型效能指標要看場景**：「準確率高 = 模型好」是錯的。醫療偏重 Recall（不能漏診）、垃圾郵件偏重 Precision（不能誤殺）。Latency 和 Cost 也是效能指標的一部分。

3. **ROI 和 TCO 方向不同**：ROI = (收益-成本)/成本，是「值不值得」的比率。TCO = 硬體+軟體+人力+維運+標註+退場成本，是「花多少」的總額。考題會考 TCO 包含哪些項目。

4. **經濟部五階段是送分題**：需求確認 → 可行性評估 → 概念驗證（PoC）→ 系統開發 → 部署維運。階段順序必須記死。

5. **AI Agent 是新考點**：與傳統 AI 工具的差異——被動回應 vs 主動規劃、單次執行 vs 多步驟決策。2025-2026 新增考題方向。

6. **風險管理不只是資安**：三大維度——倫理風險（偏見/透明度/深偽）、資安隱私（資料外洩/模型竊取）、合規性（個資法/著作權/AI 法規）。風險影響矩陣的九宮格分類要能看懂。

## 4. 學員常犯錯誤 (Common Pitfalls)

- **導入評估 = 技術評估**：忘記組織面與商業面，選擇題直接被誘答。
- **ROI 與 TCO 混淆**：ROI 是比率，TCO 是總成本金額，方向不同。
- **《AI導入指引》階段順序記錯**：需求確認必在最前面，不是可行性評估。
- **風險管理 = 資安**：考題會出倫理與合規的情境，只會資安拿不到分。
- **AI Agent 題型不熟**：新題型，容易因為沒準備直接放棄。

## 5. 提問設計 (Discussion Prompts)

1. 如果老闆說「幫我評估要不要導入 AI 客服」，你會從哪三個面向切入？（預期答：技術可行性、商業價值、組織準備度。）

2. 一家公司的 AI 專案 TCO 裡面，最容易被低估的是哪些項目？（預期答：資料標註成本、員工教育訓練、模型再訓練、法遵成本。）

3. AI Agent 和 ChatGPT 差在哪裡？考試會怎麼考這個差異？（預期答：Agent 具備感知-推理-行動循環、主動規劃；ChatGPT 被動回應。）

## 7. 常見 Q&A

**Q1：經濟部《AI導入指引》的五階段一定要背嗎？**
A：一定要。幾乎每次考試都出。不只要記順序，還要知道每個階段的核心任務和關鍵產出。

**Q2：ROI 計算會考嗎？**
A：初級可能考簡單的 ROI 概念（什麼是 ROI、包含哪些因素），不會考複雜計算。但要能識別「隱性成本」。

**Q3：AI Agent 的考題會很難嗎？**
A：目前是新題型，預期以概念辨認為主——「以下何者是 AI Agent 的特徵」「AI Agent 與傳統 AI 工具的差異」。不會考 Agent 的技術實作。

**Q4：風險影響矩陣要會畫嗎？**
A：不用畫，但要會「讀」。考題可能給你一個九宮格矩陣，問某個風險屬於哪個象限，應該用什麼策略（接受/監控/緩解/迴避）。
