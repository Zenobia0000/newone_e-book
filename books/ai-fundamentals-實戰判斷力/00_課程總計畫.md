**時間定錨：** 2026 版

## 一、課程定位

**一句話：** 2026 入門 AI，真正必學的不是把四大流派各背一遍，而是先建立一套「從問題 → 資料 → 模型 → 評估 → 上線 → 風險」的實戰判斷力。

Stanford 2025 AI Index 指出，2024 年已有 **78% 的組織表示正在使用 AI**，高於前一年的 55%。AI 已經進入工作流，而不是只停留在研究室。所以初學者最需要學的，不是追新模型，而是學會如何在真實場景中做正確技術選擇。

本課程不是傳統學科切法（統計一章、ML 一章、DL 一章、GenAI 一章），而是用「系統性決策思考流程」展開，讓學習者學一套可實戰的方法：

1. **這個問題到底值不值得用 AI？**
2. **如果要用，先用統計、傳統 ML、DL，還是 GenAI？**
3. **資料夠不夠？風險在哪？**
4. **怎麼驗證不是做出一個看起來很厲害、其實沒法上線的 demo？**

結業後，學員應能：

1. 面對新問題時，判斷該用統計、ML、DL 還是 GenAI
2. 具備資料觀：知道標籤從哪來、有沒有 leakage、類別是否失衡
3. 對四大流派各掌握「必要最小集」的核心概念
4. 建立評估能力：不靠感覺驗收，會用結構化測試
5. 理解 GenAI 系統工程：RAG / tool use / agent / evals
6. 知道 AI 治理的最小必要集：幻覺、注入、隱私、可追溯性

**本課程不教**：完整數學推導、SOTA 模型架構深入、雲端部署實作、MLOps pipeline 完整實作。

## 四、教學理念

### 1. 決策導向，不是學科導向

傳統 AI 入門按學科切（統計→ML→DL→GenAI），學完四科卻不知道該用哪個。本課程用「系統性決策思考流程」串聯：問題界定 → 理解全貌 → 建立系統模型 → 假設與驗證 → 全方位檢視 → 決策與行動。每章的練習都圍繞「做決策」而非「背定義」。

### 2. 最小可用知識（MVK）

每個流派只教「必要最小集」。統計不講整本數理統計，只講判斷可信度的骨架。DL 不鑽每個 SOTA 架構，只抓訓練機制。GenAI 不教 prompt 花招，教系統結構。

### 3. 資料觀排在模型前面

不管哪一派，初學者最需要的是資料觀：標籤從哪來、特徵可不可用、訓練與上線是否同分布、有沒有 leakage。模型可能只佔 20%，其餘 80% 是資料與驗證。

### 4. 評估能力是必修

2026 的 AI 不是「感覺答案不錯」就好。OpenAI 把 evals 定義為結構化測試，強調要避免 vibe-based evals。本課程把評估從附錄提升到獨立章節。

### 5. 風險與治理不是選修

NIST GenAI Profile 明講要把可信度考量放進 AI 的設計、開發、使用與評估。初學者至少要知道幻覺、注入攻擊、隱私洩漏、可追溯性。

## 五、章節間的串接邏輯

```
Ch00 選武器 ─────────────────────────────────────┐
  │ 問題定義三問：預測/分類/生成？可解釋？錯誤代價？│
  ├─→ 統計路線 → Ch01                              │
  ├─→ ML 路線   → Ch02                              │
  ├─→ DL 路線   → Ch03                              │
  └─→ GenAI 路線 → Ch04                             │
                                                    │
Ch01 統計 ← Ch02 ML 的「結果能不能信」               │
Ch02 ML   ← Ch03 DL 的「tabular 場景不必上 DL」     │
Ch03 DL   ← Ch04 GenAI 的「Transformer 前導」       │
Ch04 GenAI ← Ch05 的「eval 與 safety」               │
                                                    │
Ch05 評估與治理 ←──── 匯流所有章節 ─────────────────┘
```

**兩條主線**：
- **技術選型主線**：Ch00 → Ch01-Ch04（學會四種武器）
- **品質保證主線**：Ch01 統計檢定 → Ch02 ML 驗證 → Ch04 GenAI eval → Ch05 治理（貫穿全課的評估意識）
