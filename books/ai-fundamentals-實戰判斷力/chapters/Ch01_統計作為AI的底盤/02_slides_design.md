## S2 · ASK — 你看到的變化是真的還是巧合？

- 🖼️ 畫面：白底中央問句 + 資料卡

## S4 · CONCEPT-CARD — 描述統計三兄弟

- 🖼️ 畫面：左欄文字三指標定義 + 右欄兩組同平均不同分布的直方圖（真圖佔位）
- 📣 畫面上的字：
  - 集中趨勢 — mean / median / mode（平均在哪裡）
  - 離散程度 — std / range / IQR（散多開）
  - 分布形狀 — 對稱 / 偏態 / 雙峰（長什麼樣）
  - 記憶點：平均數只是故事的 1/3
- 🎙️ 講者這時說：「描述統計有三兄弟，你不能只帶大哥（平均數）出門。集中趨勢告訴你中心在哪、離散程度告訴你資料散多開、分布形狀告訴你這組資料長什麼樣。右邊這兩張圖，平均數一模一樣都是 75——但你一看分布就知道完全不一樣。這就是為什麼只看平均數會被騙。」

## S5 · PITFALL (P1) — 只看平均數就下結論

- 🖼️ 畫面：左紅錯 / 右綠對 / 底部 why 綠條
- 📣 畫面上的字：錯：Dashboard 只看 mean=75 → 報告「兩組表現相同」；對：加看 std + histogram → 發現 B 組是雙峰分布，需要拆開分析
- 🎙️ 講者這時說：「真實案例——某電商的客單價平均 500 元，PM 很開心。結果一畫分布圖：大部分人買 100 元，少數人買 5000 元把平均拉上去。策略應該針對哪群人？只看平均數你永遠不知道。養成反射：看到數字先畫分布。」

## S6 · CHECKPOINT — 三兄弟配對

- 🖼️ 畫面：白底 / 三張迷你 dashboard / 底部判斷題
- 📣 畫面上的字：三張 dashboard：A 只報 mean、B 報 mean+std、C 報 mean+std+histogram；問：哪張最可靠？哪張在騙你？
- 🎙️ 講者這時說：「三十秒看這三張圖。A 只有一個數字——最危險。B 多了離散度——好一點。C 有分布圖——這才是完整的故事。以後你做報告或看別人報告，標準就是 C。」

## S7 · CONCEPT-CARD — 相關 vs 因果 + Confounders

- 🖼️ 畫面：上方相關散佈圖 + 下方因果路徑圖含 confounder（真圖佔位）
- 📣 畫面上的字：
  - 相關（correlation）：兩個變數一起動
  - 因果（causation）：A 真的造成了 B
  - Confounder：隱藏的第三者 C 同時影響 A 和 B
  - 經典案例：冰淇淋銷量 ↑ 溺水人數 ↑ → confounder = 夏天氣溫
- 🎙️ 講者這時說：「這可能是你今天學到最重要的一張投影片。在 AI 領域，你會看到一堆相關性——feature importance 高、兩個指標一起動。但你不能直接說『A 造成了 B』。中間可能有個你沒看到的 C 在搞事。冰淇淋和溺水的例子大家都聽過——C 是夏天氣溫。」

---

## S8 · CHECKPOINT — 找出 Confounder

- 🖼️ 畫面：白底三題 / 每題一個「X 和 Y 相關」的敘述
- 📣 畫面上的字：Q1 鞋碼越大語文成績越好？Q2 咖啡喝越多薪水越高？Q3 手機用越久近視越深？各找出 confounder
- 🎙️ 講者這時說：「一分鐘小組討論。提示：Q1 的 confounder 是年齡——大人腳大、也讀更多書。Q2 是工作壓力或職位——高壓高薪的人喝更多咖啡。Q3 比較有趣，可能真的有因果，但也可能 confounder 是『室內活動時間』。」

---

## S9 · PITFALL (P2) — 把相關當因果的真實代價

- 🖼️ 畫面：左紅錯 / 右綠對 / why 綠條 / 底部銜接 A/B test
- 📣 畫面上的字：錯：「用戶停留時間↑ 轉換率↑」→ 拚命加長頁面 → 跳出率飆升；對：設計 A/B test 驗證因果 → 發現是「高意圖用戶」本來就停留久
- 🎙️ 講者這時說：「這是真實發生的事。某團隊看到停留時間和轉換率正相關，就把頁面加長、加更多內容。結果跳出率反而飆升。為什麼？因為不是停留久才買——是想買的人本來就會看久。相關的方向對了，因果的方向錯了。怎麼驗證？A/B test，我們等一下就會講。」

## S10 · PRACTICE-PROMPT — 新聞標題判讀

- 🖼️ 畫面：白底三則新聞標題 + 兩欄判斷表格
- 📣 畫面上的字：標題 A「研究發現吃巧克力的國家諾貝爾獎越多」/ 標題 B「每天運動 30 分鐘降低心臟病風險 40%」/ 標題 C「使用本產品的客戶滿意度高達 95%」；判斷：因果 or 相關？有無抽樣偏差？
- 🎙️ 講者這時說：「兩分鐘，跟旁邊的人討論。每則標題回答兩個問題：這是因果還是相關？資料的樣本有沒有問題？提示：C 的抽樣偏差最嚴重——95% 滿意，因為不滿意的人早就不用了。」

---

## S11 · CONCEPT-CARD — 抽樣偏差三類型

- 🖼️ 畫面：左欄三類型定義 + 右欄二戰飛機彈孔示意圖（真圖佔位）
- 📣 畫面上的字：
  - 倖存者偏差（Survivorship Bias）：只看到活下來的
  - 選擇性偏差（Selection Bias）：樣本不是隨機的
  - 便利抽樣（Convenience Sampling）：只問身邊的人
  - 經典案例：二戰飛機——Wald 說要裝甲「沒回來的飛機」被打到的地方
- 🎙️ 講者這時說：「二戰時軍方要幫轟炸機加裝甲，他們看回來的飛機哪裡彈孔多就加哪裡。統計學家 Wald 說錯了——你看到的是『活著回來的』，彈孔多代表打那裡不會死。要加裝甲的是『沒有彈孔的地方』——因為打中那裡的飛機都沒回來。這就是倖存者偏差。」

---

## S12 · PRACTICE-PROMPT — 判斷三個資料集的偏差

- 🖼️ 畫面：白底三情境 + 判斷欄
- 📣 畫面上的字：情境 A App 評分只看 5 星和 1 星用戶 / 情境 B 面試只分析錄取者的表現 / 情境 C 線上問卷只有年輕人填；各是什麼偏差？影響什麼結論？
- 🎙️ 講者這時說：「兩分鐘判斷。A 是選擇性偏差——中間態度的人不評分，你看到的是兩極。B 是倖存者偏差——你不知道被拒的人會不會表現更好。C 是便利抽樣——你的結論只適用於年輕人，不能推論到全體。」

## S13 · PITFALL (P4) — 資料量大 ≠ 沒偏差

- 🖼️ 畫面：左紅錯 / 右綠對 / why 綠條
- 📣 畫面上的字：錯：「我有十萬筆資料！」但全來自台北 → 代表性為零；對：100 筆全國隨機抽樣 → 代表性更高
- 🎙️ 講者這時說：「資料量大讓人安心，但安心不等於正確。十萬筆台北資料推論全台灣消費行為？不行。一百筆全國隨機抽樣反而更可靠。以後看到任何資料集，第一個問題不是『有多少筆』而是『從哪來的、誰被排除了』。」

## S14 · CHECKPOINT — 中場回顧

- 🖼️ 畫面：白底三題快問
- 📣 畫面上的字：Q1 看 dashboard 除了平均數還要看什麼？（std + 分布圖）Q2 相關≠因果，中間的隱藏變數叫什麼？（confounder）Q3 「只分析回覆問卷的人」是什麼偏差？（倖存者/選擇性偏差）
- 🎙️ 講者這時說：「三十秒快答，不看投影片。這三個問題你以後每天都會用到——看 dashboard、讀報告、審資料。答不出來的回去看 S4、S7、S11。」

## S15 · MECHANISM-FLOW — 假設檢定四步驟

- 🖼️ 畫面：flow_chain 四節點橫排，中間「p-value」節點 highlight
- 📣 畫面上的字：
  - Step 1：設定 H0（沒差異）和 H1（有差異）
  - Step 2：決定顯著水準 α（通常 0.05）
  - Step 3：收資料 → 算 p-value
  - Step 4：p < α → 拒絕 H0；p ≥ α → 不拒絕 H0
  - 白話：假設檢定就是問「如果沒差，看到這組資料有多離譜？」
- 🎙️ 講者這時說：「假設檢定不需要你手算，但你必須懂這四步。第一步：先假設沒差異，這叫 H0。第二步：定一個門檻，通常 5%。第三步：收資料算出 p-value。第四步：p-value 比門檻低就拒絕 H0，白話就是『不太可能沒差，所以我認為有差』。注意措辭——不是『證明有差』，是『不太可能沒差』。」

---

## S16 · PRACTICE-PROMPT — 寫 H0 和 H1

- 🖼️ 畫面：白底三個場景 + 空白 H0/H1 填寫區
- 📣 畫面上的字：場景 A 新推薦演算法 vs 舊的 / 場景 B 改按鈕顏色提升點擊率 / 場景 C 新藥 vs 安慰劑；請寫出各自的 H0 和 H1
- 🎙️ 講者這時說：「兩分鐘練習。提示——H0 永遠是『沒差異』或『沒效果』。A 的 H0：新舊演算法的點擊率沒有差異。H1：新演算法的點擊率比舊的高。注意 H1 可以是單尾（比較好）也可以是雙尾（不一樣），看你的問題怎麼問。」

## S17 · PITFALL (P3 + P5) — p-value 的兩大誤用

- 🖼️ 畫面：左紅兩個錯誤 / 右綠正確做法 / 底部 why 綠條
- 📣 畫面上的字：
  - 誤用 1：樣本太少就判定 → 假陽性（p 會亂跳）
  - 誤用 2：p < 0.05 = 有用 → 忽略 effect size（十萬筆資料什麼都顯著）
  - 對：看信賴區間寬度 + effect size + 商業門檻
  - 口訣：統計顯著 ≠ 實務有意義
- 🎙️ 講者這時說：「p-value 有兩個最常見的陷阱。第一個：樣本太少，p-value 會隨機亂跳，你今天 p=0.03 明天 p=0.2——不是結果變了，是你的資料不夠穩定。第二個更危險：當你有一千萬筆資料，幾乎什麼差異都能 p < 0.05——但 effect size 可能小到 0.001，毫無商業價值。所以一定要搭配看：信賴區間有多寬？effect size 有多大？對業務有意義嗎？」

## S18 · CHECKPOINT — 模型提升 2% 的判斷流程

- 🖼️ 畫面：白底情境題 + 三個問題清單
- 📣 畫面上的字：情境：同事說 accuracy 82% → 84%，問你可不可以上線；你該問：① 測試樣本量多大？② 信賴區間多寬？③ 這 2% 對業務指標影響多少？
- 🎙️ 講者這時說：「回到開場的問題。現在你有工具了。第一問樣本量——100 筆測試資料的 2% 可能只是噪音。第二問信賴區間——如果 84% 的 CI 是 [80%, 88%]，跟舊的 82% 重疊，就不能說有差。第三問 effect size——accuracy 提升 2% 但轉換率沒動，上線的意義在哪？這三個問題就是你的判斷武器。」

## S19 · CONCEPT-CARD — A/B Test 全流程

- 🖼️ 畫面：五步驟 flow 橫排 + 底部 checklist
- 📣 畫面上的字：
  - Step 1：定義目標指標（conversion rate / CTR / retention）
  - Step 2：算 MDE（最小可偵測差異）+ 所需樣本量
  - Step 3：隨機分群（確保兩組基線一致）
  - Step 4：跑到預定時間（不要偷看！peeking problem）
  - Step 5：顯著性判斷（p-value + CI + effect size）
  - Checklist：✓ 指標明確 ✓ 樣本量夠 ✓ 隨機分群 ✓ 跑滿時間 ✓ 看 CI 不只看 p
- 🎙️ 講者這時說：「A/B test 不是隨便分兩組看誰贏。五個步驟每個都能出錯。最常見的錯誤是 Step 4——跑了兩天看到 p < 0.05 就興奮地停下來，這叫 peeking problem，假陽性率會從 5% 膨脹到 30%。一定要事前決定好跑多久、需要多少樣本，然後忍住不偷看。」

## S22 · MOTIVATION — 銜接 Ch02

- 🖼️ 畫面：問句 + 下一章資料卡
- 📣 畫面上的字：「你現在會問『這個差異是真的嗎？』——下一章要問『這個 pattern 能拿來預測嗎？』」；卡片：Ch02 · 機器學習：表格世界的主戰場
