> **受眾定位**：已有 Python 基礎、準備進入 AI 領域的學習者
> **前置知識**：基本 Python 操作（變數、迴圈、函式）
> **後續銜接**：Ch02 機器學習、Ch05 評估與治理

---

## 1. 本節目標 (Learning Objectives)

完成本節後，學員應能：

1. 用描述統計三指標（集中趨勢 / 離散程度 / 分布形狀）正確摘要一組資料，不被平均數騙。
2. 區分相關與因果，識別 confounders 的存在並舉出生活實例。
3. 說明抽樣偏差（sample bias / selection bias）如何讓資料「看起來有差但其實沒有」。
4. 用假設檢定流程（H0/H1 → 收資料 → 算 p-value → 判斷）回答「差異是否顯著」。
5. 設計一個基本的 A/B test，包含樣本量估算、分群、顯著性判斷。
6. 看到模型提升 2%，知道要先問信賴區間與樣本量，再決定是否上線。

## 3. 關鍵教學點 (Key Teaching Points)

1. **描述統計三兄弟是看 dashboard 的護身符**：非統計背景的學員看到平均數就下結論，這是最常見的認知陷阱。一開始就用「兩班同平均不同分布」的例子炸開這個誤解，後面每次提到 dashboard 都會回指這張三兄弟圖。「平均數只是故事的 1/3」這句話要重複 3 次以上。

2. **相關≠因果是 AI 時代最重要的批判思維**：學員會從模型看到 feature importance 高就以為「這個特徵造成了結果」。冰淇淋與溺水的例子要講透，confounder 的概念要紮根。這件事直接影響 Ch02 特徵選擇和 Ch05 模型解釋。

3. **抽樣偏差用二戰飛機案例建立直覺**：Wald 的故事比任何定義都有效。「你只看到回來的飛機」這句話映射到 AI 場景就是「你只看到回覆問卷的用戶 / 你只看到留下來的客戶」。

4. **假設檢定不教公式，教流程**：受眾不需要手算 t-statistic。他們需要的是：(a) 知道什麼時候該用假設檢定、(b) 會設 H0/H1、(c) 看得懂 p-value 的意義、(d) 知道 p-value 不是萬能的。四步驟 flow chart 是核心教具。

5. **p-value 的兩大誤用必須當場拆穿**：「p < 0.05 = 有效」是學員最根深蒂固的迷信。要用兩個反例打破：(a) 一千萬筆資料什麼都顯著、(b) effect size 0.001 再顯著也沒商業價值。信賴區間比 p-value 更該看。

6. **每個統計觀念都配一個 AI 場景**：不講純統計，每個概念都搭配「模型提升 X% 該怎麼判斷」「A/B test 結果該怎麼讀」的場景，讓學員帶著「這些東西在 AI 工作第一天就用得到」的感覺離開。

## 4. 學員常犯錯誤 (Common Pitfalls)

- **只看平均數就下結論**：Dashboard 上只顯示 mean，忽略 std 和分布形狀。→ 養成「看數字先畫分布」的反射。
- **把相關當因果**：看到兩個指標一起動就改策略，忽略 confounders。→ 先問「有沒有第三個變數同時影響兩者？」
- **A/B test 樣本量不足就判定贏家**：跑了三天覺得 p < 0.05 就停，其實是隨機波動。→ 事前算好所需樣本量。
- **抽樣偏差不自覺**：只分析「有回覆的用戶」「有留下的客戶」，結論偏到不行。→ 先問「誰被排除了？」
- **p-value < 0.05 就等於有用**：統計顯著 ≠ 實務有意義。→ 搭配 effect size + 信賴區間一起看。

## 5. 提問設計 (Discussion Prompts)

1. 你負責的推薦系統 accuracy 從 82% 提升到 84%，PM 問你「可以上線嗎？」——你會先問哪三個問題？（預期答：測試樣本量多大？信賴區間多寬？這 2% 對業務指標（如轉換率）影響多大？）

2. 公司資料團隊報告說「使用新功能的用戶留存率高 15%」——這能證明新功能提升留存嗎？為什麼不能？（預期答：選擇性偏差——願意嘗試新功能的用戶本來就是高活躍用戶，需要 A/B test 驗證因果。）

3. 一家電商的 A/B test 結果：A 版轉換率 3.2%、B 版 3.5%，p=0.04——你會直接切到 B 版嗎？（預期答：要看信賴區間寬度、effect size 是否有商業價值、樣本量是否足夠、有無多重比較問題。）

## 7. 常見 Q&A

**Q1：我以後做 AI 真的需要懂統計嗎？不是都丟給模型就好？**
A：模型會給你一個數字，但「這個數字可不可信」是統計在回答的問題。你不需要手推公式，但你需要知道什麼時候該懷疑、該問什麼問題。這就是本章在建立的「判斷力底盤」。

**Q2：p-value 到底是什麼意思？為什麼不能直接當成「機率」？**
A：p-value 是「假設沒有差異的情況下，看到這麼極端的資料的機率」。它不是「差異真的存在的機率」。這個微妙差別很重要——p=0.03 不代表「有 97% 的機率差異是真的」。

**Q3：A/B test 要跑多久才夠？**
A：取決於你要偵測的最小差異（MDE）和你的日流量。經驗法則：用 power analysis 算出所需樣本量，除以你的日流量就是天數。千萬不要「看到顯著就停」——那叫 peeking problem，會嚴重膨脹假陽性率。

**Q4：相關≠因果我懂了，那 AI 領域有什麼方法可以逼近因果？**
A：這是進階議題。短答：A/B test（隨機實驗）是因果推論的黃金標準。觀察性資料可以用 instrumental variables、difference-in-differences、propensity score matching 等方法，但都有前提假設。本課先建立「看到相關先打問號」的反射即可。
