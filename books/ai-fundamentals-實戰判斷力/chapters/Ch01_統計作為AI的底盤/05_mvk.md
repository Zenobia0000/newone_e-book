# Ch01 · MVK 速學卡（Minimum Viable Knowledge）

> 離開本節後，你的肌肉記憶裡必須長出這三個反射 + 六個 LO 的心智模型。
> 對應 01_outline.md 的 6 個 Learning Objectives。
> 本章的核心判斷：**統計不是舊時代遺物，是判斷資料與結果可信度的語言。**

---

## ① 描述統計三兄弟（對應 LO1）

| 指標 | 問的問題 | 常用工具 | 陷阱 |
|------|---------|---------|------|
| 集中趨勢 | 中心在哪裡？ | mean / median / mode | mean 被極端值拉走 → 看 median |
| 離散程度 | 散多開？ | std / range / IQR | std=0 才代表「都一樣」 |
| 分布形狀 | 長什麼樣？ | histogram / boxplot / KDE | 雙峰分布用 mean 毫無意義 |

**一句口訣**：平均數只是故事的 1/3 → 看數字先畫分布。

## ② 相關 ≠ 因果（對應 LO2）

```
相關 (correlation)：A 和 B 一起動
因果 (causation)：A 真的造成了 B
Confounder：隱藏的 C 同時推動 A 和 B

經典案例：
  冰淇淋銷量 ↑  溺水人數 ↑
        ↖          ↗
       夏天氣溫 (C)

判斷反射：
  看到「X 和 Y 相關」→ 先問「有沒有第三個變數 C？」
  驗證因果的黃金標準 → 隨機實驗 (A/B test)
```

**記憶法**：CCC — Correlation is not Causation, Check for Confounders.

## ③ 抽樣偏差三類型（對應 LO3）

| 類型 | 白話 | AI 場景 |
|------|------|---------|
| 倖存者偏差 | 只看到活下來的 | 只分析「留下的用戶」→ 忽略流失原因 |
| 選擇性偏差 | 樣本不隨機 | 只用回覆問卷的人 → 極端意見過度代表 |
| 便利抽樣 | 只問身邊的人 | 只用公司內部資料 → 無法推論到外部 |

**一句口訣**：資料量大 ≠ 沒偏差 → 看資料先問來源，問「誰被排除了？」

**經典案例**：二戰飛機——Wald 說要裝甲「沒回來的飛機」被打到的地方，不是回來的飛機身上有彈孔的地方。

## ④ 假設檢定流程（對應 LO4）

```
Step 1：設定假設
  H0（虛無假設）：沒有差異 / 沒有效果
  H1（對立假設）：有差異 / 有效果

Step 2：決定門檻
  顯著水準 α = 0.05（業界慣例）

Step 3：收資料 → 算 p-value
  p-value = 假設 H0 為真時，看到這組資料（或更極端）的機率

Step 4：判斷
  p < α  → 拒絕 H0（「不太可能沒差」→ 認為有差）
  p ≥ α  → 不拒絕 H0（「有可能沒差」→ 無法判定）
```

**白話翻譯**：假設檢定就是問——「如果真的沒差，我看到的這組資料有多離譜？」

**兩大誤用必記**：

| 誤用 | 錯在哪 | 怎麼修 |
|------|--------|--------|
| 樣本太少就判定 | p-value 不穩定，假陽性高 | 事前用 power analysis 算所需樣本量 |
| p < 0.05 = 有用 | 十萬筆什麼都顯著；effect size 可能極小 | 搭配看信賴區間 + effect size + 商業門檻 |

**口訣**：統計顯著 ≠ 實務有意義。

## ⑤ A/B Test Checklist（對應 LO5）

```
□ 1. 定義目標指標（conversion rate / CTR / retention / revenue）
□ 2. 算 MDE（最小可偵測差異）+ 所需樣本量（power analysis）
□ 3. 隨機分群（確保兩組基線一致——年齡、地區、活躍度）
□ 4. 跑到預定時間（不要偷看！peeking problem → 假陽性膨脹）
□ 5. 顯著性判斷（p-value + 信賴區間 + effect size）
```

**最常犯的錯**：跑兩天看到 p < 0.05 就停 → peeking problem 讓假陽性率從 5% 膨脹到 30%+。

## ⑥ 模型提升 X% 的判斷武器（對應 LO6）

同事說模型 accuracy 從 82% 提升到 84%，你該問三件事：

| 問題 | 為什麼要問 | 紅旗信號 |
|------|-----------|---------|
| 測試樣本量多大？ | 100 筆的 2% 可能是噪音 | 樣本 < 1000 筆 |
| 信賴區間多寬？ | CI [80%, 88%] 跟舊的 82% 重疊 | CI 寬度 > 提升幅度 |
| Effect size 有商業價值嗎？ | accuracy +2% 但轉換率沒動 | 業務指標無變化 |

**口訣**：看差異先問顯著，問完顯著問意義。

## 三件帶走的事（全課核心）

1. **看數字先看分布** → 平均數只是故事的 1/3
   ↳ 銜接 **Ch02 特徵工程**（特徵的分布影響模型表現）
   ↳ 銜接 **Ch05 指標監控**（上線後的指標要看分布不只看均值）

2. **看差異先問顯著** → 統計顯著 ≠ 實務有意義
   ↳ 銜接 **Ch02 模型比較**（兩個模型哪個好，要用統計判斷）
   ↳ 銜接 **Ch05 A/B test 上線決策**（p-value + CI + effect size 三位一體）

3. **看資料先問來源** → 資料量大 ≠ 沒偏差
   ↳ 銜接 **Ch03 訓練資料品質**（GIGO: Garbage In, Garbage Out）
   ↳ 銜接 **Ch05 公平性與偏差審計**（模型偏差往往來自資料偏差）
