# Ch00 Minimum Viable Knowledge — MVK 速學卡

> **本文件定位：** Ch00 的核心知識濃縮為 5 張速學卡，每張含「一句話結論 + 速查表或記憶術 + 常見誤解 + 跨章節連結」。
> **讀者：** 已上過 Ch00 需複習的學員、要快速建立 AI 選型判斷力的 PM 與決策者。
> **使用情境：** 課後 48 小時記憶鞏固期的快速複習；在公司 AI 專案啟動會議前 10 分鐘的自我校準。

---

## 卡 01 — 四大流派速查表

**一句話結論：** AI 不是一種技術，是四個工具箱——統計、ML、DL、GenAI——各有甜蜜點，選錯比不選更貴。

**速查表：**

| 流派 | 驅動力 | 甜蜜點 | 典型輸入 | 典型輸出 | 不適合 |
|------|--------|--------|----------|----------|--------|
| 統計 | 假設驅動 | 因果推論、A/B Test | 小~中樣本 | p-value、信賴區間 | 非結構化大量資料 |
| ML | 特徵工程 | 表格預測、分類 | 結構化表格 | 機率值、類別標籤 | 需表徵學習的場景 |
| DL | 表徵學習 | 影像、語音、序列 | 非結構化大量資料 | 嵌入向量、標籤 | 小樣本 + 需高解釋性 |
| GenAI | 大規模預訓練 | 文本生成、對話、多模態 | 自然語言提示 | 生成內容 | 高精度表格預測 |

**記憶術（口訣）：** **「統因、機表、深像、生話」** — 統計做因果、機器學習做表格、深度學習做影像、生成式做對話。

**常見誤解：** 以為四者是進化淘汰關係。錯——Google 到今天做 A/B Test 還是用統計，Kaggle 表格冠軍還是 XGBoost。

**跨章節連結：** Ch01 統計底盤 → Ch02 ML 表格戰場 → Ch03 DL 表徵學習 → Ch04 GenAI 系統工程。

## 卡 02 — 五種任務型態分類器

**一句話結論：** 選模型之前，先把問題歸類到五種任務之一——歸錯類，後面全白做。

**分類速查：**

| 任務型態 | 一句話定義 | 輸入 → 輸出 | 業務案例 | 首選流派 |
|----------|------------|-------------|----------|----------|
| 預測 | 用歷史推未來連續值 | 時序數據 → 數值 | 營收預估、需求預測 | ML / 統計 |
| 分類 | 貼標籤 | 特徵 → 離散標籤 | 垃圾信、流失預警 | ML / DL |
| 生成 | 從無到有創造內容 | 提示 → 文字/圖/音 | 文案、程式碼、圖片 | GenAI |
| 檢索 | 從既有知識找答案 | 查詢 → 相關段落 | 內部知識庫、FAQ | GenAI (RAG) |
| 代理 | 多步自主行動 | 目標 → 工具呼叫序列 | 訂單處理、研究助手 | GenAI (Agent) |

**快速判斷法：**
```
你的問題要的是一個「數字」嗎？ → 預測
要的是一個「標籤」嗎？         → 分類
要的是「新內容」嗎？           → 生成
要的是「已有的文件段落」嗎？   → 檢索
要的是「一連串動作」嗎？       → 代理
```

**常見誤解：** 把檢索和生成混為一談。RAG 的重點是「先檢索再生成」，不是純生成。如果你的場景需要引用來源、不允許幻覺，那是檢索任務而非純生成。

**跨章節連結：** Ch02 的 ML 處理前三種任務；Ch04 的 GenAI 處理後三種任務。

## 卡 03 — 模型能力 vs 系統能力

**一句話結論：** Demo 很厲害 ≠ 能上線；模型能力佔成本 20%，系統能力佔 80%。

**對比速查：**

| 維度 | 模型能力 (20%) | 系統能力 (80%) |
|------|---------------|----------------|
| 衡量什麼 | Accuracy / F1 / BLEU | Latency p99 / Uptime / $/query |
| 誰在意 | 資料科學家 | PM / SRE / 法務 |
| 何時重要 | 實驗階段 | 上線之後（才開始） |
| 壞掉長什麼樣 | 預測不準 | 服務掛掉 / 帳單爆炸 / 被罰款 |
| 怎麼修 | 換模型 / 加資料 | 快取 / 監控 / 回饋迴圈 / 合規審查 |

**記憶公式：**
```
系統能力 = 模型能力 + 延遲 + 穩定性 + 安全 + 合規 + 成本控制 + 可觀測性
```

**常見誤解：** 以為「模型準確率 95%」就可以上線。上線需要的是：95% 準確率 + p99 延遲 < 200ms + 99.9% 可用性 + 符合 GDPR + 每月成本 < 預算。少一個都不行。

**跨章節連結：** Ch05 評估與治理會展開系統能力的完整框架。

## 卡 04 — AI 專案生命週期六階段

**一句話結論：** AI 專案是迴圈不是瀑布——上線不是結束，是治理的開始。

**六階段速查：**

```
[問題界定] → [資料] → [建模] → [評估] → [部署] → [治理]
     ↑                                              |
     └──────────── 回饋迴圈 ─────────────────────────┘
```

| 階段 | 關鍵問題 | 關鍵產出 | 最常跳過的事 |
|------|----------|----------|-------------|
| 1 問題界定 | Who / What / Why / How | 問題定義文件 | 定義成功指標 |
| 2 資料 | 資料在哪？品質如何？ | 清理後的資料集 + Schema | 資料品質評估 |
| 3 建模 | 哪種流派？哪個模型？ | 訓練好的模型 + 實驗紀錄 | 基線比較 |
| 4 評估 | 離線指標夠嗎？線上如何？ | A/B Test 結果 | 做 A/B Test |
| 5 部署 | API？批次？邊緣？ | 上線的服務 | CI/CD 流程 |
| 6 治理 | 漂移？合規？回饋？ | 監控儀表板 + 告警 | 持續監控 |

**常見誤解：** 以為做完第 3 步（建模）就完成了 60%。實際上第 3 步只完成了 20%，後面三步才是大頭。更致命的是跳過第 1 步——沒有定義清楚問題就開始建模，等於閉著眼射箭。

**跨章節連結：** Ch01-Ch04 各展開第 3 步的不同流派；Ch05 展開第 4-6 步。

## 卡 05 — 選武器五問清單

**一句話結論：** 走進任何 AI 專案前，由上到下問五個問題，能回答完再動手。

**五問清單（由上到下）：**

```
┌─────────────────────────────────────────────┐
│  Q1  這個問題值不值得用 AI？                    │
│      → 如果 Excel / 規則引擎能解決，不需要 AI    │
├─────────────────────────────────────────────┤
│  Q2  是預測、分類、生成、檢索還是代理？           │
│      → 歸類到五種任務型態之一                    │
├─────────────────────────────────────────────┤
│  Q3  資料在哪？品質如何？標註夠嗎？              │
│      → 沒有資料就沒有 AI                        │
├─────────────────────────────────────────────┤
│  Q4  統計 / ML / DL / GenAI 哪個成本效益最高？   │
│      → 用卡 01 速查表對照                       │
├─────────────────────────────────────────────┤
│  Q5  模型做完之後，系統能撐得住嗎？              │
│      → 延遲、成本、安全、合規、監控               │
└─────────────────────────────────────────────┘
```

**實戰速記（案例對照）：**

| 案例 | Q1 值得？ | Q2 任務 | Q3 資料 | Q4 流派 | Q5 系統 |
|------|----------|---------|---------|---------|---------|
| 客戶流失 | 值得（高 LTV） | 預測 | 3 年交易 log | ML | 批次推論即可 |
| 客服回覆 | 值得（降人力） | 生成+檢索 | 知識庫文件 | GenAI (RAG) | 需 7x24 + 合規 |
| 瑕疵檢測 | 值得（品質成本） | 分類 | 5000 張標註影像 | DL (CNN) | 邊緣部署 + 低延遲 |
| 知識問答 | 值得（效率） | 檢索 | 10 年文件庫 | GenAI (RAG) | 內網部署 + 權限控管 |

**常見誤解：** 跳過 Q1 直接問 Q4（「我們該用什麼模型？」）。最昂貴的錯誤不是選錯模型，是解決了一個不值得解決的問題。

**跨章節連結：** 這五個問題會在 Ch01-Ch05 每一章的開頭以不同深度重新出現。

## 如何使用這組速學卡

1. **課後 48 小時**：每天看 2 張，重點讀「常見誤解」並自我檢測。
2. **專案啟動前**：打開卡 05 的五問清單，逐題回答，寫成一頁文件帶進啟動會議。
3. **向老闆報告時**：用卡 03 的模型 vs 系統對比表，解釋為什麼不能只看 demo。
4. **團隊討論時**：用卡 01 的速查表，快速對齊「我們這個問題該用哪種流派」。
5. **面試準備**：把五張卡的「一句話結論」練到能用 30 秒對面試官解釋。
