# 課程總計畫：Python 資料分析與資料工程實戰

**版本：** 1.0  
**總時數：** 22 小時（基礎 10h + 實戰 12h）  
**階段數：** 2 階段 / 11 章節  
**投影片數：** 244 張內容頁

---

## 一、課程定位

本課程是給「**已學過 Python 基礎語法、非理工資訊背景**」學員的 **資料分析 / 資料工程銜接課**。學員背景常為商管、人文、醫護、設計、行銷轉職或自學者，已會變數 / 條件 / 迴圈 / 函式，但：
- 沒有計算機概論 / OS 基礎（看到 OOM、路徑跨平台、I/O Bound 會卡）
- 沒有 OOP 概念（無法看懂業界開源程式碼、無法寫可重用的資料管線）
- 只會「能跑就好」的腳本寫法（維護成本高、無法規模化）

結業後，學員應能：

1. 看懂進階開源資料科學程式碼（DataFrame / ndarray / class / apply）
2. 撰寫**模組化、可重用**的資料處理管線（DataCleaner 類別）
3. 熟練 NumPy 向量化、Pandas 進階清洗、時序 EDA、視覺化呈報
4. 產出互動式 dashboard（Plotly）作為 capstone 專案
5. 具備系統直覺，知道大檔 OOM 怎麼處理、I/O 慢瓶頸在哪

**本課程不教**：scikit-learn / PyTorch / TensorFlow / 雲端 / Airflow（結業後的下一站路線，於 F5、S6 給出學習地圖）。

## 四、教學理念

### 1. 分階段：基礎 → 工具

F 階段不急著教工具，先把底盤鋪好（系統直覺 + OOP 思維）。S 階段才進入 NumPy / Pandas / 視覺化，此時學員已有「為什麼要這樣寫」的理解力，不只學語法。

### 2. 教學型投影片設計（Teaching-Track）

全課程投影片套用 `shared/design_system/教學型投影片_黃金守則與泛式.md` 的 **T1–T12 黃金守則**與 **7 種教學原型**：

| 原型 | 用途 |
|------|------|
| MOTIVATION | 每章開場的痛點情境 |
| CONCEPT-CARD | 核心概念定義（左文右圖） |
| MECHANISM-FLOW | step-by-step 原理 |
| EXAMPLE-I/O | 明確標示 Input → Process → Output |
| PITFALL | 錯誤寫法 vs 正確寫法（紅綠對比） |
| PRACTICE-PROMPT | 章節中段練習題（非結尾） |
| CHECKPOINT | 每 5 張插一張檢核點 |

### 3. 節奏：四拍循環

```
概念（CONCEPT-CARD）
  ↓
例子（EXAMPLE-I/O）
  ↓
問題（ASK / PRACTICE）
  ↓
小結（SILENT / PYRAMID）
```

每 4-6 張完成一輪；一章 20-24 張 = 3-4 個循環。

### 4. Pitfall 配額

每章對應 teacher_notes 的 Common Pitfalls，每條 pitfall 必配一張 PITFALL 頁。

### 5. 章節銜接（Bridge）

每章末含「→ 銜接下一章」導引；F 階段末段（F5）與 S 階段開頭（S1）明確交棒。

## 十一、與既有 18h 課程的差異化

本倉庫另有 `python-data-engineering-18h`（10 章 Ch01-Ch10）。定位差異：

| 維度 | 18h 課程 | 本 DA_DE 課程 |
|------|---------|-------------|
| 對象 | 已有基礎、想進階資料工程 | 非理工背景 / 僅 Python 基礎轉職者 |

| 重心 | OOP 模組化 + Pandas 工程化 | 系統直覺 + 工具實戰 + dashboard |
| Capstone | OOP × Pandas 整合（DataCleaner） | DataCleaner + Plotly dashboard.html |
| 口吻 | 教科書式嚴謹 | 非理工親民 + 專業技術並重 |
| 風格 | 顧問型投影片 | 教學型投影片（7 原型 + T1-T12 守則） |

兩課程**教材資產可共享**（shared/design_system/），但獨立開發、互不耦合。
