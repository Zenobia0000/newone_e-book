# 篇章規劃（建議 3 篇最順）

## 第 1 篇：AI 寫的程式碼你會全部檢查嗎？

### 1) 開場鉤子（3 句內）

* 丟問題：**「你敢把 AI 生成的 code 直接 merge main 嗎？」**
* 立場：你不是極端派，你是中間派（有趣、真實）
* 丟一個畫面感：

  * 「完全不看很爽，爽到第三天開始長出義大利麵條…」

### 2) 你的現行 SOP（你已寫得很好，這段直接保留但更像規格書）

* **Branch + PR：**

  * 一律開分支
  * 功能切小、每個功能一個 commit
  * PR 才進 main
* **檢查深度按風險分級：**

  * 安全（登入/權限）：手動檢查，必要時重寫
  * 簡單/重複：掃過
  * 核心/複雜：AI 起草，你重構架構確保可維護
  * 後期整理：請 AI 抓 dead code / 重複邏輯，你再總 review

### 3) 你踩過的坑（這段是情緒的靈魂）

* 「為了趕案子半個月沒 review → 交付前要改功能 → 架構亂到爆 → 打掉重寫」
* **編輯建議：把這段寫成一個小故事**

  * 時間點（那段日子）
  * 痛點（改不動）
  * 代價（重寫）
  * 學到的教訓（現在寧願前期多花時間）

### 4) 提升成「可複製」的框架（讓讀者覺得：哦我也能用）

* **風險矩陣（兩軸就夠）**

  * 軸 1：影響面（改壞會影響多少人/錢/資料）
  * 軸 2：不確定性（你對這段邏輯熟不熟）
* **對應 review 深度**

  * 高影響＋高不確定：逐段推理＋補測試＋必要重構
  * 高影響＋低不確定：重點檢查＋測試補齊
  * 低影響＋高不確定：先跑起來但加監控/保護
  * 低影響＋低不確定：快速掃過即可

### 5) PR Checklist（讀者最愛收藏）

* **必看（紅區）**

  * 權限/認證/資料寫入
  * 外部輸入（任何 user input / webhook / file upload）
  * 金流、個資、刪除操作
* **快掃（黃區）**

  * 重複邏輯是否該抽共用
  * 命名一致性、錯誤處理
  * 依賴套件是否多拉了不必要的
* **放行（綠區）**

  * 純 UI 文案、樣式調整（前提：不動核心流程）

### 6) 結尾互動（把留言引出來）

* 問兩題就好，別太多

  * 「你是逐行派、信任派、還是像我這種風險派？」
  * 「你們的紅區清單會放哪些？」

## 第 2 篇：怎麼學 Vibe Coding 最快？直接仿製別人的系統！

> 這篇主軸很清楚：**仿製＋介面化最快**。把它寫成「教練式」會很有穿透力。

### 1) 開場一句（你原句就很強）

* **「仿製＋介面化，入門最快。」**
* 補一個挑釁但好笑的：

  * 「不要先研究名詞，先做一個能按的東西。按下去會出貨那種。」

### 2) 五層進化論（保留，但建議每層都加一句「可觀察行為」）

* 1 單次問答：問完就走
* 2 工作流交互：AI 進你的流程，開始幫你做事
* 3 系統化運作：流程固定化、可重複
* 4 自動化＋介面化：按鈕化、可交付給別人用
* 5 生態整合：工具互相串起來，像軍團

### 3) 打破幻想（你那句「不能！」很有戲）

* 「一堂課直衝第五層？」→ **不能**
* 原因不是你不會教，是因為：

  * 第五層本質是「大量情境的肌肉記憶」

### 4) 最大痛點：缺乏真實情境（這段可以更像宣言）

* 「AI 沒用」很多時候不是 AI 的問題
* 是你沒把它丟進戰場
* 你的方法：拿去市場驗證、看回饋迭代

### 5) 具體例子（這段超有說服力）

* Replit 印刷管理表單 → 你直接丟出做好的版本
* 一句話點破差距：**熟練的人不是更會寫 code，是更會把互動變成系統**

### 6) 起手式：學會看豬走路（保留梗，但收一下更精準）

* 「先看別人的流程怎麼跑」
* 「再把 prompt 包成按鈕」
* 「最後才談什麼生態、什麼整合」

### 7) 結尾 CTA

* 讓讀者做一個小作業

  * 「留言：你現在在第幾層？」
  * 「你想仿製哪個系統？（投票）」

## 第 3 篇：程式碼即將免費，你還在賣肝寫 Code？

> 這篇比較像「趨勢＋立場文」。很容易爆，但也最容易被誤解，所以結構要更穩。

### 1) 核心論點（一句就夠）

* 「未來勝負不在 code，在品味與擴散。」

### 2) 類比（手搖飲配方超棒）

* 配方可複製 → 品牌/通路/敘事才是護城河

### 3) 三個黃金能力（這段很適合做圖卡）

* 洞察：聽懂沒說出口的需求
* 品味：懂取捨，知道何時夠好
* 擴散：讓市場看見並願意買單

### 4) 給工程師/創業者/管理者各一句話

* 工程師：抬頭看市場，技術是槓桿不是全部
* 創業者：你要練的是「翻譯官」能力（人話→規格）
* 管理者：別只考題，找能跨界的人

### 5) 收尾（把立場收得溫柔一點）

* 「不是唱衰技術，是提醒方向盤更重要。」

# 心法內化（五歲小孩也懂）

* AI 就像一台很快的機器。
* 你可以讓它做很多東西，但你要站在旁邊看：

  * **哪裡會燙到人？哪裡會爆炸？哪裡只是貼貼紙？**
* 危險的地方你就盯緊一點。安全的地方就讓它快一點。這樣你才不會又快又翻車。
