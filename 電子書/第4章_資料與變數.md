## 盒子的比喻

我給阿威看了我桌上的收納盒。

「你看，這個盒子上貼著標籤『筆』，裡面放著各種筆。這個貼著『名片』，裡面放著收集來的名片。」

他點點頭。

「變數就是這樣的盒子。標籤是變數的名字，裡面的東西是變數的值。」

```python
筆 = "原子筆"
名片 = "王小明的名片"
```

「但為什麼叫『變』數？」他追問。

我把筆盒裡的原子筆換成鉛筆。

「盒子還是同一個盒子，標籤還是『筆』，但內容變了。這就是為什麼叫變數——內容可以變。」

```python
筆 = "原子筆"
# 過了一會兒...
筆 = "鉛筆"
# 盒子（變數）沒變，但內容（值）變了
```

阿威恍然大悟：「所以變數就像Photoshop的圖層名稱，名稱不變，但圖層內容可以隨時更換！」

沒錯，每個領域都有類似的概念，只是叫法不同。

## 資料的形態

如果變數是盒子，那資料就是放進盒子的東西。而不同的東西，需要不同的存放方式。

### 數字：精確的朋友

數字是最單純的資料。

```python
年齡 = 35
薪水 = 50000
溫度 = 23.5
```

小美發現了一個有趣的事：「為什麼有時候數字有小數點，有時候沒有？」

這就像問：為什麼有時候我們說「3個人」而不是「3.5個人」？

- 整數（Integer）：完整的數字，用來數東西
- 浮點數（Float）：有小數點的數字，用來測量

你不會說你有2.5個小孩，但你會說今天氣溫25.5度。

程式也是這樣。庫存數量用整數，價格用浮點數。選對了，程式更清晰；選錯了，可能出現奇怪的結果。

### 文字：有故事的資料

```python
姓名 = "小美"
公司 = "快樂行銷有限公司"
座右銘 = "每天進步1%"
```

文字（String）是用引號包起來的任何東西。

阿威問：「為什麼要用引號？」

「因為要告訴電腦：這是文字，不是指令。」

看看這個差別：

```python
print(Hello)    # 電腦會找一個叫Hello的變數
print("Hello")  # 電腦知道要顯示Hello這個字
```

就像在對話中，我們用引號標示別人說的話：
小美說：「我要學程式。」

引號告訴讀者：這是原話，不是我的詮釋。

### 布林：是與否的世界

這是最簡單卻最強大的資料型態。

```python
已婚 = True
有駕照 = False
是VIP = True
```

布林（Boolean）只有兩個值：True（真）或False（假）。

聽起來很限制，但想想看，多少重要決定都是二選一：
- 要不要買？
- 符不符合條件？
- 該不該執行？

小美在處理促銷活動時，大量使用布林：

```python
滿千送百 = (消費金額 >= 1000)
可以打折 = 是會員 and 非特價品
需要補貨 = (庫存 < 10)
```

每個複雜的商業邏輯，拆解到最後都是一連串的是與否。

### 清單：井然有序的集合

```python
員工名單 = ["小美", "阿威", "老陳", "小李"]
本月業績 = [120000, 98000, 150000, 87000]
待辦事項 = ["開會", "寫報告", "回覆客戶"]
```

清單（List）就像購物清單，可以放很多東西，而且有順序。

阿威特別喜歡清單，因為它像他的設計專案：

```python
設計流程 = [
    "了解需求",
    "研究競品",
    "草圖發想",
    "設計初稿",
    "客戶反饋",
    "修改定稿"
]
```

清單的神奇之處在於，你可以：
- 加入新項目（append）
- 移除項目（remove）
- 重新排序（sort）
- 取出特定位置的項目（用索引）

就像整理你的書架，隨時可以加書、抽書、重新排列。

## 命名的藝術

「我可以把變數命名為a、b、c嗎？」小美問。

「可以，但不應該。」

看看這兩段程式：

```python
# 糟糕的命名
a = 50000
b = 0.1
c = a * b

# 清晰的命名
月薪 = 50000
稅率 = 0.1
應繳稅額 = 月薪 * 稅率
```

哪個更容易理解？

好的變數名稱就像好的標籤，讓人一看就懂裡面裝什麼。

我給他們一個原則：「如果三個月後的你看不懂，那就是壞名字。」

阿威深有感觸：「就像我的設計檔案，如果命名為『最終版』、『最終版2』、『最最終版』，過幾天連我自己都搞不清楚哪個是真的最終版。」

## 資料的生命週期

變數不是永恆的。它們有生有死。

### 誕生

```python
# 變數誕生的瞬間
客戶名稱 = "王大明"
```

### 成長與改變

```python
訂單金額 = 1000
訂單金額 = 訂單金額 + 500  # 追加訂購
訂單金額 = 訂單金額 * 0.9   # 打九折
```

### 死亡

當程式結束，或變數超出範圍，它就消失了。

```python
def 處理訂單():
    臨時折扣 = 0.8  # 這個變數只在函數內存在
    return 原價 * 臨時折扣

# 函數結束，臨時折扣消失了
print(臨時折扣)  # 錯誤！找不到這個變數
```

小美說：「這就像會議室的白板，會議中寫滿東西，會議結束就擦掉了。」

正是如此。理解變數的生命週期，能幫你避免很多錯誤。

## 型態的重要性

有一天，小美遇到了一個詭異的問題：

```python
數量 = input("請輸入購買數量：")  # 使用者輸入10
價格 = 100
總價 = 數量 * 價格
```

她期待總價是1000，結果卻出現錯誤。

問題在哪？`input()`回傳的是文字，不是數字。「10」（文字）和10（數字）對電腦來說是不同的東西。

```python
"10" * 100  # 錯誤！不能用文字乘以數字
10 * 100    # 正確！得到1000
```

解決方法：

```python
數量 = int(input("請輸入購買數量："))  # 轉換成整數
```

這個錯誤教會了小美一個重要觀念：資料型態很重要。

就像cooking，鹽和糖看起來都是白色粉末，但你絕不會想搞混它們。

## 資料的靈活運用

當你理解了資料的本質，你就能靈活運用它們。

### 組合不同型態

```python
# 組合成有意義的資訊
姓名 = "小美"
年齡 = 35
部門 = "行銷部"

員工資訊 = f"{姓名}，{年齡}歲，{部門}"
# 結果："小美，35歲，行銷部"
```

### 資料轉換

```python
# 數字轉文字
金額 = 1000
顯示文字 = "總計：" + str(金額) + "元"

# 文字轉數字
使用者輸入 = "25"
實際年齡 = int(使用者輸入)

# 任何東西轉布林
有庫存 = bool(庫存數量)  # 0會變成False，其他數字變成True
```

### 資料驗證

```python
def 檢查email(email):
    有小老鼠 = "@" in email
    有點 = "." in email
    return 有小老鼠 and 有點

客戶email = "wang@example.com"
if 檢查email(客戶email):
    發送郵件()
else:
    顯示錯誤("Email格式不正確")
```

## 資料告訴我們的故事

老陳在銀行工作，他對資料有不同的看法：

「每筆資料背後都是一個故事。交易金額不只是數字，它可能是某人的第一份薪水、結婚基金、或是孩子的學費。」

這個觀點改變了大家看待資料的方式。

```python
# 不只是資料，而是故事
交易紀錄 = {
    "金額": 30000,
    "時間": "2024-06-01",
    "備註": "女兒大學註冊費",
    "帳戶餘額": 150000
}

# 資料可以回答問題
平均消費 = sum(月消費) / len(月消費)
最大支出 = max(所有支出)
存款趨勢 = 這個月餘額 - 上個月餘額
```

資料不是冰冷的數字和文字，而是幫助我們理解世界、做出決策的工具。
